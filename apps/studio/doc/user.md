用户文档：写作云 3R 工作台

概述

写作云 3R 工作台是一个面向叙事写作的分析与改写工具。它提供两层分析：

· **本地分析**（离线，即时）：基于正则引擎的空隙检测、风格评分和改写建议
· **深度分析**（在线，按需）：连接写作云 Provider 服务，基于 LLM 的语义分析

核心概念：3R

· Review（评审）：自动检测文本中的叙事空隙（时间跳跃、对话间隙、动作空隙、过渡压缩），并计算风格评分。
· Reflect（情境）：针对每个空隙生成引导问题，帮助写作者思考"这里可以写什么"。
· Rewrite（改写）：根据分析结果提供具体的改写建议（删除情绪标签、调整结尾、平衡对话/动作比例）。

界面布局

```
┌─────────────────────────────────────────────────────┐
│  ✎ 写作云 合成工作台   [▶ 评审] [🧠 深度分析] [加载样本]│  ← 顶栏
├────────┬────────────────────────┬──────────────────┤
│ 📄 底稿 │                        │  Review  轮次 1   │
│        │  文本编辑器              │  评分 50          │
│ 咖啡厅  │  (Markdown 编辑/预览)   │                   │
│ 重逢    │                        │  📋 评审 🎯 情境  │
│        │  ·                      │  ✏️ 改写           │
│        │  ·                      │                   │
│        │  空隙 · · ·            │  空隙列表         │
│        │                        │  风格进度条       │
│        │                        │  ───────────      │
│        │                        │  深度分析结果      │
│        │                        │  段落结构 + 建议   │
├────────┴────────────────────────┴──────────────────┤

└─────────────────────────────────────────────────────┘
```

三栏布局：
· 左栏：底稿列表
· 中栏：Markdown 编辑器 + 空隙标记列（左侧圆点）
· 右栏：3R 分析面板（三标签页切换 + 深度分析结果区域）

所有面板之间的分隔条可拖拽调节宽度。

使用流程

1. 加载或输入文本

· 点击"加载样本"载入示例文章进行体验。
· 直接在编辑器区域输入或粘贴你自己的文本。

2. 运行本地评审

点击顶栏的"▶ 评审"按钮，系统会自动：

· 逐行扫描文本，检测 4 类叙事空隙
· 计算 3 项风格评分
· 生成综合评分（满分 100）

评审结果会同时更新到三个标签页中。此步骤完全在本地完成，无需网络。

3. 查看评审结果（Review 标签）

空隙列表：显示检测到的叙事空隙，每条标注类型和所在行号。点击可跳转到编辑器的对应行。

风格评分：
· 对话→动作：叙事中动作描写占比越高，评分越高
· 状态句结尾：结尾是否停在未化解的感受或动作上
· 半秒钟密度：空隙密度反映节奏感

综合评分：三项风格评分的平均分。

4. 运行深度分析（可选）

点击顶栏的"🧠 深度分析"按钮，将文本发送给写作云 Provider 服务进行语义级分析（需先启动 provider 服务并以 `--dart-define=PROVIDER_URL` 指定地址）。

深度分析返回：
· 段落叙事结构标注（起/承/转/合）
· 每段的分析评语
· 改进建议列表（含优先级编号 P1/P2/P3）

结果显示在评审标签页底部，以分割线与本地分析结果隔开。

5. 阅读情境引导（Reflect 标签）

每个高分空隙都会生成一条引导问题，帮助你思考如何充实该处：

· 时间跳跃 → "房间里有什么声音？光线有变化吗？"
· 对话间隙 → "说话的人在开口前做了什么？"
· 动作空隙 → "第一个动作结束时，角色的手在哪？"
· 过渡压缩 → "把它展开，让读者体验每一秒"

点击"✎ 写在这里"可跳转到编辑器的对应位置开始写作。

6. 查看改写建议（Rewrite 标签）

系统根据分析结果自动生成具体建议：

· 情绪标签建议：识别出"悲伤地""开心地"等副词，建议用身体动作替代
· 结尾建议：如果结尾落在情节推进上，建议停在更紧张的时刻
· 对话/动作平衡：如果对话占比偏高，建议在角色说话之间插入动作

点击"✎ 定位到此处"可跳转到对应位置进行修改。

7. 迭代改进

根据建议修改文本后，再次点击"▶ 评审"重新分析，验证改进效果。

8. 编辑 / 预览切换

工具栏右侧提供"编辑" / "预览"切换按钮：
· 编辑模式：显示文本编辑器和空隙标记
· 预览模式：将 Markdown 渲染为格式化文档

常见场景

· 短篇叙事诊断：写完一段故事后评审，发现节奏问题和结构空隙。
· 改写迭代：根据 Reflect 引导和 Rewrite 建议逐步修改，每轮评审验证提升。
· 深度分析：当需要语义级结构诊断时，启动 provider 服务进行深度分析。
· 风格训练：关注风格评分的变化，培养对对话/动作比例、结尾处理等技巧的敏感度。

启动方式

```bash
# 仅本地分析（离线）
flutter run -d chrome

# 本地 + 深度分析（需先启动 provider 服务）
flutter run -d chrome --dart-define=PROVIDER_URL=http://localhost:9000
```

注意事项

· 本地评审基于正则引擎，无需网络连接，响应即时。
· 深度分析需要连接写作云 Provider 服务，未配置时按钮可点击但会提示错误。
· 空隙检测针对中文叙事文本设计，英文支持有限。
· 风格评分仅供参考，创意写作没有绝对标准。
