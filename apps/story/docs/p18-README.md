# p18 — 角色驱动的母题推理实验

验证：从文本中提取结构化角色档案后，能否基于角色特质、动机和关系动态推理出自然的情节走向。

## 假设

1. 角色的性格特质、内隐动机和关系张力编码了情节发展方向——给定角色状态，可推理的下一个情节不是任意的，而是被角色"限定"的
2. LLM 能从单篇或多篇场景文本中提取与 story.yaml 一致的结构化角色档案
3. 基于提取的角色档案推理的情节，与实际的后续成稿情节在核心 beat 上有可检测的吻合度
4. 角色驱动力（人物做某事的内在理由）比母题约束力（系列反复出现的主题元素）更能预测具体情节走向

## 背景

前序实验建立了两种情节分析路径：

| 路径 | 视角 | 代表实验 |
|------|------|---------|
| 母题驱动 | 跨场景的重复元素约束情节走向 | p05-p10 |
| 场景诊断 | 场景内部因果链的断裂修复 | p14 |

但两者都缺少**角色视角**——情节最直接的驱动力是角色在特定情境下"会做什么"。"社交陌生人+心理熟人"这个核心约束之所以有效，不是因为它是母题，而是因为两个角色的性格和处境决定了他们只能以这种方式互动。

p18 追问：如果我们直接从文本中提取角色（而不是依赖人工定义的 story.yaml），LLM 能否基于这些角色推理出符合他们性格的情节？

## 数据来源

### 角色 Ground Truth

| 来源 | 内容 |
|------|------|
| `assets/fiction/story.yaml` → `characters[]` | 角色名 + 描述（性格、处境、核心矛盾） |
| `assets/fiction/story.yaml` → `plots[].tensions` | 每场戏的角色内心冲突和社交边界 |

### 测试文章

从成稿中选取 6 篇覆盖角色弧线关键节点的场景：

| # | 场景 | 角色事件 | 用作 |
|---|------|---------|------|
| T1 | 1_1 咖啡厅重逢 | 初遇/触发 | 提取角色初始状态 |
| T2 | 4_1 便利店谈心 | 第一次深入交流 | 提取关系变化 |
| T3 | 6_2 海边散步 | 自白/第一次肢体接触 | 推理下一场 |
| T4 | 7_2 公园拥抱 | 拥抱/情感突破 | 推理下一场 |
| T5 | 8_2 酒吧表白 | 表白/关系确认 | 评估角色一致性 |
| T6 | 4_2 夜市约会 | 日常亲密/试探 | 交叉验证 |

### 推理目标

对 T1-T4，以当前场景的文本为输入：
1. 提取该场景中两个角色的状态
2. 推理下一个自然的情节走向（核心 beat + 角色动机解释）
3. 与实际下一个成稿场景（story.yaml 中的下一场）对比吻合度

## 实验步骤

```
步骤 1: 角色状态提取
  └─ 对 T1-T4 逐篇，让 LLM 从场景文本中提取结构化角色状态
  └─ 不提供 story.yaml——纯文本输入
  └─ 输出格式：

      {
        "characters": [
          {
            "name": "林远亭",
            "state": {
              "当前情感": "从重逢的惊讶转向小心翼翼的试探",
              "行为倾向": "想靠近但不断提醒自己'她是陌生人'",
              "未说出口的话": "我每天都在想你，但我不敢告诉你",
              "对对方的态度": "十年暗恋的对象突然出现在面前，不知所措"
            },
            "arc_signal": "从'独自承受'到'开始接受被关心'的临界点"
          }
        ],
        "relationship": {
          "当前阶段": "社交陌生人 + 心理熟人",
          "核心张力": "他知道自己喜欢她，但她是否知道/接受？",
          "已突破的边界": ["递毛巾（身体的越界）"],
          "未突破的边界": ["承认'我记得你'", "问'你为什么来'"]
        }
      }

步骤 2: 情节推理
  └─ 基于步骤 1 提取的角色状态，推理下一个自然的情节走向
  └─ 约束：推理必须完全基于角色状态，不允许引入外部知识
  └─ LLM 推理 prompt：

      "以下是{场景名}结束时两个角色的状态。请推理他们下一场戏最可能的
       情节走向。推理必须基于角色性格和当前关系张力，而不是通用叙事套路。
       输出格式：核心事件（一句话）→ 角色动机拆解 → 具体场景设想"

  └─ 输出格式：

      {
        "inferred_next": {
          "core_beat": "他主动发第一条消息",
          "motivation_breakdown": {
            "林远亭": "海边散步时她拉他跑、听了他最深的孤独——他第一次感受到被接纳。夜里反复回想，想确认她是否真的在意",
            "陆知微": "她等了十年，现在已经陪他走到海边，在等他走完最后一步"
          },
          "predicted_scene": "深夜，他打开和她的对话框，打了又删——最后发了一个表情包。她秒回。",
          "tension_remaining": "他们仍以'朋友'名义互动，'喜欢'两个字还压在舌尖"
        },
        "alternative": {
          "core_beat": "她主动约他",
          "reason": "如果陆知微的勇气 > 林远亭的犹豫，她会先迈出一步",
          "trigger": "她发现他发了朋友圈但没给她看，意识到自己还在'陌生人'行列"
        }
      }

  └─ 每个场景输出 1 个主推理 + 1 个替代推理（基于不同角色勇气值假设）

步骤 3: 与 story.yaml 对比
  └─ 将推理的核心事件与实际 story.yaml 中下一场景的 title + description 对比
  └─ 评估维度：

  | 维度 | 定义 | 方法 |
  |------|------|------|
  | beat 吻合 | 推理的核心事件与实际的下一场景是否匹配 | 人工判断（是/部分/否） |
  | 动机一致 | 推理中归因的角色动机与 story.yaml 的 tensions 是否一致 | embedding 余弦相似度 |
  | 张力延续 | 推理保留的未解决张力是否与实际下一场景的 tensions 重叠 | Jaccard 相似度 |
  | 替代合理性 | 替代推理是否同样合理（即使与实际不同） | 人工 1-5 评分 |

  └─ 对 T3→T4（海边散步→公园拥抱）和 T4→T5（公园拥抱→酒吧表白）做深度分析
  └─ 对 T1→T2（咖啡厅→深夜失眠）作为基准——两个场景几乎同时发生，预期 beat 吻合度低

步骤 4: 反向验证
  └─ 给定一个后续场景的文本，让 LLM 反向推理"前一场景结束时角色的状态是什么"
  └─ 对比正向提取（步骤 1）和反向推理的角色状态是否一致
  └─ 一致 → 角色的行为是内在驱动的（角色一致性高）
  └─ 不一致 → 情节可能依赖外部事件而非角色驱动

步骤 5: 角色驱动的边界测试
  └─ 极端情况 1：给定一个完全未知的角色设定（如从校园言情提取的角色），
      推理都市言情场景——跨作品推理是否有效？
  └─ 极端情况 2：只给角色名 + 一句话设定，不给任何场景文本，
      推理整条情节线——角色设定是否足够约束情节？
```

## 验收标准

### 角色提取

| 指标 | 合格线 | 良好线 | 优秀线 |
|------|--------|--------|--------|
| 提取的"核心张力"与 story.yaml 的 tensions 重合 | ≥ 60% | ≥ 75% | ≥ 90% |
| 提取的角色"未说出口的话"在后续场景中被验证 | ≥ 1/场景 | ≥ 2/场景 | ≥ 3/场景 |
| Arc signal 与人工标注的角色弧线方向一致率 | ≥ 50% | ≥ 70% | ≥ 85% |

### 情节推理

| 指标 | 合格线 | 良好线 | 优秀线 |
|------|--------|--------|--------|
| beat 吻合率（主推理与实际的下一场景） | ≥ 40% | ≥ 60% | ≥ 80% |
| 动机一致性（推理归因 vs story.yaml tensions） | Spearman ρ ≥ 0.4 | ≥ 0.6 | ≥ 0.8 |
| 张力延续（保留的未解决张力与实际重叠率） | ≥ 50% | ≥ 70% | ≥ 85% |
| 替代推理合理性（人工评分 1-5） | ≥ 3 | ≥ 4 | ≥ 4.5 |
| 至少 1 个场景的替代推理被评估为"比实际更有趣" | — | 1 个 | 2+ 个 |

### 反向验证

| 指标 | 合格线 | 良好线 | 优秀线 |
|------|--------|--------|--------|
| 正向提取 vs 反向推理的角色状态一致性 | ≥ 50% | ≥ 70% | ≥ 85% |

### 边界测试

| 指标 | 预期 |
|------|------|
| 跨作品推理 | 角色设定差异过大时推理无效——这是确认"角色驱动"边界的重要发现 |
| 仅角色名推理 | 无法生成具体情节——说明角色必须与具体情境交互才能产生情节 |

### 输出产物

1. `results/char_states_T1.json` ~ `results/char_states_T4.json`（角色状态提取结果）
2. `results/inference_T1.json` ~ `results/inference_T4.json`（主推理 + 替代推理）
3. `results/comparison.json`（与 story.yaml 的对比 + 各维度评分）
4. `results/reverse_validation.json`（反向验证结果）
5. `results/boundary_tests.json`（边界测试结果）

## 不依赖

纯 Python 脚本 + LLM API。依赖：
- `assets/fiction/story.yaml`（仅用于评估阶段，不用于提取）
- `assets/fiction/*.md`（场景文本）

## 结果预期

- **最强信号**：角色提取的"核心张力"应与 story.yaml 的 tensions 高度重合（≥ 70%），因为 tensions 本身就是从文本中归纳的角色内心冲突
- **最关键的检验**：T3(海边散步)→T4(公园拥抱) 的推理。海边散步结束时，他第一次完整坦白孤独、她拉他跑、他拽她手——这是"陌生人"防线最薄弱的时刻。LLM 应能推理出"下一场是某种形式的拥抱或肢体安慰"。
- **最难的推理**：T4(公园拥抱)→T5(酒吧表白)。公园拥抱中她说"以后我陪你"，已经给出了承诺但未解除"陌生人"身份。从"我陪你"到"我喜欢你"的跨越需要勇气突破——LLM 必须判断谁会先开口，在什么场景下开口。
- **最有趣的发现**：如果替代推理（基于不同角色勇气值假设）被人工评为"比实际更有趣"，说明实际作者选择了一条更克制/更安全的路径——这本身就是关于作者审美的重要洞察。
- **失败条件**：
  - 如果 beat 吻合率 < 40%，说明角色状态不足以约束情节——情节可能更多依赖外部事件（母题/作者意图）而非角色驱动
  - 如果反向验证一致性 < 50%，说明角色行为前后不一致——要么是 LLM 提取不准，要么是文本本身角色刻画不够一致

## 产品意义

如果 p18 验证成功：
1. **角色档案成为情节预测器**——给定一组角色状态，系统可以建议"最符合角色的下一步"
2. **角色一致性的自动化检测**——反向验证可作为检测角色行为是否"出戏"的工具
3. **作者可探索"角色分支"**——"如果角色在此刻做出了不同选择，情节会怎么走？"
