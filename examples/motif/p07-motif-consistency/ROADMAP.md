# p07 ROADMAP

## 当前状态

约束组吻合度高于对照组（+20~45%），方向确认 motif 具有生成约束力。但 keyword-map 检测同时高估和低估了吻合度，净效果难以精确定量。具象母题（雨、论坛）强，抽象母题（孤独、歌声、协作书写）弱。

## 立即修复 (P0)

### 修复 ①：检测算法从 keyword map 升级为语义匹配
**文件**：`run.py` — `compute_alignment()`
**问题**：
- keyword map 过宽："身体"匹配"手势"、"关怀"匹配"随身携带的温柔" → false positive 抬高对照组
- keyword map 遗漏："围巾"不匹配"外套"、"随时都带着"不匹配"准备" → false negative 压低约束组
**修复**：用 LLM 语义判断替代 keyword 匹配：
```python
prompt = f"判断以下提取的母题'{detected_title}'是否体现了目标母题'{target_title}'的含义。" \
         f"目标母题定义：{target_description}。只输出 yes/no。"
```
**预期影响**：约束组吻合度提升 10-20%，对照组吻合度下降 5-10%，净差异从 +20~45% 扩大至 +35~55%（进入良好线）

### 修复 ②：交叉验证检测管线
**READM 要求**：用 GPT-4o-mini 做交叉验证，消除同源偏差（DeepSeek 生成 → DeepSeek 检测）
**实施**：在步骤 4 检测阶段，对同一批文本同时用 DeepSeek 和 GPT-4o-mini 做母题检测，对比两组吻合度
**预期**：如果差异 >15%，最终报告应优先采信 GPT-4o-mini 的结果（打破同源偏差）

## 短期改进 (P1)

### 改进 ③：场景模板重新设计
**问题**：5 个现有场景模板无法触发"歌声""协作书写""旁观者"——不是生成问题，是模板设计问题
**实施**：为每个低覆盖率母题设计专属场景模板：

| 母题 | 当前覆盖 | 新增场景模板 |
|------|---------|-------------|
| 歌声 | 0% | "KTV唱歌" / "音箱随机播放了那首歌" |
| 协作书写 | 0% | "一起起草一份声明" / "共同编辑文档到深夜" |
| 旁观者 | 0% | "闺蜜盘问" / "同事窃窃私语" / "帖子被转发" |
| 孤独 | 0% | "独自加班到深夜" / "翻看旧照片" |

**预期**：场景模板匹配后，这些母题的覆盖率从 0% → 40-60%

### 改进 ④：抽象母题翻译为可执行约束
**问题**："孤独"的母题定义是分析性的（"读者从文本中推断的"），而非生成性的（"写作者刻意加入的"）
**实施**：将 gallery 中的母题 description 改写成行为指令：

| 母题 | 原始 description（分析性） | 改写为（生成性） |
|------|------------------------|---------------|
| 孤独 | 全书最高级的情感表达不在台词里，在手里 | "写一段内心独白，角色觉得无人可说、说不出来" |
| 旁观者 | CP粉/闺蜜/室友构成叙事的有机部分 | "在场景中插入一个第三方角色的评论或反应" |
| 随身携带的温柔 | 随身物品承载情感 | "让一个角色在另一个角色需要时恰好掏出某个物品" |

**预期**：生成性约束使覆盖率提升 20-30%

## 中期探索 (P2)

### 探索 ⑤：母题约束力与文本质量的相关性
**发现**：约束组文本比对照组长 50%+、更有质感——但这是相关性还是因果性？
**实验**：
1. 对 20 篇生成文本做 2 人独立质量评分（1-5 分、IAA 计算）
2. 对比"吻合度 × 质量评分"的 scatter plot
3. 检验：高吻合度 = 高质量？还是约束本身(无论吻合度) = 高质量？
**产品含义**：决定 motif 约束是"质量提升工具"还是"一致性保证工具"

### 探索 ⑥：motif 约束 vs 风格约束的交互
**背景**：p07 仅测试 motif 约束，但产品中 motif + style 会同时存在
**实验**：4 组：①无约束 ②仅 motif ③仅 style ④motif + style
**产品含义**：motif 和 style 的联合约束是否 1+1 > 2？

### 探索 ⑦：生成场景的跨场景连贯性
**读 READM 要求**："场景不矛盾 → 角色性格一致 → 可拼成一篇完整故事"
**实施**：将 5 个约束场景拼接，给 LLM 审查：角色名是否一致？时间线是否合理？性格是否连贯？标注 inconsistencies
**产品含义**：如果 5 个场景有 3+ 处 inconsistency → motif 约束需要加入"角色一致性"约束
