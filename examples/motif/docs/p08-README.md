# p08 — 母题缝隙分析与多向改进实验

验证 LLM 能否检测初稿中的母题缝隙，并从多个方向提出差异化改进建议。

## 假设

1. 初稿必然存在母题缝隙——某些场景缺少该系列应有的母题，或某个母题在所有场景中分布不均
2. LLM 能检测这些缝隙，并解释"为什么这里缺了那个母题"
3. LLM 能从不同方向提出差异化改进建议（增强/引入/借用/转化/克制/反向），而非单一"加入 X 母题"的单调建议
4. 不同方向的建议在质量、可行性和母题覆盖率上有显著差异

## 背景

p05 证明母题可被提取，p06 证明母题可跨作品识别，p07 证明母题可作为生成约束。p08 合三者为一：**输入初稿 → 检测母题期望与实际的差距 → 输出多方向改进建议**。

这是三个前序实验的产品化整合——不验证新能力，而是验证能力组合后的实用性。

### 测试初稿（3+1 篇）

从初稿中选取覆盖不同母题覆盖率的文章，确保有足够缝隙可分析：

| # | 来源 | 场景 | 文件 | 母题覆盖率 |
|---|------|------|------|-----------|
| D1 | 都市言情 | 咖啡厅重逢 | `assets/fiction/职场言情/3_初稿/1_1_咖啡厅重逢.md` | p05 已知 60%（缺孤独、歌声） |
| D2 | 都市言情 | 酒吧表白 | `assets/fiction/职场言情/3_初稿/8_2_酒吧表白.md` | p05 已知 60%（缺手势、雨） |
| D3 | 校园言情 | 第六章 | `assets/fiction/校园言情/3_初稿/6_第六章.md` | **盲测**——未参与 p05，覆盖率未知 |
| D4 | 校园言情 | 第十章（KTV） | `assets/fiction/校园言情/4_成稿/10_第十章.md` | p05 已知 25%（仅 1/4） |

选择理由：
- D1 缺低 weight 母题（孤独、歌声，weight 5-7）——测试"克制"方向
- D2 缺高 weight 母题（手势，weight=9）——测试"增强/引入"方向，这是真正的写作失误
- D3 是真正的盲测——系统对一篇从未见过的文章能做什么？
- D4 大面积缺失——测试系统面对"场景类型不匹配"时的建议质量

## 实验步骤

```
步骤 1: 母题提取
  └─ 对 D1-D3 提取母题（复用 p05 的提取 prompt）
  └─ 输出每篇文章的实际母题 profile
  └─ 对比 target profile → 生成"母题报告卡"：
      ├─ 已覆盖母题（title + LLM 提取的 weight vs gallery weight）
      ├─ 缺失母题（gallery 有但 draft 未提取到的）
      └─ 弱母题（被提取但 LLM weight < gallery weight*0.5）

步骤 2: 缝隙归因
  └─ 对每个缺失/弱化母题，让 LLM 分析原因（支持多归因）：
      "这篇场景缺少母题 X。请分析原因——
       可能是场景类型不支持？可能是作者用了替代母题？可能是单纯遗漏？
       可以同时标注多个原因。"
  └─ 输出格式：{"motif": "孤独",
      "gap_types": ["scene_incompatible", "alternative_used"],  // 数组，支持多归因
      "alternative_motif": "手势（如果被手势替代）",
      "reasoning": "..."}

步骤 3: 多向改进建议生成
  └─ 对每篇文章的每个缝隙，从 **6 个方向** 各生成一条改进建议：

  | 方向 | 定义 | 示例（D1 缺"歌声"） | 触发条件 |
  |------|------|-------------------|---------|
  | **增强** | 强化已存在但偏弱的母题 | "第 8 段他写日记的场景可以加入一段歌词——他写了一句，删掉，又写了一句" | 目标母题已被检测到但 weight 偏低 |
  | **引入** | 添加一个完全缺失的母题 | "咖啡厅背景音乐可以放一首对他们有意义的歌，让她在对话中停顿一秒" | 目标母题完全缺失 |
  | **借用** | 从跨作品母题镜像（p06）借变体；若无跨作品变体，降级为同系列其他场景的同一母题变体 | "校园版的'随身携带的温柔'用了纸巾——你可以让她从包里掏出一张演唱会门票" | 素材库有可用变体 |
  | **转化** | 将现有场景元素改造成母题载体 | "第 3 段的雨声已经写得很好——让雨声渐弱时，窗外传来街头艺人唱的歌" | 场景中已存在可承载母题的元素 |
  | **克制** | 建议不做某事，保持克制 | "不需要加歌。十年母题已经足够重。再加歌声会 overload 这个场景" | weight ≤ 6 的母题，或场景中已有 2+ 高 weight 母题 |
  | **反向** | 有意违抗母题以制造张力 | "让咖啡厅音乐是一首完全不属于他们年代的歌——她发现他不认识这首歌，形成代际/趣味落差" | 无限制，但需标注风险等级 |

  └─ "反向"方向额外输出 `reverse_risk: 1-3`（1=低风险/微小张力，2=中等风险/需要后续场景弥补，3=高风险/可能破坏人物一致性或场景基调）
  └─ 使用 DeepSeek API，temperature=0.7（需要创意）
  └─ 每条建议 80-150 字 + 具体段落引用

步骤 4: 建议质量评估（两阶段）
  └─ 阶段 A（LLM 自动初评）：
      对全部建议（4 篇 × 1-3 个缝隙 × 6 个方向 ≈ 48-72 条），从 4 个维度做 1-5 评分：

  | 维度 | 定义 |
  |------|------|
  | **可行性** | 这条建议是否真的可以在不改动文章结构的情况下执行？ |
  | **母题契合度** | 按建议修改后，目标母题是否确实会被增强？ |
  | **自然度** | 修改后的场景是否自然，是否显得刻意？ |
  | **可操作性** | 建议是否有明确的执行步骤？创作者无须二次解读就能实施？ |

  └─ 阶段 B（人工抽样验证，18 条）：
      从 LLM 初评结果中按"高/中/低分各 1/3"分层抽样 18 条（1 篇 × 1 缝隙 × 6 方向）
      人工二次评分，计算 4 个维度的 LLM-vs-human 偏差
      同时额外标注"创作价值"维度（需要创作者视角）：
      "作为一个创作者，你会考虑采纳这条建议吗？1-5"

步骤 5: 方向对比分析
  └─ 按方向分组，计算各维度的平均分
  └─ 按文章分组，看不同缝隙类型（缺失高 weight / 低 weight / 多重缺失）的最佳方向是否不同
  └─ 输出"方向-场景适配矩阵"
```

## 验收标准

### 母题报告卡

| 指标 | 合格线 | 良好线 | 优秀线 |
|------|--------|--------|--------|
| 缝隙识别准确率（vs p05 已知覆盖率） | > 80% | > 90% | 100% |
| 缝隙归因合理率（人工审阅 10 条） | > 60% | > 80% | > 90% |

### 多向建议质量

| 指标 | 合格线 | 良好线 | 优秀线 |
|------|--------|--------|--------|
| 各方向平均可行性 | > 2.5 | > 3.5 | > 4.0 |
| 各方向平均母题契合度 | > 2.5 | > 3.5 | > 4.0 |
| 方向间多样性（同篇文章 6 条建议的 Jaccard 距离） | > 0.4 | > 0.6 | > 0.8 |
| LLM 初评 vs 人工复评偏差（avg of 4 维度） | < 1.5 分 | < 1.0 分 | < 0.5 分 |
| 存在至少 1 条"克制"方向建议被评估为"高于引入" | — | 至少 1 篇 | 至少 2 篇 |
| "反向"方向 `reverse_risk` 分布（1/2/3） | — | — | 1 级 > 40%、3 级 < 20% |

### 方向对比

| 指标 | 预期发现 |
|------|---------|
| **增强** 和 **转化** 自然度最高 | 改动小，不引入新元素 |
| **借用** 母题契合度可能最高 | p06 证明跨作品变体结构一致 |
| **引入** 可行性最低 | 凭空加新元素比改造现有元素更难 |
| **反向** 创作价值可能最高 | 违抗母题带来意外性和讨论价值 |
| **克制** 是独特方向 | 不做任何修改本身是一种有意义的建议 |

### 输出产物

1. `results/motif_report_D1.json` ~ `results/motif_report_D4.json`（母题报告卡）
2. `results/gap_attribution_D1.json` ~ `results/gap_attribution_D4.json`（缝隙归因，含 `gap_types` 数组）
3. `results/suggestions_D1.json` ~ `results/suggestions_D4.json`（每个缝隙 × 6 个方向的建议，含 `reverse_risk`）
4. `results/evaluation_llm.json`（LLM 自动初评，4 维度）
5. `results/evaluation_human.json`（人工抽样 18 条，含创作价值维度 + LLM-vs-human 偏差报告）

## 不依赖

纯 Python 脚本 + LLM API。依赖：
- p05 的母题提取 prompt 模板
- `docs/gallery/fiction/urban-romance/motif.yaml`
- `docs/gallery/fiction/campus-romance/motif.yaml`
- p06 的跨作品母题镜像对（用于"借用"方向的素材库；若无跨作品变体则降级为同系列其他场景的同一母题变体）
- `assets/fiction/` 中的初稿文本

## 结果预期

- **缝隙检测**：D1/D2 与 p05 已知覆盖率一致，D4 与 p05 一致（25%），D3 为盲测——覆盖率未知，但预期校园系列的 论坛/协作书写/旁观者 可能在第六章中出现
- **缝隙归因**：D2"缺手势"最可能被归因为 `["genuine_miss"]`（酒吧场景天然适合手势），D1"缺歌声"为 `["scene_incompatible"]`（咖啡厅场景不强依赖音乐），D4 大面积缺失为 `["scene_incompatible"]`
- **方向分化**（最关键）：
  - 增强和转化是"安全建议"（可行性 > 3.5、自然度 > 3.5）
  - 借用是最有趣的建议（引入跨作品元素，创作价值 > 3.5）
  - 克制是最有智慧的"不建议"——对 weight ≤ 6 的母题，克制可能是最优解
  - 反向是最有风险的——`reverse_risk` 预期分布：> 50% 为 2-3 级
- **盲测 D3**：如果 D3 的缝隙分析质量与 D1/D2 无显著差异，说明系统不需要前验知识 —— 这是产品化的关键前提
- **失败条件**：如果所有方向的建议都趋向同质（Jaccard 距离 < 0.3），说明"多方向"是伪多样性 —— 母题缝隙本质上只有一种修复方式

## 产品意义

如果 p08 验证成功，意味着：
1. **母题系统可以成为编辑工具**——不只是事后分析，而是事前提供改进方向
2. **"克制"方向的确立**是重要的产品设计决策——"不做什么"和"做什么"同样有价值
3. **6 方向框架**可以封装为 CLI 命令或 API 端点（`POST /suggest-motif`）
