# p05 ROADMAP

## 立即修复 (P0)

### 修复 ①：覆盖率去重
**文件**：`run.py` — `compare_with_ground_truth()`
**问题**：`matched_count` 用 count 而非 unique set → 都市单篇覆盖率被高估 28%（84%→56%）
**修复**：将 `matched += 1` 后的匹配结果改为 `len(set(unique_gt_matches))`
**预期影响**：覆盖率报告值下降 15-28%，但不改变合格/良好/优秀的判定结论

### 修复 ②：语义匹配改用 embedding 模型
**文件**：`run.py` — `semantic_similarity()`
**问题**：当前用 LLM 判断语义相似度（"LLM 提取 → LLM 评估" 循环论证）
**修复**：引入 `openai.embeddings.create(model="text-embedding-3-small")` 替代 LLM 调用，用余弦相似度做匹配
**预期影响**：消除循环论证风险，匹配结果更稳定（不受 temperature 影响）。同时大幅降低步骤 3 的 API 调用成本（embedding 单次 < LLM 判定调用）

### 修复 ③：盲品平局处理
**文件**：`run.py` — `blind_clustering()`
**问题**：第 3 轮一个组出现 4vs4 平局，代码计为完全正确
**修复**：平局组按 50% 正确率计算（随机基线）
**预期影响**：平均盲品聚类率从 ~79% 可能微降至 ~75%

## 短期改进 (P1)

### 改进 ④：evidence 准确性人工抽查
**READM 要求**：随机抽取 1 篇，人工核对 evidence 是否真的来自原文
**实施**：对 U3（咖啡厅重逢）或 C1（第四章）的 5-6 个母题 × 2-3 条 evidence，逐条标注"准确/部分准确/虚构"
**预期输出**：evidence 准确率报告 → 决定提取 prompt 的 evidence 约束是否需要强化

### 改进 ⑤：人工校准 10 对语义匹配
**READM 要求**：人工抽样验证 LLM/embedding 判定合理性的上界
**实施**：从 `comparison.json` 中随机抽取 10 对（LLM 提取的母题 vs GT 母题），人工标注"匹配/不匹配/存疑"
**预期输出**：人工确认率 vs 代码匹配率对比 → 校准自动化指标

### 改进 ⑥：联合提取相似度映射人工审查
**问题**：联合提取中将"身体接触"映射到"旁观者"（sim=0.85）、"共同创作"映射到"论坛"（sim=0.85），语义上错误
**实施**：对联合提取的每个匹配对做人工纠正，将错误映射从覆盖率中剔除
**预期影响**：校园联合覆盖率从 75% 修正至 ~50%

## 中期探索 (P2)

### 探索 ⑦：单篇 vs 多篇联合提取的 ROI 分析
**发现**：单篇 avg 56-58%、联合 60-75%，差距仅 4-17%
**问题**：联合提取需要多篇文章 × N 次 API 调用，ROI 是否值得？
**实验**：在 1/2/3/5 篇样本量上逐级测试覆盖率增长曲线，找到覆盖率饱和点
**产品含义**：如果 2 篇已经饱和，StyleStore 的"作者风格画像"只需 2 篇代表文章即可

### 探索 ⑧：母题提取的文本窗口敏感性
**发现**：C3（KTV）仅 25% 覆盖率 → 公共/表演性场景母题信号弱
**问题**：截取文本前 3000 字是否足够？不同类型的场景是否需要不同窗口？
**实验**：对 C3 取前 1000/2000/3000/全文 分别提取母题，看覆盖率变化
**产品含义**：确定母题提取的最小可靠窗口
