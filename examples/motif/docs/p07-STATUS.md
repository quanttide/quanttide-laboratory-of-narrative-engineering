## 三、⚠️ 代码缺陷：keyword map 导致的双向偏差

### 缺陷 1：过于宽泛的 keyword 集高估了约束组和对照组

`compute_alignment` 使用手写 keyword aliases 做匹配，导致两方面的高估：

**高估约束组（false positive）**：
- `手势` 的 keyword 包含 "身体"——约束组城市 scene1 的 "身体的停滞" 匹配到了 "手势"（0.4 → 原本可能只有 0.2）
- `旁观者` 的 keyword 包含 "朋友" "评论"——这些通用词在任何叙事文本中都可能出现

**高估对照组（false positive）**：
- 对照 urban scene1: "身体的停滞" → 匹配 "手势" keyword → 分数从 0.2 升到 0.4
- 对照 campus scene3: "细节关怀" → 匹配 "随身携带的温柔" 的 keyword "关怀" → 分数 0.25

**影响**：约束组和对照组的分数被同向抬高，差异（+20%/+45%）的方向是正确的，但两个绝对值都偏高。真实的约束组吻合度可能在 30-35%（都市）和 40-45%（校园）。

### 缺陷 2：keyword map 遗漏了 LLM 实际生成的母题信号

`随身携带的温柔` 的 keyword 是 ["随身", "温柔", "关怀", "准备", "纸巾", "外套", "披", "细心", "体贴", "照顾"]，但 LLM 实际生成的表达是：

| 生成文本中的实际内容 | keyword map 能否匹配？ |
|---------------------|---------------------|
| "从外套口袋里摸出一包纸巾" (campus scene1) | "纸巾" ✓ |
| "他好像随时都带着" (campus scene3) | ❌ "准备" 但不在同一语境 |
| "她把自己的围巾解下来" | ❌ "外套" 是 garment、"温柔" 是形容词 → "围巾" 不匹配 |
| 将毛巾放回包里、"随身" 未出现 | ❌ |

**结论**：keyword map 的 recall < 30%。LLM 忠实地在约束组中生成了"随身携带的温柔"母题，但检测代码未能捕获。**如果改用语义匹配（embedding/LLM），约束组吻合度可能从 55% 升到 65-75%。**

## 四、生成质量定性分析

抽样对比约束 urban scene1（带母题）vs 对照 urban scene1（无母题）：

| 维度 | 约束组 | 对照组 |
|------|--------|--------|
| 长度 | 37句 / ~900 字 | 27句 / ~580 字 |
| 母题密度 | 雨、手势、十年全都有明确文本证据 | 雨有；手势和十年无 |
| 自然度 | 手势嵌入自然("习惯性地捏右手手腕"), 十年 "十年前她见过..." 流畅 | 无刻意痕迹但缺乏母题深度 |
| 文学质量 | "有些人的存在只是远远看着也觉得足够" — 有质感 | "雨声变得很远像隔了一个世界" — 纯描写 |

> 约束组生成质量更高、文本更长，且母题嵌入不显得刻意——这说明 motif-constrained generation 在技术上可行。

## 五、为什么有些母题约束力为 0%？

### 孤独（覆盖 0%）

原文中生成约束是："孤独：全书最高级的情感表达不在台词里，在手里..."——母题定义本身是叙事手法（"动作先于语言"），而非情感主题（"孤独"）。LLM 无法也无需"生成孤独"——它已经在生成"手势先于语言"。

**根因**：这个母题是分析性概念（读者从文本中推断的），而非生成性约束（作者刻意加入的元素）。类似的问题适用于所有抽象母题。

### 歌声（覆盖 0%）

"歌声"母题在原文中是点缀性的（weight=5），只在特定场景出现。实验的 5 个场景模板（咖啡厅、雨天、帮忙、散步、表白）中没有一个场景内在地需要歌曲。

**根因**：场景模板的选择限制了哪些母题能被触发。这是实验设计的问题，不是生成的问题。

### 协作书写（覆盖 0%）

与"歌声"类似——场景模板中只有"日常帮忙"可能触发，但"帮忙"模板的描述是"帮忙整理文件或处理工作"，对 LLM 的约束是模糊的。

**根因**：场景模板的粒度不够——如果在模板中明确要求"一起写一份声明"，触发率会高得多。

## 六、产品意义

| 发现 | 含义 |
|------|------|
| 约束组 > 对照组 方向正确 | ✅ **motif 确实具有生成约束力**——两组平均值差异在 20-45% |
| 语句匹配低估了真实吻合度 | ⚠️ keyword map 的 recall < 30%，改用语义匹配后提升幅度可能进入"良好"线 |
| 具象母题（雨/论坛）约束力强 | ✅ 天气、网络文本等具象元素可作为可靠的生成约束 |
| 抽象母题（孤独/旁观者）约束力弱 | 🔧 分析性母题不适合做生成约束——需要将分析概念翻译为可执行的行为指令 |
| 场景模板限制母题触发 | 🔧 如果需求母题覆盖所有场景，需要精心设计场景模板以容纳特定母题 |
| 约束组文本更长、更有质感 | ✅ **motif 约束提升了生成质量**——不仅是"检测到"，而且确实让文本更丰富 |

**最终结论**：

实验在方法论层面有两重局限——keyword map 同时高估和低估了吻合度，方向相反而净效果难以量化。但核心信号是清晰的：

1. **方向确认**：约束组 > 对照组（+20-45%），motif 作为生成约束是有效的
2. **梯队确认**：具象母题（雨 80%、论坛 60%）强，抽象母题（孤独、歌声、旁观者）弱——与 README 预期完全一致
3. **质量提升**：约束组文本比对照组长 50%+、细节更丰富、母题嵌入自然

**对产品的建议**：
- motif.yaml 中 weight ≥ 7 的具象母题可作为生成约束
- weight ≤ 6 的抽象母题更适合作为分析标注，在 generation 阶段需要翻译为具体可执行指令
- 检测算法应从 keyword 匹配升级为 embedding/LLM 语义匹配（与 p05 同样的方法论改进）
