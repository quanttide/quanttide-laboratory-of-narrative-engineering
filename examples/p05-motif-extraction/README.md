# p05 — 母题可提取性实验

验证 LLM 能否从文本中提取母题，以及提取结果与人工定义的 motif.yaml 的吻合度。

## 假设

1. 同一系列的文章存在可识别的共同母题（如都市言情的"十年""手势""雨"）
2. LLM 从单篇文章中提取的母题与人工从全书提炼的 motif.yaml 部分重合
3. 多篇文章联合提取的母题覆盖率随样本量增加而上升

## 数据来源

案例库 `docs/gallery/fiction/` 提供三层母题标注：

| 层级 | 文件 | 内容 |
|------|------|------|
| 跨系列 | `fiction/motif.yaml` | 6 个共同母题（手势先于语言、日常的缝隙、双向奔赴等） |
| 都市言情 | `fiction/urban-romance/motif.yaml` | 5 个专属母题（十年、手势、雨、孤独、歌声） |
| 校园言情 | `fiction/campus-romance/motif.yaml` | 4 个专属母题（论坛、协作书写、旁观者、随身携带的温柔） |

从两位作者/两个系列的各 5 篇和 3 篇成稿中选取，覆盖不同场景类型：

## 实验步骤

```
步骤 1: 单篇母题提取
  └─ 对 U1-U5 和 C1-C3（共 8 篇），让 LLM 逐篇独立提取母题
  └─ 提示词结构：
      ├─ 母题定义："在叙事中反复出现的主题元素，包括具体意象、关系模式、
      │    行为习惯、叙事惯例。母题与主题不同——主题是'说什么'（如'爱情'），
      │    母题是'怎么说'（如'通过手势而非语言表达爱意'）"
      ├─ 输出约束：每篇提取 3-6 个母题，每个母题须有原文线索支撑
      └─ 输出格式：{
            "motifs": [
              {
                "title": "母题名",
                "description": "一句话描述",
                "weight": 1-10,
                "evidence": ["原文线索1", "原文线索2"]
              }
            ]
          }
  └─ 使用 DeepSeek API（与产品一致），temperature=0.3
  └─ 8 次 LLM 调用，结果存为 `results/motif_u1.json` 等

步骤 2: 多篇联合提取
  └─ 将 U1-U5 合并为一组，C1-C3 合并为另一组
  └─ LLM 从合并文本中提取该系列的共同母题
  └─ 输出格式同步骤 1（但母题应跨场景复现）
  └─ 2 次 LLM 调用，结果存为 `results/motif_urban_joint.json`、`results/motif_campus_joint.json`

步骤 3: 与人工标注对比
  └─ 加载 gallery 中的三层 motif.yaml 作为 ground truth
  └─ 对步骤 1 的单篇提取结果，计算以下指标：
      ├─ 语义匹配率（主指标）：用 text-embedding-3 或同类 embedding 模型
      │    计算 LLM 提取的 description 与人工标注的 description 的余弦相似度，
      │    相似度 > 0.7 视为匹配（避免用 LLM 评估 LLM 输出的循环论证）
      ├─ 精确匹配率（参考指标）：LLM 提取的 title 与人工标注完全一致或音近别名
      │    但由于 LLM 措辞变体较多（如"非言语表达"→"手势先于语言"），
      │    精确匹配率预期很低，不作为主指标
      └─ 覆盖率：人工标注的母题中有多少比例被 LLM 检测到（至少 1 篇覆盖）
  └─ 对步骤 2 的多篇联合提取，计算同样的指标
  └─ 输出对比表：单篇平均覆盖率 vs 多篇联合覆盖率
  └─ 人工抽样验证（至少 10 对）：从语义匹配结果中随机抽取 10 对
      （LLM 提取的母题 vs 人工标注的母题），人工判断匹配是否合理，
      计算人工确认率作为语义匹配率的上界校准
  └─ evidence 准确性抽查：从单篇提取结果中随机抽取 1 篇（如 U3 或 C1），
      人工核对 evidence 字段中的原文线索是否确实来自该文章，
      每条 evidence 标注"准确/部分准确/虚构"

步骤 4: 盲品归因
  └─ 将 8 篇单篇提取结果的母题描述合并，去除系列标签
  └─ 打乱顺序后交给 LLM，要求分成 2 组（每组内文章母题"显著相似"）
  └─ 衡量：同系列文章落入同一组的比例
  └─ 提交 3 次独立运行取平均（降低 LLM 随机性影响）
```

## 验收标准

### 母题提取质量

| 指标 | 合格线 | 良好线 | 优秀线 |
|------|--------|--------|--------|
| 单篇语义匹配率（avg, embedding 计算） | > 40% | > 60% | > 80% |
| 多篇联合覆盖率 | > 60% | > 80% | > 95% |
| 单篇 vs 多篇覆盖率提升 | > 10% | > 20% | > 30% |
| 人工确认率（10 对抽样） | > 70% | > 85% | > 95% |
| evidence 准确性（抽样 1 篇） | > 60% 准确 | > 80% 准确 | > 90% 准确 |

### 盲品归因

| 指标 | 合格线 | 良好线 | 优秀线 |
|------|--------|--------|--------|
| 同系列聚类率 | > 50%（随机基线） | > 75%（6/8） | > 87%（7/8） |

### 输出产物

1. `results/motif_u1.json` ~ `results/motif_c3.json`（8 篇单篇提取结果）
2. `results/motif_urban_joint.json`、`results/motif_campus_joint.json`（2 篇联合提取结果）
3. `results/comparison.json`（单篇 vs 多篇 vs 人工标注 三栏对比表）
4. `results/blind_clustering.json`（3 次盲品聚类结果）

## 结果预期

- **最可能**：单篇语义匹配率 50-70%。单篇文章只覆盖该系列母题的子集（如 U1 咖啡厅重逢中"十年"和"手势"明显，但"雨"和"歌声"可能不出现）。多篇联合覆盖率 > 80%。
- **关键发现**：都市言情的"手势"和校园言情的"随身携带的温柔"可能被 LLM 检测为语义相近的母题（都是"动作承载情感"），这恰好验证了 gallery 跨系列 motif.yaml 中"手势先于语言"的共同母题——如果 LLM 也能自主发现这个跨作品模式，说明母题具有客观性而非人工附会。
- **方法论注意**：步骤 3 的语义匹配使用 embedding 模型（如 text-embedding-3）而非 LLM，用于打破"LLM 提取 → LLM 评估"的循环论证。人工抽样 10 对对各评估方法做校准。
- **如果失败**（语义匹配率 < 40%）：说明"母题"在这个文本粒度上不可被 LLM 稳定提取，MoLoRA（Motif-LoRA）或 motif-aware 写作助手的基础假设需要重新审视。可能需要更大的文本窗口或更高层的摘要来提取母题。
- **如果盲品归因 > 75%**：母题可作为 StyleStore 的作品指纹，用于"按母题相似度推荐作品"。
