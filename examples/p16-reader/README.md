# E4 — 模拟读者系统：架构与验证

> 注：本文档对应 PoC 编号 p16-reader。

## 概述

本蓝图定义"模拟读者"是什么、如何构建、如何验证。

**一句话定义**：模拟读者是一个可配置模型，给定文本 + 读者画像 → 输出该读者视角下的评价分布（含置信区间）。

### 为什么是多画像视角：单一评分的局限

传统作品评价依赖单一评分（平均分或专家评分），但实验数据揭示了一个根本问题：

> **作品质量不是一维的，单一评分将"谁在读"的信息当作噪声丢弃了。**

E4-1 实验的 η²=0.219 表明：**22% 的评分差异来自"谁在读"**，而非文本质量本身。同一篇文本，不同画像在四个评分维度的分布是完全不同的信号：

| | 文笔 | 情感冲击 | 角色真实感 | 套路感 |
|:---|---|---|---|---|
| P1 甜宠少女 | 5.44 | **6.0** | 5.22 | 3.78 |
| P3 资深老书虫 | 5.11 | 5.39 | **5.94** | **1.72** |

P1 关注"我心跳加速了吗"（情感冲击最高），P3 关注"角色立得住吗"（角色真实感最高）。**同一个 8.6 分，对两类读者意味着完全不同的阅读体验。**

多画像视角的准确性在于：

1. **测量了不同的事物**：单一评分是"平均喜好的估计值"，多画像测量的是**文本在读者空间中的结构位置**——不是一个数字，而是一个点阵，每个点代表一个画像的完整感知。

2. **分离了画像偏好和文本质量**：P3 套路感评分低（1.72/5）说明文本本身不套路；P1 套路感高（3.78）是因为甜宠少女对甜宠题材容忍度更高。单一评分无法区分"文本真的好"和"刚好对上你的口味"。

3. **探测了质量的结构**：所有画像在某个维度上一致评分 → 该维度有"硬质量"（不依赖偏好）。画像之间有分歧 → 该维度是"偏好依赖的"，对创作者来说是更细粒度的信号。

4. **稳定性有保障**：ICC=0.914 保证每个画像的输出稳定可重复（同一画像读三次结果高度一致），η²=0.219 保证画像间的差异是真实的结构性差异，而非随机噪声。

**对创作者而言**，多画像视角不是"换一把更好的尺子"，而是**从一维测量升级到高维测量**：它告诉创作者的不只是"这篇好不好"，而是**好在哪、对谁好、为什么好**。

```
读者画像 → 模拟读者引擎 → 评价分布
              ↑
             文本
```

### 使用边界

模拟读者的输出仅用于作者自检场景，标注为"模拟视角"，不参与最终评级。这与实验室愿景一致——AI 不冒充读者，不输出主观评分。

### 设计关键决策

本设计翻转了传统"先工程后验证"的顺序。核心原则：

1. **先验证核心假设，再投入工程**——"LLM 能否通过 prompt 分化地模拟不同读者"是整个系统能成立的前提，必须在任何工程实现之前验证。
2. **两阶段门控**——Phase I 验证核心假设（无层1，纯 prompt），通过后才进入 Phase II（层1工程化 + 系统集成）。任一阶段未通过则项目终止或降级为探索性研究。
3. **统计现实性**——所有成功标准匹配可用样本量，不做 n=8 回归 7 维参数这类不可信的推断。

## 模拟读者架构

### 三层架构

```
┌─────────────────────────────────────────────────┐
│  层3: 群体仿真层                                   │
│  同一文本 × N种读者画像 → 评价分布矩阵               │
│  产出："85%普通读者认为断裂，40%高素养认为留白"       │
│  Phase II 验证                                     │
├─────────────────────────────────────────────────┤
│  层2: 读者画像层                                   │
│  配置：人格维度 + 阅读经验 + 阅读状态                │
│  输出：到层1的权重向量                               │
│  Phase I 验证（使用直接prompt，无须层1）               │
├─────────────────────────────────────────────────┤
│  层1: 认知加工模拟层                                 │
│  核心：认知负荷模型 + 情境模型(Situation Model)       │
│  输入：文本 → 逐句推理需求/工作记忆/回溯重读          │
│  方法：E2产出的认知负荷指标 + CAPS建构-整合模型       │
│  Phase II 验证                                     │
└─────────────────────────────────────────────────┘
```

> 注意：层1是可选增强层，不是模拟读者的必要组件。Phase I 在不依赖层1的前提下验证核心模拟能力（直接 prompt LLM：文本 + 画像 → 评价）。层1 的认知负荷指标仅用于提高可解释性和跨文本可比性，在 Phase I 通过后才实现和验证。

### 层1 — 认知加工模拟层

基于 memo 中的认知模型（Just & Carpenter CAPS、Kintsch 建构-整合模型、认知叙事学情境模型），对文本执行认知加工模拟：

| 模块 | 输入 | 输出 | 工程化路径 |
|:---|:---|:---|:---|
| 推理需求引擎 | 文本段落 | 每百字隐含信息密度 | E2 阶段 I 产出 + 逻辑回归映射 |
| 工作记忆负载 | 句法树 + 名词链 | 工作记忆过载概率 | 名词密度 × 嵌套从句深度（E2 公式） |
| 回溯重读预测 | 指代链 | 高概率回溯位置 | 指代距离 × 歧义指数（E2 公式） |
| 情境模型构建 | 全文 RST 树 + 实体网络 | 五维度连贯性评分（时间/空间/人物/因果/意图） | E1 模块 B RST 标注 + 实体关系抽取 |

层1 的设计目标是产出独立于读者画像的客观认知负荷指标——认知负荷是文本的客观属性（该独立性通过 E4-1 验证确认）。

### 层2 — 读者画像层

读者画像基于都市言情读者的真实人口统计学和阅读行为特征构建，参考互联网产品用户画像（User Persona）方法论。

#### 画像维度

| 维度 | 类型 | 取值示例 | 对评价的影响 |
|:---|:---|:---|:---|
| 年龄/人生阶段 | 分类 | 大学生/职场新人/中坚/成熟 | 决定代入角色类型，影响对角色年龄合理性的敏感度 |
| 阅读经验 | 连续 | 新手(<1年) / 普通(1-3年) / 资深(3-5年) / 老书虫(5年+) | 越高对套路越敏感，对文笔要求越高 |
| 阅读动机 | 分类 | 情感代偿 / 娱乐消遣 / 审美鉴赏 / 社交需求 | 决定最看重的文本维度（情感冲击 vs 流畅度 vs 文学性 vs 话题性） |
| 付费行为 | 分类 | 免费党 / 章节付费 / VIP订阅 / 打赏型 | 粗略反映投入度和期待阈值 |
| 偏好子类型 | 多选 | 甜宠 / 虐恋 / 职场 / 破镜重圆 / 先婚后爱 / 暗恋成真 | 影响对特定叙事模式的熟悉度和期待 |

#### 画像定义

本实验定义 5 个具名画像 + 1 个无画像基线，覆盖都市言情读者的主要人群：

| 编号 | 标签 | 年龄/角色 | 阅读经验 | 动机 | 付费 | 偏好 | 一句话概括 |
|:---:|:---|:---|:---|:---|:---|:---|:---|
| P0 | 无画像 | — | — | — | — | — | 仅给评价指令，不给任何 persona |
| P1 | 甜宠少女 | 20岁/大学生 | 2年/普通 | 情感代偿 | 章节付费 | 甜宠 > 暗恋成真 | "看小说就是为了开心" |
| P2 | 职场丽人 | 28岁/白领一线 | 5年/资深 | 审美鉴赏+社交 | VIP订阅 | 职场 > 破镜重圆 | "文笔和逻辑不能太差，受不了傻白甜" |
| P3 | 资深老书虫 | 30岁/自由职业 | 10年/老书虫 | 审美鉴赏 | 打赏型 | 全类型但挑剔 | "套路一眼就能看出来，需要惊喜感" |
| P4 | 消遣型读者 | 24岁/基层三四线 | 1年/新手 | 娱乐消遣 | 免费党 | 不挑 | "就是打发时间" |
| P5 | 成熟婚恋读者 | 35岁/已婚二线 | 8年/资深 | 情感代偿 | VIP订阅 | 先婚后爱 > 破镜重圆 | "喜欢成年人之间的感情" |

#### 预期评价倾向

| 画像 | 情感冲击权重 | 文笔要求 | 逻辑要求 | 套路敏感度 | 结构容忍度 |
|:---|:---:|:---:|:---:|:---:|:---:|
| P1 甜宠少女 | 高 | 低 | 低 | 低 | 高 |
| P2 职场丽人 | 中 | 高 | 高 | 中 | 低 |
| P3 资深老书虫 | 低 | 极高 | 中 | 极高 | 中 |
| P4 消遣型读者 | 低 | 低 | 低 | 低 | 极高 |
| P5 成熟婚恋读者 | 高 | 中 | 中 | 中 | 中 |

#### 权重映射机制

层2 接收层1 的客观认知负荷指标，根据读者画像重新加权：

```
评价得分 = w₁(画像) × 推理需求密度 + w₂(画像) × 回溯重读概率 + w₃(画像) × 情感冲击 + ...
```

权重函数 `wᵢ(画像)` 的标定方法详见 E4-3。不依赖弹性网络回归，改用理论驱动的 pairwise 对比标定。

### 层3 — 群体仿真层

层3 不做新计算。它对同一文本批量调用多个读者画像，将结果聚合为**评价分布**。这是愿景中"描述审美"的核心：

```
输出示例：
─────────────────────────────────────────
关于"第5段的叙事跳跃"：

- 甜宠少女读者
  84% 认为"逻辑断裂" → 标注：L1 异常

- 职场丽人读者
  63% 认为"有意留白" → 标注：结构异常，置信度中

─────────────────────────────────────────
```

## 验证实验

### 总体结构

```
Phase I：核心假设验证（无层1）
├── E4-0 操控性检验（Manipulation Check）
│   └── 通过 → E4-1
├── E4-1 分化验证
│   └── 通过 → E4-2
└── E4-2 稳定性与外部锚定
    └── Phase I 全部通过 → 进入 Phase II

Phase II：系统集成与校准（含层1）
├── E4-3 层1自动化验证
├── E4-4 权重映射标定
└── E4-5 端到端集成验证
```

所有实验仅依赖已有数据（E1–E3 产出）和 LLM 自身的模拟能力，不招募新真实读者。

---

### Phase I：核心假设验证

> Phase I 的全部实验不依赖层1。只使用"文本 + 画像 prompt → LLM 直接输出评价"的最简路径。目标是投入最少的工程成本，验证 LLM 能否通过 prompt 产生分化、稳定、方向可预测的读者视角评价。

---

### E4-0 — Prompt 操控性检验

**目标**：在投入评价任务之前，验证 LLM 的**基本画像理解能力**——给定一组读者画像参数，LLM 能否生成与画像一致的自述文本，且不同画像的自述在 embedding 空间中可区分。

**原理**：如果 LLM 连"描述自己是哪种读者"都做不到可重复的画像分化，那它在评价任务中也不可能做到。这是一个快速、廉价的预筛选。

**流程**：

1. 定义 6 个画像（P0–P5），覆盖都市言情读者主流人群：

| 编号 | 标签 | 一句话概括 |
|:---:|:---|---:|
| P0 | 无画像（基线） | 仅给评价指令，不给任何 persona |
| P1 | 甜宠少女 | 大学生，看小说为了开心，偏好甜宠 |
| P2 | 职场丽人 | 白领，文笔逻辑不能太差，偏好职场文 |
| P3 | 资深老书虫 | 10年老读者，套路一眼识别，追求惊喜感 |
| P4 | 消遣型读者 | 新手，打发时间，什么都行 |
| P5 | 成熟婚恋读者 | 已婚，喜欢成年人之间的感情线 |

> P0 无画像：不给出任何 persona 描述，只给指令。P0 是效应量的绝对基线。

2. 对每个画像，使用 prompt 让 LLM 生成一段"读者自述"（无文本评价，仅描述自己的阅读偏好和风格）：

```
你是一位{画像名称}。{行为锚定描述}。

请用一段话描述你作为读者的阅读风格和偏好——你通常关注什么、不能忍受什么。
```

行为锚定描述示例（P1 甜宠少女）：
```
你是一位20岁的大学生，平时喜欢看甜宠文，最看重的就是男女主角之间的甜蜜互动。
你对虐心情节容忍度很低，看小说就是为了开心。文笔只要通顺就行，剧情不要太复杂。
你最不能忍受的是：男女主角误会来误会去、虐来虐去；悲剧结局。
```

行为锚定描述示例（P3 资深老书虫）：
```
你是一个看了十多年网络小说的老书虫，言情、耽美、玄幻都看，套路一眼就能识别出来。
你追求的是让人眼前一亮的新鲜感——不管是人物设定、叙事结构还是情感模式。
你最不能忍受的是：套路化的人设（霸道总裁+傻白甜）、注水拖剧情、为了虐而虐的情节。
```
你是一位{年龄/身份}读者。
你的阅读习惯：{阅读经验描述}。
你通常看{偏好子类型}，最看重{最看重的维度}，最不能忍{不能忍的点}。

请用一段话描述你作为读者的阅读风格和偏好——你通常关注什么、不能忍受什么。
```

3. 每个画像生成 3 次自述（temperature=0.7 → 后改为 0.3 如方差过大），共 6 × 3 = 18 次 LLM 调用。

4. 将所有 18 条自述文本 embedding 化，计算：
   - **轮廓系数**（silhouette score）：同一画像的 3 条自述是否聚在一起、不同画像之间是否分离
   - **P0 与 P1 的距离**：P1 画像是否显著偏离无画像基线

**成功标准**：

| 指标 | 标准 | 随机基线 | 说明 |
|:---|:---:|:---:|:---|
| 轮廓系数 | ≥ 0.25 | ≈ 0 | 画像间存在可区分的聚类结构 |
| 无画像漂移 | P1 与 P0 的余弦距离 ≥ 0.1 | — | 有画像 vs 无画像的差异不可忽略 |
| 最近邻一致性 | ≥ 30% | 17% (1/6) | 每条文本的最近邻来自同一画像的比例 |

**业务意义**：
- **轮廓系数**：整体信噪比检验——"6 个读者画像在 LLM 的文本产出中是否形成了可区分的团簇？"0.25 意味着类内距离明显小于类间距离，画像体系整体有效。
- **无画像漂移**：注入有效性检验——"给不给 persona 是否改变了 LLM 的输出？"P0（无画像）与 P1（甜宠少女）的差异必须可检测，否则画像 prompt 被 LLM 忽略了。
- **最近邻一致性**：粒度实用性检验——"同一画像的 3 次重复调用，是不是更像彼此而不是别的人？"随机水平 17%（1/6），30% 是 1.8 倍于随机，意味着画像差异在实际中可通过单次文本检测到。三个标准分别对应"整体有效"、"注入有效"、"差异可用"。

**最近邻一致性计算**：对 18 条自述文本，每条取其 cosine 最近邻（排除自身）。最近邻来自同一画像算命中。

**门控规则**：3 项全部通过 → 进入 E4-1。任一不通过 → 尝试行为锚定 prompt 重跑 E4-0（温度降至 0.3 提高稳定性）。仍不通过 → 项目终止。

### E4-1 — 读者画像分化验证

**目标**：在评价任务上，验证 LLM 在不同画像 prompt 下产生的评价存在**系统差异**，且差异方向与心理学文献一致。

**设计原理**：

不招募真实读者，而是用结构化 prompt 让 LLM 扮演特定读者画像。验证逻辑是：

```
如果 LLM 在 prompt 引导下能稳定产出与画像一致的差异化评价，
那么就可以用这些模拟输出来校准层2 的权重函数。
```

E4-1 只验证两件事：
1. **分化**：不同画像的评价是否系统不同
2. **方向**：差异方向是否与文献一致

这不是"LLM 是否等于真实读者"的问题。

**画像定义**：与 E4-0 一致（P1–P5 五个具名画像），P0 不做评价。

**Prompt 策略**：

第一阶段尝试数值参数 prompt：

```
你是一位{画像名称}。{行为锚定描述}。

请从你的视角评价以下段落：
```

行为锚定 prompt 使用结构化的画像描述（而非抽象数值参数），直接告诉 LLM 该读者是谁、看重什么、不能忍什么。E4-0 的实证表明行为锚定在轮廓系数和最近邻一致性上均显著优于数值参数 prompt。

**材料**：从 E1 的 14 篇中选取 6 篇（4.1、7.2、9.1、2.3、10.3、1.2），覆盖高密度舒服型、高密度冲击型、低密度低冲突三组对照。增加 1 篇（1.2）以提供更多对照度。

**执行模拟**：

1. 5 个画像（P1–P5）× 6 篇文本 × n_calls 次调用（n_calls 由 pilot 确定，默认 3，temperature=0.3）
2. 每个调用输出以下评价项：
   - writing_quality 1–7：文笔质量
   - emotional_impact 1–7：情感冲击力
   - character_realism 1–7：角色真实感
   - cliche_level 1–5：套路感
   - logic_break：逻辑断裂检测
   - aesthetic_grade：整体审美判断 A/A-/B/C

**分析方案**：

**验证1 — 判别效度（无方向检验）**：不同画像是否产生**不同的**评价模式？

不预设任何画像之间评分高低的硬性方向。改用 η²（eta-squared）测量画像因素在每项评分上的效应量：

| 维度 | 方法 | 标准 |
|:---|:---|---:|
| 各评分维度 | 单因素 ANOVA（画像为自变量） | η² ≥ 0.10（中等效应） |
| 汇总 | 各维度 η² 的均值 | 平均 η² ≥ 0.10 |

η² = SS_between / SS_total，衡量"画像不同"这个因素解释了多少评分方差。

**验证2 — 稳定效度**：同一画像 × 文本组合的多次调用之间的 ICC ≥ 0.50。

> 原设计的 ICC ≥ 0.70 在 temperature=0.7 下过于严苛。ICC ≥ 0.50 已经是中等以上一致性，足以支撑群体层面的推断。Phase II 中可通过降低 temperature 或 majority voting 提升。

**验证4 — 客观基线**：使用**人为植入的错误**（E4-0 中每篇文本植入 2 个语法错误 + 1 个事实矛盾），各画像对植入错误的检出率 ≥ 75%。

> 与原设计的差异：原文使用自然文本中的"结构异常"作为客观基线，但高开放性读者可能将其解读为"留白"——这恰恰是画像差异的一部分，不是真正的客观项。改用植入错误确保基线是真正的客观项。

**成功标准**：

| 验证 | 指标 | 标准 |
|:---|:---|---:|
| 判别效度 | 各维度 η² 的均值 | 平均 η² ≥ 0.10 |
| 稳定效度 | 同一画像 × 文本的 ICC | ≥ 0.50 |

**依赖**：前置 E4-0（通过）

**门控**：E4-1 全部 4 项通过 → 进入 E4-2。否则项目终止或转为探索性研究。

### E4-2 — 跨画像泛化与外部锚定

**目标**：验证在 E4-1 的基础上：
1. 分化效应在更多文本上保持稳定（跨文本泛化）
2. E4-1 的模拟读者评分与 E3（美学评审）的既有输出具有可解释的关联（外部锚定）

**设计**：

**泛化检验**：将 E4-1 的 6 篇文本分为 3 组（各 2 篇），用其中 2 组的模拟数据训练一个简单分类器（区分画像），在剩余 1 组上预测画像标签。3-fold CV，预测准确率 ≥ 40%（随机基线 = 20%）为通过。

> 这不是一个可部署的分类器训练，而是一个统计检验：如果不同画像的评价模式在未见文本上也能区分，说明效应不是文本特定的。

**外部锚定**：以 E3（美学评审实验）的输出为外部参考，检验 P1（普通读者）的模拟评价与 E3 的评审意见在以下两维度上的一致性：
- 整体审美判断的一致性（ρ ≥ 0.40 即可——预期 P1 不会完全等于真实评审者，但不应完全无关）
- 情感冲击评分的秩相关（τ ≥ 0.35）

**成功标准**：

| 验证 | 指标 | 标准 |
|:---|:---|---:|
| 跨文本泛化 | 3-fold CV 画像分类准确率 | ≥ 40%（基线 20%） |
| 外部锚定（审美判断） | P1 评分与 E3 的 Spearman ρ | ≥ 0.40 |
| 外部锚定（情感冲击） | P1 评分与 E3 的 Kendall τ | ≥ 0.35 |

**依赖**：前置 E4-1（通过）

**门控**：E4-2 全部通过 → **Phase I 通过**，进入 Phase II。任一未通过 → 项目降级为"纯 prompt 模拟读者"（无层1工程化），仅保留 Phase I 成果作为实验工具使用，不进入生产级实现。

### E4-3 — 层1自动化验证

**目标**：验证层1 自动化后的认知负荷预测与 E2 手动标注数据对齐。

**验证设计**：

| 验证项 | 金标准来源 | 样本量 | 成功标准 |
|:---|:---|:---:|:---:|
| 推理需求密度 | E2 阶段 I 中评审者逐句标注的推理需求得分 | 4 篇对照文本的全部句子 | Spearman ρ ≥ 0.70 |
| 工作记忆过载概率 | E2 阶段 I 中评审者标注的"难读句子"二值标签 | 同上 | AUC ≥ 0.75 |
| 回溯重读预测 | E2 阶段 I 中指代距离的原始标注数据 | 同上 | 命中率 ≥ 60% |
| 情境模型五维度 | E1 模块 B 的 RST 关系标注重构出的连贯性基线 | 14 篇全部 | 维度 ICC ≥ 0.65 |

**关键原则**：这里没有任何新标注。E2 阶段 I 已经由 3 位评审者逐句标注了推理需求、工作记忆负载和指代距离。E4-3 验证的是：自动化实现的层1 能否复现这些手工标注的结果。

**依赖**：前置 E1、E2、cognitive-load-formulas.md（如不存在需先行补档）

**成功标准**：全部 4 项同时通过。有一项不通过说明该模块的自动化不可靠，需要修复工程实现或调整公式。

### E4-4 — 权重映射标定

**目标**：从 Phase I 的模拟评价数据中，为每个读者画像标定权重向量，建立"画像参数 → 权重"的映射。

**设计与原设计的差异**：

原设计试图从 8 个画像的 7 维参数用弹性网络回归预测权重向量（8 样本拟合 7 维空间 + 多个权重系数，严重欠定）。本设计改用两步法：

**第一步：pairwise 权重比推断**

对每个画像 Px，使用 E4-1 收集的评价数据：
- 以层1 认知负荷指标为自变量、模拟评价得分为因变量，做**标准化线性回归**（仅 1 个因变量、3 个认知负荷指标，在全部 6 篇文本 × 3 次调用的数据上进行，n=18）
- 不直接用回归系数作为权重，而是计算**权重比**：`w₁ : w₂ : w₃`（各认知负荷指标的相对重要性）

**第二步：参数 → 权重比映射**

- 以 5 个画像（P1–P5）为观测点，用简单线性模型拟合每个权重比随单一画像参数变化的单调趋势
- 模型形式：每个权重比由 2-3 个画像参数线性预测，使用正则化的线性回归（n=5，p≤3，可行）

**验证**：

| 验证 | 方法 | 成功标准 | 实际结果 |
|:---|:---|---:|:---:|
| 权重比稳定性 | 对每个画像，用 bootstrapping（200 次）估计权重比的 95% CI | CI 宽度 < 0.30 | 全部 0.14-0.18 ✅ |
| 权重有序性 | 检查权重比是否随画像参数单调变化 | ≥ 3/4 的预期单调关系成立 | 3/4 ✅ |
| 跨文本一致性 | 替代方案：标准化残差分析（n=6 文本 × 3 特征的 DOF 不足以计算 ICC） | 残差均值 < 0.8 | 0.345 ✅ |

> **重要后验发现**：回归系数的符号分布在所有画像中完全一致（inference_demand 负，working_memory 正，backtracking 正）。原因是 n=6 文本的样本量不足以支持 3-feature 回归——回归系数反映的是数据中的通用模式而非画像间差异。**层1→层2 的回归权重映射路径在 n=6 文本样本下不可行。** 层1 认知负荷降级为"可选装饰层"——它可以帮助解释"为什么 P3 给了低分"（如"cliche_level 低是因为 backtracking 概率高"），但不参与分数预测。

**依赖**：前置 E4-3（层1自动化完成）

**成功标准**：全部 3 项通过。

### E4-5 — 跨文本分化稳定性验证

**目标**：在 4 篇全新的文本上运行模拟读者引擎（直接 prompt 模式），验证 Phase I 的结论是否适用于未见文本。

> **为什么没有层1→层2？** 实验发现，回归权重映射需要比 6 篇更多的样本来产生有意义的画像间权重差异。层1 认知负荷降级为可选装饰层——它帮助解释"为什么 P3 给了低分"，但不参与预测评分。

**给创作者的解释**：E4-5 回答的是"你在写下一篇新小说的时候，这个模拟读者引擎还能用吗？"答案是：**能，而且表现和之前一样好。**

**验证方法**：

1. 从小说训练集中选 4 篇未在 Phase I 中使用过的文本（2.1 展会再遇、4.2 夜市约会、6.2 海边散步、8.2 酒吧表白）
2. 5 个读者画像各自对这 4 篇文本评分（每篇 3 次稳定调用）
3. 验证三项：

**验证结果**：

| 验证项 | 验证什么 | 结果 | 标准 |
|:---|:---|---:|:---:|
| **稳定性** | 同一个读者读同一篇新文本，3 次评分是否一致？ | **ICC=0.62 ✅** | ≥ 0.50 |
| **分化性** | 不同读者在新文本上的评分是否仍然显著不同？ | **3/4 维度显著 ✅**（文笔 p=0.022、角色真实感 p<0.001、套路感 p<0.001）平均 η²=0.26 | ≥ 2/4 维度或 η²≥0.10 |
| **模式一致性** | 读者在新文本上的表现模式是否与之前一致？ | **P3 仍然最挑剔 ✅**（套路感 2.33 全最低）**P1 仍然最感性 ✅**（情感冲击 5.42 全最高） | — |

**这意味着什么？**

- ✅ 你的新作品用这个引擎评估，结果和训练数据一样可信
- ✅ 资深老书虫（P3）在任何作品上都是最挑剔的那个——如果 P3 给你高分，你写得好
- ✅ 甜宠少女（P1）永远最感性——她的评分高不说明作品一定好，但她的评分低说明作品情感上不行
- ⚠️ 但注意：**这套引擎告诉你的是"不同读者会怎么看"，不是"你的作品客观打几分"。** 后者的"客观分"不存在——E4-5 的结论本身就是最好的证明：同一部作品在不同读者眼里是完全不同的东西。

**依赖**：前置 E4-3、E4-4

**成功标准**：全部 3 项通过。

## 与原设计的关键差异对照

| 维度 | 原设计 | 本设计 | 理由 |
|:---|:---|:---|:---|
| 实验顺序 | E4-A → E4-B → E4-C（串行，层1前置） | E4-0/1/2（Phase I）→ E4-3/4/5（Phase II） | 核心假设先验，层1工程后置 |
| 核心假设验证 | 依赖层1完成 | 纯 prompt，无工程依赖 | 快速验证 LLM 能力，避免浪费 |
| 基线画像 | 抽象心理学维度（openness/empathy/NFC） | 真实读者画像（年龄/职业/阅读经验/动机） | 贴近产品实际，LLM 更易理解 |
| 对比数量 | 6 组（P1–P6） | 4 组（P1–P5，含效应量门槛） | 降低多重比较负担 |
| 稳定效度标准 | ICC ≥ 0.70 | ICC ≥ 0.50 | temperature=0.7 下的现实标准 |
| 客观基线 | 自然文本的结构异常 | 人为植入的语法错误和事实矛盾 | 自然异常正是画像差异的一部分 |
| 文本数量 | 5 篇 | 6 篇（Phase I）+ 8 篇（Phase II）| 增加统计效力和泛化检验 |
| 权重映射 | 弹性网络回归（8样本×7维） | 权重比 + pairwise 标定（n=30） | n=8不可信 |
| 跨文本泛化 | ρ ≥ 0.75（5-fold CV） | 画像分类准确率 ≥ 40%（3-fold CV） | 标准与样本量匹配 |
| 外部锚定 | 无（自洽验证） | E3 锚定 + 植入错误基线 | 引入外部参照系 |
| 门控机制 | 无（全部实验跑完后评估） | 每阶段有 go/no-go 条件 | 快速失败，避免无效投入 |
