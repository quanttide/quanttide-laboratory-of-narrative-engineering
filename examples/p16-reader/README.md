# E4 — 模拟读者系统：架构与验证

> 注：本文档中 E4 / E4-A / E4-B / E4-C 对应 PoC 编号 p16-reader。

## 概述

本蓝图定义"模拟读者"是什么、如何构建、如何验证。

**一句话定义**：模拟读者是一个可配置模型，给定文本 + 读者画像 → 输出该读者视角下的评价分布（含置信区间）。

```
读者画像 → 模拟读者引擎 → 评价分布
              ↑
             文本
```

### 使用边界

模拟读者的输出仅用于作者自检场景，标注为"模拟视角"，不参与最终评级。这与实验室愿景一致——AI 不冒充读者，不输出主观评分。

## 模拟读者架构

### 三层架构

```
┌─────────────────────────────────────────────────┐
│  层3: 群体仿真层                                   │
│  同一文本 × N种读者画像 → 评价分布矩阵               │
│  产出："85%普通读者认为断裂，40%高素养认为留白"       │
│  验证：画像间差异与文献已知方向一致                    │
├─────────────────────────────────────────────────┤
│  层2: 读者画像层                                   │
│  配置：人格维度 + 阅读经验 + 阅读状态                │
│  输出：到层1的权重向量                               │
│  验证：不同画像对同一文本产出系统不同评价               │
├─────────────────────────────────────────────────┤
│  层1: 认知加工模拟层                                 │
│  核心：认知负荷模型 + 情境模型(Situation Model)       │
│  输入：文本 → 逐句推理需求/工作记忆/回溯重读          │
│  方法：E2产出的认知负荷指标 + CAPS建构-整合模型       │
│  验证：与E2已有标注数据的对齐度                       │
└─────────────────────────────────────────────────┘
```

### 层1 — 认知加工模拟层

基于 memo 中的认知模型（Just & Carpenter CAPS、Kintsch 建构-整合模型、认知叙事学情境模型），对文本执行认知加工模拟：

| 模块 | 输入 | 输出 | 工程化路径 |
|:---|:---|:---|:---|
| 推理需求引擎 | 文本段落 | 每百字隐含信息密度 | E2 阶段 I 产出 + 逻辑回归映射 |
| 工作记忆负载 | 句法树 + 名词链 | 工作记忆过载概率 | 名词密度 × 嵌套从句深度（E2 公式） |
| 回溯重读预测 | 指代链 | 高概率回溯位置 | 指代距离 × 歧义指数（E2 公式） |
| 情境模型构建 | 全文 RST 树 + 实体网络 | 五维度连贯性评分（时间/空间/人物/因果/意图） | E1 模块 B RST 标注 + 实体关系抽取 |

层1 的设计目标是产出独立于读者画像的客观认知负荷指标——认知负荷是文本的客观属性（该独立性需通过 E4-A 验证确认）。

### 层2 — 读者画像层

读者画像由三个维度的参数定义：

#### 维度 A：人格特质

| 参数 | 取值范围 | 对应读者类型 | 影响效应（文献预期方向） |
|:---|:---:|:---|:---|
| `openness` | 0–1 | 高开放性 → 文学爱好者 | 对结构异常的容忍度↑、对隐喻的解读深度↑ |
| `empathy` | 0–1 | 高共情 → 情感沉浸型 | 情感冲击力的权重↑、角色认同深度↑ |
| `need_for_closure` | 0–1 | 高闭合需求 → 类型读者 | 对叙事完整性的敏感度↑、对模糊的排斥↑ |

#### 维度 B：阅读经验

| 参数 | 取值范围 | 说明 |
|:---|:---:|:---|
| `literary_expertise` | 0–1 | 文学素养（识别修辞、结构创新的能力） |
| `genre_familiarity` | 0–1 | 对该类型的阅读量（类型 schema 的丰富度） |

#### 维度 C：状态

| 参数 | 取值范围 | 说明 |
|:---|:---:|:---|
| `time_pressure` | 0–1 | 时间压力（高 = 对认知负荷更敏感） |
| `reading_purpose` | {学习, 消遣, 评审} | 阅读目的（影响注意力分配） |

#### 权重映射机制

层2 接收层1 的客观认知负荷指标，根据读者画像重新加权：

```
评价得分 = w₁(画像) × 推理需求密度 + w₂(画像) × 回溯重读概率 + w₃(画像) × 情感冲击 + ...
```

权重函数 `wᵢ(画像)` 的实现路径（详见 E4-B）不依赖真实读者数据——而是通过 prompt 引导 LLM 模拟特定读者视角，收集模拟评价，再回归出权重。

### 层3 — 群体仿真层

层3 不做新计算。它对同一文本批量调用多个读者画像，将结果聚合为**评价分布**。这是愿景中"描述审美"的核心：

```
输出示例：
─────────────────────────────────────────
关于"第5段的叙事跳跃"：

- 普通读者（openness=0.4, expertise=0.3）
  84% 认为"逻辑断裂" → 标注：L1 异常

- 高文学素养读者（openness=0.8, expertise=0.9）
  63% 认为"有意留白" → 标注：结构异常，置信度中

─────────────────────────────────────────
```

## 验证实验

模拟读者系统需要逐层验证。以下实验全部依赖已有数据（E1–E3 产出）和 LLM 自身的模拟能力，不招募新真实读者。

### E4-A — 认知加工模拟层验证

**目标**：验证层1 自动化后的认知负荷预测与 E2 手动标注数据对齐。

**验证设计**：

| 验证项 | 金标准来源 | 样本量 | 成功标准 |
|:---|:---|:---:|:---:|
| 推理需求密度 | E2 阶段 I 中评审者逐句标注的推理需求得分 | 4 篇对照文本的全部句子 | Spearman ρ ≥ 0.70 |
| 工作记忆过载概率 | E2 阶段 I 中评审者标注的"难读句子"二值标签 | 同上 | AUC ≥ 0.75 |
| 回溯重读预测 | E2 阶段 I 中指代距离的原始标注数据 | 同上 | 命中率 ≥ 60% |
| 情境模型五维度 | E1 模块 B 的 RST 关系标注重构出的连贯性基线 | 14 篇全部 | 维度一致率（加权 κ 或 ICC）≥ 0.65 |

**关键原则**：这里没有任何新标注。E2 阶段 I 已经由 3 位评审者逐句标注了推理需求、工作记忆负载和指代距离。E4-A 验证的是：自动化实现的层1 能否复现这些手工标注的结果。

**依赖**：前置 E1、E2

**E2 认知负荷公式**：定义见 `docs/memo/cognitive-load-formulas.md`（名词密度 × 嵌套从句深度、指代距离 × 歧义指数）。如该文档不存在，需在实施 E4-A 前补档。

**LLM 模型**：默认使用与 p09/p10 一致的模型。如切换模型，E4-B 需重新验证分化效度。

### E4-B — 读者画像分化验证

**目标**：验证通过 LLM prompt 引导的不同读者画像，对同一文本产生**系统不同的**评价，且差异方向与心理学文献一致。

这是整个蓝图的核心实验——它用 LLM 自身的模拟能力替代真实读者招募。

#### 设计原理

不招募真实读者，而是用结构化 prompt 让 LLM 扮演特定读者画像。验证逻辑是：

```
如果 LLM 在 prompt 引导下能稳定产出与画像一致的差异化评价，
那么这些模拟输出就可以用来校准层2 的权重函数。
```

这不是"LLM 是否等于真实读者"的问题——E4-B 只验证：
1. **分化**：不同画像的输出是否系统不同（判别效度）
2. **方向**：差异是否与心理学文献中的已知效应方向一致（定向效度）
3. **稳定**：同一画像在多次调用中是否产出相近结果（重测信度）

#### 流程

1. **定义画像 prompt 模板**（一套结构化系统指令）：

   每条 prompt 的骨架：
   ```
   你是一位{openness: 值}、{empathy: 值}、{need_for_closure: 值}的读者，
   你的文学素养为{literary_expertise: 值}，对该类型熟悉度为{genre_familiarity: 值}，
   当前阅读状态为{time_pressure: 值}、目的是{reading_purpose: 值}。

   请从你的视角评价以下段落：
   ...
   ```

   > 如果仅靠数值 prompt 无法驱动稳定分化（即验证1 方向成立比例 < 5/6），改为行为锚定 prompt：例如"你是一位习惯于欣赏叙事实验的读者，对时间跳跃和非线性结构持开放态度……"。p09 的经验表明，行为锚定描述比纯数值更能稳定驱动 LLM 的视角切换。

2. **画像格点采样**：选取 8 个画像格点，覆盖画像空间的关键区域：

   | 编号 | 标签 | openness | empathy | NFC | expertise | familiarity |
   |:---:|:---|---:|:---:|:---:|:---:|:---:|
   | P1 | 普通读者（基线） | 0.4 | 0.5 | 0.5 | 0.3 | 0.3 |
   | P2 | 高文学素养 | 0.8 | 0.5 | 0.3 | 0.9 | 0.5 |
   | P3 | 情感沉浸型 | 0.5 | 0.9 | 0.3 | 0.3 | 0.5 |
   | P4 | 类型读者（悬疑） | 0.4 | 0.4 | 0.8 | 0.3 | 0.8 |
   | P5 | 低开放性保守型 | 0.2 | 0.4 | 0.7 | 0.3 | 0.3 |
   | P6 | 高开放+高经验 | 0.9 | 0.5 | 0.2 | 0.9 | 0.7 |
   | P7 | 时间压力型 | 0.4 | 0.5 | 0.5 | 0.3 | 0.3 |
   | P8 | 消遣型读者 | 0.5 | 0.6 | 0.3 | 0.2 | 0.4 |

   > P7 的 openness/empathy/NFC 与 P1 相同，仅 time_pressure 设为 0.8，reading_purpose 设为"消遣"（P1 默认均为 0.3/"评审"），用于独立验证状态维度的影响。

3. **执行模拟**：每个画像在每个文本上执行 3 次独立调用（temperature=0.7 <sup>①</sup>），评估任务包括：

   > ① temperature=0.7 的方差受底层模型影响。如 ICC 不达标，可尝试 temperature=0.3 或 majority voting 降噪后重测。
   - L1 子集：逻辑断裂检测、语法/拼写错误（应保持跨画像一致——所有读者都能识别客观错误）
   - L2：情感倾向标签、表层意象识别、阅读难度（应出现适度画像间差异）
   - L3：情感冲击力 1–7 点量表、结构异常容忍度 1–5 点量表、整体审美判断 A/A-/B

4. **材料**：从 E1 的 14 篇中选取 5 篇（4.1、7.2、9.1、2.3、10.3），覆盖高密度舒服型、高密度冲击型、低密度低冲突三组对照。

#### 分析方案

**验证1 — 判别效度**：画像 P1–P6 之间能否产生系统差异？

| 预期方向 | 对比 | 预期效应 | 文献依据 |
|:---|:---|---:|:---|
| P2 比 P1 更易识别结构异常为"有意留白" | P2 vs P1 在结构异常标注上的比例差 | ≥ 30% | 高开放性=对新奇结构的接纳度更高 |
| P3 的情感冲击评分高于 P1 | P3 vs P1 情感冲击均值的 Cohen's d | ≥ 0.6 | 高共情=更强情感反应 |
| P4 比 P1 对逻辑断裂更敏感 | P4 vs P1 在逻辑断裂检测上的召回率差 | ≥ 20% | 高闭合需求=对叙事连贯性要求更高 |
| P5 比 P1 更倾向于将偏离标注为"错误" | P5 vs P1 的异常分类中"逻辑异常"占比差 | ≥ 15% | 低开放性=对偏离的容忍度更低 |
| P6 与 P2 方向相似但效应更强 | P6 vs P2 在"有意留白"识别率上的差 | ≥ 10% | 高开放+高经验=最接近专家读者 |
| P7 与 P1 在客观项上一致，在主观项上更宽松 | P7 vs P1 在阅读难度评分上的差 | ≥ 0.5 | 时间压力→更追求流畅，对难度更敏感 |

**验证2 — 定向效度**：所有观察到的差异方向是否与文献一致？

逐一检查验证1 的各项，如果所有方向符合率 ≥ 5/6，定向效度通过。

**验证3 — 稳定效度**：同一画像的 3 次调用之间的 ICC ≥ 0.70 为通过。

**验证4 — 客观项基线**：各画像在 L1 客观项（逻辑断裂检测、语法/拼写错误）上的 κ ≥ 0.70——验证模拟读者在客观错误面前"说人话"。

> P7（时间压力=0.8）因状态限制可能漏过客观错误，其 κ 单独报告不作硬性要求。

#### 成功标准

| 验证 | 指标 | 标准 |
|:---|:---|---:|
| 判别效度 | P1–P6 六组对比中预期方向成立的比例 | ≥ 5/6 |
| 定向效度 | 所有成立对比中方向与文献一致的占比 | 100% |
| 稳定效度 | 同一画像 3 次调用的 ICC | ≥ 0.70 |
| 客观基线 | 除 P7 外各画像在 L1 客观项上的 κ；P7 单独报告 | ≥ 0.70（P7 不作硬性要求） |

**依赖**：前置 E4-A

### E4-C — 权重映射与跨文本泛化

**目标**：从 E4-B 的模拟数据中提取每个画像的权重向量，建立"画像参数→权重"的映射函数，并在未见文本上验证泛化能力。

#### 流程

1. **权重提取**：对 E4-B 中每个画像（P1–P8），以层1 认知负荷指标为自变量、模拟评价得分为因变量做线性回归，得到该画像的权重向量 `wᵢ`

2. **映射函数训练**：以 8 个画像的 7 维参数为自变量、对应的权重向量为因变量，训练映射函数（弹性网络回归，因样本量小不使用 MLP）

3. **跨文本交叉验证**：将 E4-B 使用的 5 篇文本分为训练集（4 篇）和测试集（1 篇），做 5-fold 交叉验证。每次用训练集文本的模拟数据提取权重→训练映射→预测测试集文本的评价→计算预测误差

4. **跨画像留出验证**：留出一个画像（如 P6），用其余 7 个画像训练映射函数，预测 P6 在全部 5 篇文本上的评价

#### 分析方案

| 验证 | 方法 | 成功标准 |
|:---|:---|---:|
| 跨文本泛化 | 5-fold CV，逐篇计算预测值与模拟值的 Spearman ρ | ρ ≥ 0.75 |
| 跨画像泛化 | 留一画像法，预测被留出画像的评价 | ρ ≥ 0.65 |
| 权重稳定性 | 8 个画像的权重向量间欧氏距离矩阵 | 相近画像（如 P2 vs P6）的距离 < 相异画像（如 P2 vs P5） |


#### 成功标准

| 验证 | 指标 | 标准 |
|:---|:---|---:|
| 跨文本泛化 | 5-fold CV 平均 ρ | ≥ 0.75 |
| 跨画像泛化 | 留一画像法平均 ρ | ≥ 0.65 |
| E3 合理性对齐 | P1 效度与 E3 效度之差的绝对值 | ≤ 0.15 |
