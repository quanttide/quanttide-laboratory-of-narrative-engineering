# E4-7 — 读者视角的局部修改建议（Rewrite）

> 对应能力：Rewrite（修复层）

## 动机

p16-reader 能告诉作者"P3 觉得这个场景套路感强"，E4-6（Reflect）能告诉作者"因为角色行为缺乏内隐动机"。但作者还需要**具体怎么改**——不是"加强情感表达"，而是"第三段的对话加上林远亭的心理描写"。

E4-7 的核心假设是：**给定 Reflect 诊断 + 目标画像偏好，LLM 能生成可采纳的结构性修改。**

## 核心洞察

从 p10（母题-风格交叉）和 p14（片段内部情节建议）的经验看，有效的修改建议需要满足：

1. **可执行**：作者拿到后知道在文本里改哪个位置、改什么
2. **有理由**：建议附带"为什么要这么改"（基于读者画像偏好）
3. **最小改动**：只改诊断指出的薄弱点，不重写全篇

## 实验设计

### 输入

| 来源 | 内容 | 格式 |
|------|------|------|
| E4-6 诊断 | reflect_*.json — root_cause + evidence + fix_direction | JSON |
| 原文 | assets/fiction/4_成稿/ — 诊断涉及的文本全文 | Markdown |
| 画像定义 | profiles.json — 目标画像的详细行为锚定 | JSON |

### 步骤

**Step 1: 构造 Rewrite Prompt**

输入：
1. 文本片段（诊断指出的薄弱段落）
2. Reflect 诊断（root_cause + evidence + structural_type）
3. 目标画像定义（"这是针对 P3 资深老书虫的修改，他要求每句话都有心理深度"）

输出结构：
```json
{
  "target_profile": "P3",
  "target_dimension": "character_realism",
  "original_segment": "原文段落...",
  "suggested_rewrite": "改写后的段落...",
  "what_changed": ["新增了林远亭的心理描写", "删去了第三句的套路化对白"],
  "why_for_reader": "P3 无法接受缺乏动机的台词——改写后每句话都有了内隐目标",
  "minimal_changes": true
}
```

**Step 2: A/B 盲测验证**

对每篇文本的每个诊断点，生成 2 个版本的修改建议（2 个不同 prompt 风格），与原文件 3 组对比：

- A = 原文本
- B1 = 修改版本 1
- B2 = 修改版本 2

3 个 LLM 评委（不加温度）对 A/B1、A/B2 做盲测：
- "相比原版，这个版本是否更符合目标画像的偏好？"（是/否）
- "改动是否仍然保留了原文的核心场景结构？"（是/否）

### 输出

```
data/output/e4-7_rewrite_{tid}.json
data/output/e4-7_rewrite_ab_test.json
data/output/e4-7_rewrite_summary.json
```

### 验证

**验证1 — 评审偏好提升**：修改版本是否更符合目标画像的偏好？

- 方法：3 个 LLM 评委盲测 A/B
- 指标：修改版本被评为"更符合目标画像偏好"的比例
- 标准：≥ 2/3 的修改版本优于原版

**验证2 — 场景完整性保持**：修改是否破坏了原文的场景结构？

- 方法：3 个 LLM 评委判断"仍然保留核心场景结构"的比例
- 指标：保持率（平均 / 每篇文本）
- 标准：保持率 ≥ 80%

**验证3 — 修改特异性**：针对不同画像的修改是否产生不同的文本？

- 方法：对比相同文本针对 P1 vs P3 的修改版本编辑距离
- 指标：编辑距离差异（针对不同画像的改法是否系统不同）
- 标准：针对不同画像的修改平均编辑距离 ≥ 20%（即改法不同）

### 成功标准

| 验证 | 标准 | 方法 |
|:---|:---:|:---|
| 偏好提升 | ≥ 2/3 优于原版 | 3 评委盲测 A/B |
| 场景完整性 | ≥ 80% 保持率 | 评委判断 |
| 修改特异性 | 跨画像编辑距离 ≥ 20% | 编辑距离分析 |

### 依赖

- 前置：E4-6 完成（依赖 Reflect 诊断）
- LLM 调用量：6 篇文本 × 2 prompt 风格 × 1 画像 = ~12 次生成 + 评委调用

### 估计耗时

编码 1 天 + 运行 ~1 小时 + 分析 1 天
