# E4-6 — 多画像评价的视角译码（Reflect）

> 对应能力：Reflect（视角译码层）

## 重新定位：E4-6 不诊断文本，而是翻译视角

p14 已经做了文本诊断（行为链标注 + 薄弱点定位 + 修改建议），且粒度更细、结构更清晰。

E4-6 的独特价值不是做另一套文本诊断，而是回答 p16-reader 独有的问题：

> **同一个文本片段，在 P1 眼中是什么样，在 P3 眼中是什么样？**

当 P3 给 character_realism 打 5.94、P1 打 4.17，这不是文本的"真实"问题——这就是**两部作品**：P3 读到的是"林远亭擦头发的动作有社交越界的张力"，P1 读到的是"男主给女主擦头发，好甜"。

E4-6 的核心假设是：**评分差异背后是两个读者读了不同的文本（不是同一个文本的两个评价）。**

## 核心假设

多画像评价产生的差异，本质上是 LLM 模拟不同读者时，同一个文本触发了**不同的注意分配**和**不同的情感归因**。

- P1 读到咖啡厅重逢，注意的是"他记得她爱喝焦糖玛奇朵"→ 甜的
- P3 读到同一段，注意的是"作为社牛在公共场合掏毛巾擦别人头发是社交越界"→ 有意思的写作
- 两者都对，但注意点完全不同

E4-6 的 Reflect 就是把这个"不同"翻译出来——不是线性地告诉作者"哪里不对"，而是**同时展示 5 个读者眼中各自看到的文本是什么**。

## 输出：不是"诊断"，是"解读"

E4-6 不为文本做诊断（那是 p14 的领域），而是为**每对画像 × 维度**做解读：

```json
{
  "text_id": "1.1",
  "highlight_dimension": "character_realism",
  "pairs": [
    {
      "profile_low": "P1",
      "profile_high": "P3",
      "low_score": 4.17,
      "high_score": 5.94,
      "diff": 1.77,
      "divergence": "P1 关注的是男主动作的'甜'属性——擦头发、点咖啡都是甜宠标配，够了。P3 关注的是动作的'社交边界'属性——在公共场合擦陌生人的头发是一种越界，这个细节让角色变得复杂",
      "key_moment": "林远亭掏出毛巾擦她头发的段落"
    }
  ]
}
```

## 设计

### 输入

| 来源 | 内容 | 格式 |
|------|------|------|
| `e4-1_raw.json` | 5 画像 × 6 文本 × 3 call 的完整评分 | JSON |
| `assets/fiction/4_成稿/` | 6 篇测试文本的全文 | Markdown |

### Step 1: 差异识别（纯数据处理）

对每篇文本：

1. 计算所有画像 × 维度的评分均值
2. 找到画像间差异最大的 3 个覆盖对（覆盖最大 |diff| 的画像对 × 维度）
3. 对每个显著差异对，提取原文中该维度最相关的段落

不需要 LLM。

### Step 2: 视角译码（LLM）

对每个差异对，输入：

1. 文本全文
2. 低分画像的行为锚定 + 评分
3. 高分画像的行为锚定 + 评分
4. 差异量（diff + 维度名）

输出：

```json
{
  "divergence": "解释低分画像和高分画像分别看到了什么不同的东西",
  "key_moment": "差异最集中体现的文本片段位置",
  "low_reader_view": "低分画像眼中这一段是什么样子的——他们忽略了什么、注意到了什么",
  "high_reader_view": "高分画像眼中这一段是什么样子的——他们看到了什么低分画像没看到的东西"
}
```

### 验证

#### 验证 1 — 一致性（3 种 prompt 风格）

同一差异对，3 种不同 prompt 风格的译码，在"key_moment"上是否指向同一段落？

- 标准：≥ 2/3 的差异对在 key_moment 上一致（交集段落长度 ≥ 原文半句）

比 embedding 好：key_moment 是具体的文本位置，可客观判定。

#### 验证 2 — 画像对称性交换

对同一个差异对，交换低分/高分画像的角色，看译码是否随之反转。

例如：P1=4.17, P3=5.94 的差异对，译码为"P1 没看到社交越界，P3 看到了"。如果交换为"P3=4.17, P1=5.94"（伪造的假数据），译码应该变为"P3 没看到甜宠细节，P1 看到了"。

- 方法：对 3 个差异对，构造假数据（交换画像和评分），比较真实译码和假译码的 divergence 内容
- 标准：**真实译码和假译码在内容上不同**（3 评委投票，≥ 2/3 判为"不同解释"）

#### 验证 3 — 可读性

译码是否对作者有帮助？

- 方法：3 评委对每个译码回答"这段译码是否可以帮助作者理解为什么不同读者打出了不同的分？"
- 标准：**≥ 80% 的译码判为"有帮助"**

### 成功标准

| 验证 | 标准 | 方法 |
|:---|:---:|:---|
| 一致性 | ≥ 2/3 差异对的 key_moment 一致 | 3 prompt 风格交集判断 |
| 画像对称性 | ≥ 2/3 评委判为"不同解释" | 真实 vs 假译码盲测 |
| 可读性 | ≥ 80% "有帮助" | 3 评委投票 |

### LLM 调用量

6 文本 × 3 差异对 × 3 prompt 风格 = 54 次译码 + 验证调用

### 估计耗时

编码 1 天 + 运行 2 小时 + 分析 1 天
