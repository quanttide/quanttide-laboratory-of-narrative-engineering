# E4-6 — 多画像评价→文本问题逆推（Reflect）

> 对应能力：Reflect（诊断层）

## 动机

p16-reader 当前能回答"谁喜欢、谁不喜欢、在哪方面喜欢"。但它不能回答"为什么"——为什么 P3 和 P1 在 character_realism 上差了 1.78 分？这个差异背后是文本的什么结构问题？

E4-6 的核心假设是：**画像间的评分差异模式，是文本结构问题的信号。**

当 P1（甜宠少女）和 P3（资深老书虫）在 character_realism 上产生分歧，这不只是"P3 更挑剔"——它可能意味着文本的角色行为只有外显动机、没有内隐动机。P1 不介意（甜宠套路够用），P3 介意（需要心理深度）。

## 设计要点

### 输出：只诊断，不修复

E4-6 的输出**只包含诊断**（root_cause + evidence + structural_type + severity），不包含 suggested_fix_direction。把"怎么改"留给 E4-7。

```
E4-6 (Reflect):     这里不对，因为______
E4-7 (Rewrite):     把这一行改成______
```

### 样本量

6 篇 Phase I 文本 × 3 prompt 风格 = 18 次 LLM 调用。用 3 种不同 prompt 表述方式，减少单风格偏见。

### 阴性对照

4.1（咖啡厅重逢）是所有画像公认的高分文本，画像间分歧小。它的诊断应显著不同于 2.3（傍晚小龙虾，低分且分歧大）。如果两个的诊断结果相同或相似，则诊断系统没有区分能力。

### 验证：LLM 评审 + forced choice，不依赖 embedding

所有验证通过 LLM 评审（3 次投票 + 多数决），不用 embedding cosine。因 embedding 测的是"说得像不像"，不是"说得对不对"。

## 输入

| 来源 | 内容 | 格式 |
|------|------|------|
| `e4-1_raw.json` | 5 画像 × 6 文本 × 3 call 的完整评分 | JSON |
| `assets/fiction/4_成稿/` | 6 篇测试文本的全文 | Markdown |

## 步骤

### Step 1: 差异维度分析

对每篇文本，计算画像间差异：

- 各维度的画像间方差 / η²（确定"最分歧的维度"）
- 各画像对在各维度的 Cohen's d（确定"谁和谁最不一致"）
- 输出每篇文本的"差异特征剖面"——差异最大的维度、分歧最大的画像对、效应量

输入 E4-1 数据输出差异特征，纯数据处理，不需要 LLM。

### Step 2: 逆推诊断

构建 3 种不同风格的 Reflect Prompt，输入：

1. 文本全文
2. 画像定义（P1-P5 行为锚定）
3. 5 画像在该文本上的完整评分分布
4. 差异特征剖面（Step 1 的产出）

3 种 prompt 风格在措辞和结构上不同（例如：一个偏结构化的"逐维度分析-汇总差异-给出诊断"，一个偏自由的"作为资深编辑，你看到了什么问题？"，一个偏数学的"给定这个差异矩阵，最合理的解释是什么？"）。

输出结构：

```json
{
  "text_id": "2.3",
  "diagnosis": {
    "root_cause": "角色行为缺乏内隐动机——林远亭的多句对话只有外显含义，没有社交越界的张力",
    "evidence": "P1（甜宠少女）可以接受简单的'他喜欢她所以说了这句话'逻辑，P3（资深老书虫）要求每句话都有心理深度",
    "structural_type": "因果链断裂",
    "severity": "medium"
  }
}
```

`structural_type` 取值为预定义的有限集合：
- 因果链断裂（行为与动机之间缺乏合理连接）
- 角色平面化（人物只有功能型特征，没有内在矛盾）
- 对话无张力（对话只有信息传递，没有情感暗流）
- 节奏问题（场景内部的情感曲线太平或太陡）
- 信息过载/不足（读者需要或不需要的信息量错位）
- 其他（需说明）

`severity` 取值：low / medium / high

### Step 3: 验证

#### 验证 1 — 诊断合理性

对每个诊断（3 风格 × 6 文本 = 18 份），让 3 个 LLM 评委（不加温度）独立回答：

> "这个诊断是否合理——即 root_cause 是否与文本中的具体内容相符？evidence 是否提供了有说服力的引用？选择：合理 / 不合理"

- 计算方式：18 份诊断 × 3 评委 = 54 次判断。取每份诊断的多数决结果
- 标准：**≥ 14/18 诊断被判为"合理"**（≈ 80% 通过率）

#### 验证 2 — 诊断可区分性（阴性对照）

对两篇文本的诊断做配对比较：

> "下面两段诊断指向的是同一个问题，还是两个不同的问题？选择：同一问题 / 不同问题"

对比 4.1（高分+低分歧）vs 2.3（低分+高分歧）的诊断。3 评委 × 18 种配对（3 风格 × 3 风格对比）= 54 次判断。

- 标准：**≥ 80% 的配对判为"不同问题"**

#### 验证 3 — 诊断稳定性

同一文本的 3 种 prompt 风格的诊断之间，structural_type 是否一致？

- 计算方式：对每篇文本，3 个诊断的 structural_type 取多数决。如果 3 个中有 ≥ 2 个选择了同一个 structural_type，记为该文本"稳定诊断"
- 标准：**≥ 4/6 文本的 structural_type 稳定**

### 成功标准

| 验证 | 标准 | 方法 |
|:---|:---:|:---|
| 诊断合理性 | ≥ 14/18 合理 | 3 评委 × 18 诊断，多数决 |
| 诊断可区分性（阴性对照） | ≥ 80% 判为不同问题 | 3 评委 × 18 配对，4.1 vs 2.3 |
| 诊断稳定性 | ≥ 4/6 文本类型一致 | 3 风格 majority vote |

- 前置：Phase I 全部通过（E4-0/1/2✅）
- 数据：e4-1_raw.json（已有）
- LLM 调用量：6 文本 × 3 prompt 风格 = 18 次诊断 + 验证调用
