# E4-6 — 多画像评价→文本问题逆推（Reflect）

> 对应能力：Reflect（诊断层）

## 动机

p16-reader 当前能回答"谁喜欢、谁不喜欢、在哪方面喜欢"。但它**不能回答"为什么"**——为什么 P3 觉得 cliche_level 高？为什么 P1 觉得 character_realism 低？这些"为什么"藏在文本的结构里，而 p16-reader 的多画像评价本质上是一个**差异信号**。

E4-6 的核心假设是：**画像间的评分差异模式可以映射到文本的特定结构问题。**

## 核心洞察

例如在 Phase I 数据中，2.3（傍晚小龙虾）的 cliche_level 画像间差异最大：
- P3（老书虫）给了高分（4.0）→ 套路感强
- P1（甜宠少女）给了中分（3.67）→ 对甜宠套路容忍度高
- 差值 = 0.33，但 P3 在 character_realism 上比 P1 高 1.67

这告诉我们 2.3 的问题**不是套路本身，而是角色行为缺乏真实动机**——套路感强只是症状，角色单薄才是根因。

## 实验设计

### 输入

| 来源 | 内容 | 格式 |
|------|------|------|
| `e4-1_raw.json` | 5 画像 × 6 文本 × 3 call 的完整评分 | JSON |
| `assets/fiction/4_成稿/` | 6 篇测试文本的全文 | Markdown |

### 步骤

**Step 1: 差异维度识别**

对每篇文本，计算各评分维度的画像间差异矩阵：

```
对于文本 T，画像 P1-P5 在维度 D 上的评分分布
→ 找到"分歧最大"的维度（η² 最大或方差最大）
→ 找到"谁和谁最不一致"（两两 Cohen's d）
```

输出：每篇文本的"画像差异特征"——最分歧的维度、最分歧的画像对、效应量。

**Step 2: 逆推诊断**

构建 Reflect Prompt，输入：
1. 文本全文
2. 画像定义（P1-P5）
3. 5 画像在该文本上的完整评分分布
4. 差异分析（分歧最大的维度 + 分歧最大的画像对）

输出诊断：
```json
{
  "text_id": "2.3",
  "primary_disagreement": {
    "dimension": "character_realism",
    "divergent_profiles": {"P1": 3.0, "P3": 5.33},
    "cohens_d": 1.67
  },
  "diagnosis": {
    "root_cause": "角色行为缺乏内隐动机——林远亭的多句对话只有外显含义，没有社交越界的张力",
    "evidence": "P1（甜宠少女）可以接受简单的'他喜欢她所以说了这句话'逻辑，P3（资深老书虫）要求每句话都有心理深度",
    "structural_type": "因果链断裂/动机缺失",
    "severity": "medium",
    "suggested_fix_direction": "为林远亭的每句关键对话补充内隐动机（他为什么在此时说这句话？不说会怎样？）"
  }
}
```

### 输出

每篇文本一个诊断 JSON：
```
data/output/e4-6_reflect_{tid}.json
汇总：data/output/e4-6_reflect_summary.json
```

### 验证

**验证1 — 诊断一致性**：两套独立的提示表述（不同 prompt 风格）对同一文本的诊断是否一致？

- 方法：构造 2 个不同风格的 reflect prompt（例如一个偏结构化差异分析、一个偏自由诊断），分别调用 LLM
- 指标：对同一篇文本，两个诊断在"root_cause"层面的语义相似度（embedding cosine ≥ 0.70）
- 标准：平均 ≥ 0.70

**验证2 — 诊断可区分性**：不同文本的诊断是否显著不同？

- 方法：取所有 6 篇文本的诊断 root_cause embedding，做跨文本余弦距离矩阵
- 指标：同一文本的两个 prompt 风格的距离 < 不同文本的距离
- 标准：类内距离 / 类间距离 < 0.80

**验证3 — 诊断可操作性**：诊断中是否包含 actionable 的修改方向？

- 方法：LLM 判断（3 次投票）每篇文本的诊断中 suggested_fix_direction 是否"知道具体改什么"
- 标准：≥ 4/6 文本的诊断判断为 actionable

### 成功标准

| 验证 | 标准 | 方法 |
|:---|:---:|:---|
| 诊断一致性 | 平均 cosine ≥ 0.70 | 双 prompt 风格诊断的语义相似度 |
| 诊断可区分性 | 类内/类间距离 < 0.80 | 跨文本 embedding 比较 |
| 诊断可操作性 | ≥ 4/6 actionable | LLM 评委 3 次投票 |

### 依赖

- 前置：Phase I 全部通过（E4-0/1/2✅）
- 数据：e4-1_raw.json（已有）
- LLM 调用量：6 篇文本 × 2 prompt 风格 = 12 次
