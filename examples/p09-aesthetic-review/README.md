# p09 — 复刻作者审美评审实验

验证 LLM 能否内化特定人类作者的审美偏好，并以该作者的审美标准评审新场景。

## 假设

1. 一个人类作者的审美偏好可以编码为 style.yaml（10 维度 + clues + confidence + excerpts + 完整场景 samples）
2. LLM 阅读该 style 后，可以内化这个作者的审美框架——不只是理解"什么是好文字"，而是理解"这个作者认为什么是好文字"
3. LLM 用这种内化审美评审新场景时，其评分模式与作者自身的 confidence 分布显著相关
4. 同一 style 内化的 LLM 评审比"通用文学评审"更接近作者本人的审美判断

## 背景

p04 验证了"LLM 能否理解 Review 意图"（通用评审能力）。p09 进一步问："LLM 能否学会一个特定作者的审美？"

Gallery 中已经存在这种审美的编码：
- 每个维度有 `confidence`（作者对这个维度的确信度）
- 每个维度有 `clues`（为什么这个维度存在？证据是什么？）
- 每个维度有 `tensions`（这个维度的例外在哪里？）
- `excerpts` 是作者亲手筛选的"最能代表这个维度的段落"

这些信息合在一起，构成了一套完整的审美逻辑——不只是在说"这个文本写了什么"，而是在说"我为什么认为这些是这个文本最值得关注的特质"。

**p09 的核心贡献**：将 style.yaml 从"分析工具"升级为"审美代理人"——LLM 可以带着作者的审美框架去评审作者未曾提供样本的新场景。

## 数据来源

### 审美训练材料（从作者文本中提炼的审美编码）

| 材料 | 来源 | 内容 |
|------|------|------|
| 维度定义 | `style.yaml` → `dimensions[]` → `title` + `description` | 10 个维度的详细描述（含具体场景和手法） |
| 维度确信度 | `style.yaml` → `dimensions[]` → `confidence` | 从文本中提炼该维度时归纳的置信程度（0-1） |
| 支撑线索 | `style.yaml` → `dimensions[]` → `clues[]` | 支撑每个维度判断的原文线索 |
| 审美张力 | `style.yaml` → `dimensions[]` → `tensions[]` | 维度的边界情况或内在冲突 |
| 示例段落 | `style.yaml` → `excerpts[]` → `paragraph` + `dimension` + `note` | 代表性短片段 + 选这段的原因 |
| 完整场景 | `samples/` → `sample1.md` ~ `sample8.md` | 精选的完整场景（都市 8 篇、校园 2 篇） |
| **负样本** | 都市初稿 | 1-2 篇初稿场景作为"不够好的反面示例"——标注在哪些维度上未达到成稿标准 |

### 测试文章（用于评审）

| # | 来源 | 文章 | 文件 | 说明 |
|---|------|------|------|------|
| T1 | 都市成稿 | 深夜失眠 | `assets/fiction/职场言情/4_成稿/1_2_深夜失眠.md` | 成稿——接近样本质量 |
| T2 | 都市初稿 | 咖啡厅重逢 | `assets/fiction/职场言情/3_初稿/1_1_咖啡厅重逢.md` | 初稿——应比成稿评分低 |
| T3 | 都市成稿 | 书房陪伴 | `assets/fiction/职场言情/4_成稿/10_1_书房陪伴.md` | 成稿——预期高分 |
| T4 | 都市初稿 | 赏雪谈心 | `assets/fiction/职场言情/3_初稿/赏雪谈心.md` | 独立初稿——未参与任何前序实验 |
| T5 | 校园成稿 | 第五章 | `assets/fiction/校园言情/4_成稿/5_第五章.md` | 校园系列——测试审美跨系列转移 |

选择理由：
- T1-T3 来自都市言情系列，与 style 匹配，有明确的审美基线
- T2 是初稿，理论上应在某些维度低于成稿（量化初稿→成稿的审美提升）
- T4 是完全独立的盲测文章——没有任何前序实验接触过
- T5 来自另一个系列——测试"审美是否可跨系列转移"（都市审美能否评校园文字？）

### 对照组（通用审美）

| 评审方式 | 输入 | 说明 |
|---------|------|------|
| **作者审美**（实验组） | style.yaml + 3 篇 samples | LLM 带着从文本提炼的审美框架评审 |
| **通用文学评审**（对照组） | 无 style 约束 | LLM 用通用文学标准评审 |
| **盲审(A)**（消融组） | 3 篇 samples only | 测试少量样本能否内化审美 |
| **盲审(B)**（消融组） | 6 篇 samples only | 测试样本量的边际收益

## 实验步骤

```
步骤 1: 审美内化
  └─ 将都市言情 style.yaml（10 维度 + clues + tensions + excerpts）+
      3 篇 samples（sample1/3/7）打包为"审美训练集"
  └─ 构建系统 prompt："以下是《都市言情》系列的风格框架，从该系列的成稿中提炼而成。
      请用它来评审下面的场景——判断该场景在多大程度上体现了这个风格框架的各个维度。"
  └─ 对照组：无 style 约束，纯通用文学评审
  └─ 消融组 A：仅 3 篇 samples，不含维度定义
  └─ 消融组 B：仅 6 篇 samples（full set），不含维度定义
      （测试样本量的边际收益——3 篇 vs 6 篇是否显著提升审美内化？）
  └─ 所有组的 scoring 统一使用 temperature=0（确保可复现），reasoning 使用 temperature=0.2

步骤 2: 维度对齐评分
  └─ 对 T1-T5 逐篇在 10 个维度上做 1-10 评分
  └─ 每个评分附带：
      ├─ 引用原文证据（类似 style.yaml 中的 clues）
      ├─ 如有边界/张力也标注（类似 tensions）
      └─ 一句话理由（类似 excerpts 中的 note）
  └─ 输出格式：
      {"dimension_scores": [
        {"dimension": "叙事视角", "score": 8, "evidence": ["原文引用"],
         "contradiction": null, "note": "..."
      ]}

步骤 3: 审美一致性检验
  └─ 指标 A（主指标）：方向匹配率
      ├─ confidence 反映的是"维度提炼的确信度"而非场景评分——
      │   高 confidence 意味着该维度在系列整体中是确定存在的，
      │   但不保证当前特定场景在该维度上必然高分。
      ├─ 因此不以 ρ 作为唯一指标。补充方向匹配率：
      │   作者的 top-3 confidence 维度是否也在 LLM 的 top-3 评分维度中？
      └─ 辅助：全维度 Spearman's ρ（预期天花板 0.5-0.7）
  └─ 指标 B：LLM 能否识别作者的"矛盾"
      ├─ 作者在 style.yaml 中标注了 tensions（如"互动段落偏轻快，与深情底色有违和"）
      └─ LLM 在评审时是否独立发现了同样的 tension？
  └─ 指标 C：初稿→成稿的评分梯度
      ├─ T2（初稿）在"语言风格"和"情感表达"维度上是否低于 T1/T3（成稿）？
      ├─ 注意：部分维度（如"情感表达"）在初稿中可能反高于成稿——
      │   初稿有未被克制滤镜筛选的原始能量。梯度方向不作为硬性错误。
      └─ 差异是否与 style.yaml 中同维度的 clues 密度一致？

步骤 4: 样本排序一致性
  └─ 作者在 style.yaml 中按维度组织了 excerpts
  └─ 从 T1-T4 中为每个维度摘取 1 个对应段落（与 excerpts 相同粒度）
  └─ 将 excerpts 与测试摘取段落混合（同粒度对比，不做短段落 vs 完整场景的跨粒度比较）
  └─ 让 LLM 对每个维度做排序："这些段落中，哪个最能体现[叙事视角]？从 1（最）排到 N"
  └─ 作者选的 excerpts 是否仍然排在最前面？
      ├─ 如果 excerpts 排 top → 说明 LLM 的排序与作者一致
      └─ 如果某些测试摘取段落超越 excerpts → 说明这些测试文章在该维度上特别强

步骤 5: 审美跨系列转移
  └─ 用都市言情的审美框架评审 T5（校园言情）
  └─ 计算跨系列评分与校园自身 style 评分的差异
  └─ 对差异做归因：是"审美框架不匹配"（genre_fit）还是"LLM 评审不准"？
  └─ 为每个维度标注 `genre_fit`（1-3）：
      ├─ 3 = 该维度定义是跨系列可用的（如"细节美学"）
      ├─ 2 = 该维度部分适用但需要调整（如"叙事节奏"）
      └─ 1 = 该维度定义是系列专属的（如都市"情感表达=含蓄" vs 校园"半直球"）
  └─ 预期：genre_fit=3 的维度分差 < 1，genre_fit=1 的维度分差 > 3
```

## 验收标准

### 审美内化

| 指标 | 合格线 | 良好线 | 优秀线 |
|------|--------|--------|--------|
| 方向匹配率（作者 top-3 confidence vs LLM top-3 score） | 2/3 匹配 | 2/3 + 无交叉错配 | 3/3 |
| Spearman's ρ（辅助，预期天花板 0.5-0.7） | > 0.3 | > 0.5 | > 0.65 |
| LLM 独立发现 ≥1 个作者标注的 tension | — | 1 个 | 2+ 个 |

### 评审质量

| 指标 | 合格线 | 良好线 | 优秀线 |
|------|--------|--------|--------|
| 初稿(T2)评分 < 成稿(T1)评分（语言风格 + 情感表达维度） | 至少 1 个维度 | 至少 2 个 | — |
| 作者 excerpts 排序仍在前 50%（vs 同粒度测试摘取段落） | > 50% | > 70% | > 85% |
| "作者审美"评审 vs "通用评审"的评分方差 > 通用评审内部方差 | 显著差异 | p < 0.1 | p < 0.05 |

### 跨系列转移

| 指标 | 合格线 | 良好线 | 优秀线 |
|------|--------|--------|--------|
| genre_fit=3 维度的分差（都市审美 vs 校园自身） | < 2 分 | < 1.5 分 | < 1 分 |
| genre_fit=1 维度的分差（预期失败是框架不匹配，非 LLM 问题） | > 2 分（confirming） | — | — |

### 消融对比

| 对比 | 预期 |
|------|------|
| 作者审美 > 盲审(6) ≥ 盲审(3) > 通用评审 | 维度定义的增量价值 + 样本量的边际价值 |
| 如果 盲审(6) ≈ 作者审美 | → 维度定义无增量价值，style.yaml 可简化为 samples |
| 如果 盲审(3) ≈ 盲审(6) | → 3 篇 samples 已饱和，不需要完整 8 篇 |
| 如果 盲审(3) ≈ 通用评审 | → samples 本身未携带足够审美信息（最不可能的结果） |

### 输出产物

1. `results/dimension_scores_T1.json` ~ `results/dimension_scores_T4.json`（主实验文章 × 10 维度评分）
2. `results/dimension_scores_T5.json`（跨系列子分析文章，仅用于步骤 5）
3. `results/aesthetic_alignment.json`（方向匹配率 + Spearman's ρ + tension 发现 + 初稿/成稿梯度）
4. `results/ranking_consistency.json`（同粒度 excerpts vs 测试摘取段落排序对比）
5. `results/cross_series_transfer.json`（都市审美评校园 vs 校园自我评审 + genre_fit 标注）
6. `results/ablation.json`（作者审美 vs 盲审(3) vs 盲审(6) vs 通用评审 四栏对比）

> 评估范围：主指标（方向匹配率 + ρ + tension + 初稿梯度）基于 T1-T4；T5 仅用于跨系列子分析。预计总数据点约 400 个（4 篇 × 10 维度 × 4 组 × 评分），而非原始设计的 600+。

## 结果预期

- **方向匹配率**（主指标）：预期作者 top-3 confidence 维度（总体风格 0.95、情感表达 0.95、整体美学 0.90）中至少 2 个出现在 LLM 的 top-3 评分维度中。之所以可能不是全部 3 个，因为 confidence 反映的是"该维度在系列整体中的确定性"，而非"当前特定场景在该维度上的表现"。
- **Spearman's ρ**（辅助）：预期在 0.4-0.6 之间——confidence 和 score 是不同量度，天花板明显。如果 ρ > 0.7，说明作者 confidence 意外地也是一个好的场景级评分预测器。
- **最关键的检验**：LLM 能否独立发现作者标注的 tensions？如果能，说明 LLM 不仅理解了"这个框架的评价标准"，还理解了"这个框架对自己适用范围的怀疑"——这是审美内化的高阶标志。"互动段落轻快 vs 深情底色违和"这个矛盾在 T3（书房陪伴）中反复出现，LLM 应能捕获。
- **消融对比**：作者审美 > 盲审(6) ≥ 盲审(3) > 通用评审 的方向应该成立。最关键的消融发现是盲审(3) vs 盲审(6) 的对比——如果 3 篇样本已饱和，则 6 篇无增量价值，samples 规模可以缩减。
- **跨系列转移**：genre_fit=3 的维度（细节美学、整体美学）分差应 < 1.5；genre_fit=1 的维度（情感表达）分差应 > 2——这不是 LLM 的失败，而是"都市=含蓄 vs 校园=半直球"框架不完全对齐的体现。
- **如果失败**：如果方向匹配率 = 0/3 且 LLM 无法发现任何 tension，说明 style.yaml 的 10 个维度作为审美编码过于抽象——LLM 需要更具体的可执行检查项，而非描述性的维度定义。

## 产品意义

如果 p09 验证成功，意味着：

1. **每个作者可以有专属的评审 Agent**——不只是"AI 帮你改文章"，而是"一个理解你审美的 AI 帮你改文章"
2. **style.yaml 的定位升级**——从"分析参考"升级为"审美配置文件"（aesthetic config），类似 ESLint 之于代码风格
3. **跨系列审美转移的边界**——如果跨系列转移失败，说明审美是 series-specific 的，需要每个系列独立配置
4. **初稿到成稿的量化路径**——如果 LLM 的评分梯度与初稿→成稿的改进方向一致，可以为作者提供"从当前版本到成稿的差距量化"
