# p02 — 3R 循环效果度量实验

验证 3R 循环（Review → Reflect → Rewrite）能否量化提升文章质量，以及收益递减点在哪里。

## 假设

1. 运行 3R 循环后，文章的多维度评分显著提升
2. 存在收益递减拐点（第 n 轮后改善不再显著）
3. 不同写作问题类型（空隙 vs 风格 vs 结构）对 3R 的响应不同

## 数据来源

### 实验组（低质量起点）

从案例库中选 3 篇初稿（人工判定为"可改进"）：

| # | 文章 | 类型 | 文件 |
|---|------|------|------|
| A | 咖啡厅重逢 | 职场言情 draft | `../../../assets/fiction/职场言情/3_初稿/1_1_咖啡厅重逢.md` |
| B | 赏雪谈心 | 校园言情 draft | `../../../assets/fiction/校园言情/3_初稿/赏雪谈心.md` |
| C | 第六章（论坛热搜） | 校园言情 draft | `../../../assets/fiction/校园言情/3_初稿/6_第六章.md` |

### 对照组（高质量基准）

取同一作者已完成定稿的篇章，作为"好文章"的风格基准：

| # | 文章 | 文件 |
|---|------|------|
| R1 | 深夜失眠 | `../../../assets/fiction/职场言情/4_成稿/1_2_深夜失眠.md` |
| R2 | 傍晚小龙虾 | `../../../assets/fiction/职场言情/4_成稿/2_3_傍晚小龙虾.md` |
| R3 | 书房陪伴 | `../../../assets/fiction/职场言情/4_成稿/10_1_书房陪伴.md` |

对照组用途：提供风格参照（非用于对比评分，而是让 LLM reviewer 有"好文章长什么样"的上下文）。

## 评分维度

| 维度 | 定义 | 评分方式 |
|------|------|---------|
| 空隙密度 | 每百字含几处可检测空隙（时间跳跃、对话间隙、动作空隙、视角切换、过渡压缩） | LLM 检测计数 / 总字数 × 100 |
| 风格一致度 | 与对照组风格特征的吻合程度 | LLM 对比打分 0-10 |
| 情感张力 | 情感表达的层次感和释放节奏 | LLM 打分 0-10 |
| 细节密度 | 每百字含几个具体可感的物象细节 | LLM 检测计数 / 总字数 × 100 |
| 整体评分 | 综合可读性、完整性、感染力 | LLM 打分 0-10 |

## 实验步骤

```
步骤 1: 基线评分
  └─ 对 A/B/C 三篇文章运行首次 Review，记录 5 维评分
  └─ 将 R1/R2/R3 作为风格上下文提供给 LLM（不评分，仅作参考）

步骤 2: 执行 3R 循环（共 5 轮）
  └─ 每轮结构：
      Review  → 输出空隙列表 + 各维度评分
      Reflect → 对每个空隙/低分维度生成改写提示
      Rewrite → 根据提示改写文章
  └─ 每轮后记录新的评分

步骤 3: 对比策略（选做）
  └─ 选 1 篇文章额外用两种变体策略跑 3 轮：
      ├─ Review-heavy：每轮 review 2 次才 reflect
      └─ Rewrite-heavy：根据同一份 review 连续 rewrite 2 次再 reflect
  └─ 对比标准策略与变体策略的效果差异

步骤 4: 收益递减分析
  └─ 对每篇文章，绘制 5 轮各维度评分变化曲线
  └─ 找出首个"改善 < 上一轮改善 50%" 的轮次（定义为递减拐点）
```

## 验收标准

1. 3 篇文章 × 5 轮 × 5 维度的完整评分数据
2. 每篇文章的评分变化曲线图（5 维叠加）
3. 标出每篇文章的收益递减拐点
4. 变体策略对比数据（如执行了）
5. 定性总结：3R 在哪些维度改善最明显、对哪类文章最有效

## 不依赖

不依赖产品代码，但可复用 `examples/p01-writing-poc/write_poc.py` 中的 ReviewEngine 逻辑做参考。核心评分工作由 LLM 完成。

## 结果预期

最可能的产出：
- 首轮改善最大（+15-30%），此后逐轮递减
- 空隙密度在第 1-2 轮快速改善，风格一致度需要 2-3 轮才见效
- 拐点大概率出现在第 3-4 轮之间

如果 5 轮后整体改善 < 10%，说明 3R 循环作为方法论需要重新设计，产品不能依赖它。
