# p02 实验结果

## 结论

**LLM 评审的评分波动太大，无法可靠度量 3R 循环的逐轮改善效果。** 这不是 3R 无效，而是实验设计有缺陷——评分工具不够稳定。

## 关键数据

### 评分波动

```
咖啡厅重逢: 8 → 7.5 → 8 → 7 → 7   (总趋势 ≈ 不变)
赏雪谈心:   7 → 8 → 8 → 7 → 7   (总趋势 ≈ 不变)
第六章:     8 → 7 → 8 → 7 → 7.5 (总趋势 ≈ 不变)
```

每轮评分在 ±1 分内波动，看不出任何改善趋势。波动量 ≈ 改善量，信号被噪声淹没。

### 空隙数量变化

```
咖啡厅重逢: 始终 5 个
赏雪谈心:   5 → 5 → 5 → 8 → 6 (波动)
第六章:     始终 5 个
```

空隙数量也没有减少。两种可能：
1. 空隙是结构性缺陷（需要人类判断而非 AI 改写）
2. 评审器每次检测的空隙不一致

### 风格一致度

style 维度在 7-8 之间稳定，说明 3 篇初稿本身风格与成稿差距不大。

## 实验设计的缺陷

| 缺陷 | 表现 | 影响 |
|------|------|------|
| **评分器即评审器** | 同一 LLM 既评审又改写，评分偏向"自己改过的版本" | 评分不独立 |
| **评分粒度太细** | 0-10 分制，LLM 偏好打 7-8，不产生区分度 | 无法捕捉微小改善 |
| **参考文章风格不一致** | 对照组成稿与实验组初稿来自不同章节，风格基线不准 | 风格一致度评分不可靠 |
| **缺少人类基线** | 没有人类评审对照，不知 LLM 评分是否有偏差 | 无法校准 |

## 发现（有意义的结论）

1. **3 篇初稿的基线评分都在 7-8**——说明初稿本身质量已经较高，改善空间有限。应该换成真正的低质量文本（如 AI 生成的初稿、英文机翻）
2. **风格一致度始终稳定**——说明改写不会破坏作者原有的风格特征（风格在语义层面是鲁棒的）
3. **改写在增加字数**——A 从 ~1500 字增长到 ~2000 字，说明改写倾向于"增加内容"而非"精简优化"
4. **空隙难以通过 AI 改写消除**——5 轮后空隙数量未减少，说明空隙需要作者本人识别和补充，AI 改写无法自动填充

## 对产品的启示

1. **3R 评审器不能同时做评分**——产品中 Review 应输出可操作的评审意见，评分应用独立的评价模型或人工
2. **改善度量需要独立的评价集**——用成稿做参照（如 style.md 中的美学关键词），而不是让同一模型既改又评
3. **3R 用在低质量文本上可能更有效**——类似 p02 的"初稿质量已经较高"的场景，3R 改善空间有限；但在 AI 初稿、翻译稿等场景可能效果显著
4. **空隙检测不应依赖 LLM Review 的副产品**——空隙检测需要基于规则的引擎（如 p01 的 ReviewEngine），与 LLM 评分解耦

## 实验元数据

- 执行时间：2026-06-03
- 模型：deepseek-chat
- Temperature：0.3
- 总调用：45 次 LLM API（3 文章 × 5 轮 × 3 步骤）
- 实验组：3 篇初稿（咖啡厅重逢/赏雪谈心/第六章）
- 对照组：3 篇成稿（深夜失眠/傍晚小龙虾/书房陪伴）
