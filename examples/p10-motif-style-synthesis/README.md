# p10 — 母题驱动的风格改进实验

验证"先诊断风格维度弱点 → 用母题调整来修复"的双层改法是否优于单一策略。

## 假设

1. 风格问题是表征，母题缺失是根因——一个场景"情感表达"偏弱，往往是因为缺少该系列的核心情感母题（如都市言情的"手势优先于语言"）
2. 针对根因的改法（调整母题）比针对表征的改法（直接要求"加强情感表达"）更有效
3. 母题-风格的映射关系是可学习的——LLM 能建立"某维度弱→应增强/引入某母题"的诊断链
4. 三组改法各有差异化优势（非简单层级）：组合强在具体性和自然度，母题单层强在根因对准和母题一致，风格单层强在风格覆盖

## 背景

p05-p09 建立了两个独立的管线：

| 管线 | 能力 | 代表性实验 |
|------|------|-----------|
| **风格管线** | 编码审美框架 → 评审场景 → 诊断维度弱点 | p09 |
| **母题管线** | 提取母题 → 识别缺口 → 生成多向改进建议 | p05, p08 |

p10 将两条管线**交叉**：风格维度诊断告诉我们"这个场景哪里不对"，母题知识告诉我们"怎么调整能让它对"。

**核心 insight**（来自跨实验综合）：p07 发现"孤独"母题的 0% 覆盖率不是因为 LLM 无法生成孤独，而是因为"孤独"是分析性概念——它在文本中表现为"手势先于语言""日常的缝隙"等具体母题。同理，如果 p09 的审美评审报告说"情感表达维度偏低"，改法不应该是"加强情感表达"（跟说"写得好一点"一样无用），而应该是"这个场景缺了'手势'母题——加入递毛巾/擦眼泪/拥抱的动作序列"。

## 实验步骤

```
步骤 1: 双层诊断
  └─ 对 T1-T4 先做风格评审（p09 方法），得到每篇文章的维度评分
  └─ 再做母题提取（p05 方法），得到每篇文章的母题覆盖
  └─ 生成"风格-母题关联报告"：
      ├─ 弱维度列表（score < 6）
      ├─ 缺失母题列表（未提取到的）
      └─ LLM 自主推断的关联："[弱维度] 可能与 [缺失母题X] 相关，因为..."

步骤 2: 三组改法生成
  └─ 对每篇文章的每个弱维度，从三组各生成一条改法：

  | 组 | 提示词约束 | 示例（T1 "情感表达"维度 = 5 分） |
  |----|-----------|------------------------------|
  | **组合改法** | "这个场景的情感表达维度偏低。已知该系列的核心情感母题是手势（动作先于语言）。请给出引入该母题的具体改写建议" | "在第三段'她看着他'之后，加入一个无意识的动作——她抬起手想碰他的肩膀，手在空中停了半秒又放下。这个动作不需要解释" |
  | **风格单层** | "这个场景的情感表达维度偏低。请给出加强情感表达的建议" | "建议在对话中加入更多情感的流露，让角色说出内心真实的感受" |
  | **母题单层** | "这个场景缺少'手势'母题（定义：全书最高级的情感表达在手里）。请从6个方向给改法" | p08 风格建议 |

  └─ 使用 DeepSeek API，temperature=0.3（需要具体的改法而非创意发散）
  └─ 每条改法 80-150 字 + 具体段落引用

步骤 3: 改法质量评估
  └─ 对三组改法从 5 个维度做 1-5 评分：

  | 维度 | 定义 |
  |------|------|
  | **具体性** | 改法是否给出了具体的、可执行的文本操作？vs 泛泛的"写得好一点" |
  | **根因对准** | 改法是否对准了风格问题的真实原因？ |
  | **母题一致** | 改法是否确实会增强目标母题？ |
  | **自然度** | 按改法修改后的场景是否自然？ |
  | **风格覆盖** | 按改法修改后，目标风格维度是否会被提升？ |

  └─ LLM 自动初评（5 维度）+ 人工抽样 12 条验证（T1/T2 各选 2 个维度 × 3 组）

步骤 4: 风格-母题映射学习
  └─ 收集步骤 1 中 LLM 自主推断的风格-母题关联
  └─ 与 gallery 中人工定义的关联做对比：

  | 风格维度（p09） | 预期关联的母题 | 是否被 LLM 自主发现？ |
  |---------------|-------------|---------------------|
  | 情感表达 | 手势、十年 | ？ |
  | 时间结构 | 十年 | ？ |
  | 细节美学 | 手势（毛巾/外套等物件）| ？ |
  | 叙事节奏 | 孤独（内心独白密度）| ？ |
  | 时代质感 | 歌声 | ？ |
  | 语言风格 | ？ | 探索性——有独立关联吗？ |

  └─ 输出"风格-母题映射矩阵"（human vs LLM-inferred）
  └─ 计算 LLM 映射与人工映射的 Jaccard 相似度

步骤 5: 成稿对照
  └─ 对 T3（成稿）做同样的双层诊断
  └─ 对比初稿(T1/T2) vs 成稿(T3)的风格评分 + 母题覆盖
  └─ 问题：成稿在哪些微调和母题组合上优于初稿？这些差异是否被步骤 1 的 LLM 诊断捕获？
```

## 验收标准

### 双层诊断

| 指标 | 合格线 | 良好线 | 优秀线 |
|------|--------|--------|--------|
| LLM 自主发现的风格-母题关联与人工标注的 Jaccard | > 0.3 | > 0.5 | > 0.7 |
| 初稿(T1+T2) avg 风格评分 < 成稿(T3) | ⚠️ 降级为探索性——当前数据中 T3 评分低于 T1，跨场景的风格评分可比性存疑 |

### 改法质量

| 指标 | 合格线 | 良好线 | 优秀线 |
|------|--------|--------|--------|
| 组合改法的具体性 > 风格单层 | 显著 > | 差距 > 1 分 | 差距 > 1.5 分 |
| 组合改法的自然度 > 风格单层 | 显著 > | 差距 > 0.5 分 | 差距 > 1 分 |
| 母题单层的根因对准 > 组合改法 | 显著 > | 差距 > 0.5 分 | 差距 > 1 分 |
| 三组各有独立优势维度（非层级） | 2 组有 | 3 组各有 | 3 组 + 差异显著 |

### 风格-母题映射

| 指标 | 合格线 | 良好线 | 优秀线 |
|------|--------|--------|--------|
| 人工标注的 5 个映射中 LLM 成功恢复数 | > 2 | > 3 | > 4 |
| LLM 提出的新映射中有 ≥1 个被人工确认为合理 | — | 1 个 | 2+ 个 |

### 输出产物

1. `results/diagnosis_T1.json` ~ `results/diagnosis_T4.json`（风格评分 + 母题覆盖 + 关联推断）
2. `results/suggestions_{combined|style|motif}_T1.json` ~ `results/suggestions_{combined|style|motif}_T4.json`（三组改法）
3. `results/evaluation.json`（三组 × 5 维度评分矩阵）
4. `results/style_motif_mapping.json`（人类 vs LLM 映射对比 + 新发现映射）

## 结果预期

- **双层诊断**：LLM 能自主发现 2-3 个人工标注的关联（情感表达↔手势 是最可能被发现的，因为 style 的描述中已经写了"动作先于语言"）。但完整 5 对全部发现不太可能。
- **核心对比（实际发现）**：三组并非简单的层级关系，而是各有优势维度：
  - **组合改法**胜在具体性和自然度——风格诊断为其注入"为什么"，母题根因为其注入"怎么改"，执行质量最高
  - **母题单层**胜在根因对准和母题一致性——纯母题改法 cleaner，不被风格诊断稀释
  - **风格单层**胜在风格覆盖——直接对准风格维度的回应最精准，但具体性不足
- **风格单层 vs 母题单层**：风格单层的改法可能是最空洞的（"增强情感""丰富细节"）——这是 p04 已经遇到的"通用建议"问题。母题单层具体但缺乏风格诊断的上下文解释。
- **风格-母题映射**：如果 LLM 发现人工未标注的关联（如"语言风格↔歌声"），这是 p10 最具原创价值的发现——说明母题和风格的映射关系不只有 gallery 定义的那些。
- **成稿对照**：T3 应在"情感表达""细节美学""叙事节奏"等维度上高于 T1/T2，且差异可归因于特定母题的覆盖差异。
- **失败条件**：如果组合改法 ≈ 母题单层（无增量），说明风格诊断层没有提供额外信息——可以直接用 p08 的母题缺口分析替代。

## 产品意义

如果 p10 验证成功，意味着：
1. **风格诊断不再是终点**——评审报告不只是"你这里分数低"，而是"你这里分数低，因为缺了 X 母题——试试这样改"
2. **母题-风格的映射是可学习的**——新作者加入平台后，系统可以通过分析其成稿自动建立其个人的"风格维度 ↔ 母题"映射
3. **完整的产品管线**：p05(提取) → p09(诊断) → p10(根因分析) → p07/p08(生成建议) → p09(再评审验证改进)
