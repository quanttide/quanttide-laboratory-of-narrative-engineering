# p11 — 情节方向生成实验

验证给定人物状态和当前场景，LLM 能否生成多个差异化的情节走向，供创作者选择而非替代决策。

## 假设

1. 给定当前场景的人物状态（关系阶段、情绪状态、冲突状态），LLM 能生成多个合理的情节走向
2. 不同走向在母题使用、情感强度、叙事节奏上有可量化的差异
3. 走向之间不是同一模板的词语替换，而是真正不同的叙事策略

## 背景

期刊日志："两个人的互动没有蓝本可以参照，也非常难写。现在写出来的有点太熟了，太陌生的也不好写。"

当创作者卡在"不知道接下来怎么写"时，需要的不是 AI 替ta做决定，而是提供"这里还可以怎么走"的选项菜单。p11 测试 LLM 能否成为这个菜单的生成器。

## 数据来源

### 测试场景（2 个关键转折点）

从 story.yaml 中选取两个情节上最开放的场景——之后的情节走向可以有多种合理选择，且都是"巧合→必然"弧线上的关键节点：

| # | 场景 | 当前状态 | 为什么适合测试 |
|---|------|---------|--------------|
| S1 | 咖啡厅重逢后 | 两人刚刚重逢。他递了毛巾，她接过了。彼此确认了身份但还没确认心意。 | 之后可以走向"装作没事继续聊天""各自回家失眠""他直接表白"等多种方向 |
| S2 | 酒吧表白前一刻 | 两人都喝了酒。她回忆了大四聚餐。空气里弥漫着"随时可能说出口"的张力。 | 表白可以用不同方式触发——她先说/他先说、直接/婉转、酒醒后后悔/酒醒后确认 |

### 人物状态模板

对每个场景，提供当前的人物状态卡：

```yaml
scene: 便利店谈心后
lin_yuanting:
  relationship_stage: 重逢后第一次深夜独处
  emotional_state: 倾诉了压力后感到脆弱但放松，她的手放在肩上时心跳加速
  conflict: 想靠近但害怕越界
lu_zhiwei:
  relationship_stage: 终于见到他，听到他真实的心声
  emotional_state: 心疼，想继续靠近但不敢太明显
  conflict: 想说我十年前就在看你的演讲了，但怕暴露暗恋
```

## 实验步骤

```
步骤 1: 生成情节走向
  └─ 对每个测试场景，提供状态卡 + 场景原文（前 2000 字），让 LLM 生成 5 个不同的情节走向
  └─   约束：每个走向 80-150 字，必须是具体的情节推进（谁做了什么、导致了什么变化），
      不能是氛围描述或情感概括
  └─ 使用 DeepSeek API，temperature=0.8（需要创意发散）
  └─ 2 次 LLM 调用（每个场景 1 次生成 5 个走向）
  └─ 输出格式：[{"direction_id": 1, "title": "走向名称", "what_happens": "...",
      "character_motivation": "为什么这个人物会这样做", "plot_consequence": "这个走向会带来什么后续影响"}]

步骤 2: 走向多样性评估
  └─ 对每个场景的 5 个走向，从 4 个维度评估：

  | 维度 | 定义 |
  |------|------|
  | **合理性** | 这个走向是否符合已建立的人物性格？ |
  | **差异性** | 与其他 4 个走向是否真正不同（非词语替换）？ |
  | **母题延续** | 是否延续了该系列的母题（手势/十年/雨/孤独/歌声）？ |
  | **叙事价值** | 这个走向是否打开了有趣的后续发展空间？ |

  └─ LLM 自动初评 + 人工抽样验证（每个场景选 1 个走向做人工评分）

步骤 3: 走向聚类
  └─ 将 10 个走向（2 场景 × 5 走向）混合，让 LLM 按叙事策略聚类
  └─ 验证：同一场景的 5 个走向是否分布在至少 2 个不同的策略类别中？
  └─ 如果全部归为同一类 → 走向多样性不足

步骤 4: 创作者盲选
  └─ 对每个场景，混入 1 个"作者实际写的走向"（来自成稿原文）作为第 6 个选项
  └─ 打乱顺序，让 LLM 或人工判断哪个走向最像"实际发生的"
  └─ 验证：作者实际走向是否在 TOP 2 中被选中？
```

## 验收标准

### 走向质量

| 指标 | 合格线 | 良好线 | 优秀线 |
|------|--------|--------|--------|
| 5 个走向中 ≥ 3 个人工评定为"合理且有差异" | 3/5 | 4/5 | 5/5 |
| 走向间母题使用模式的 Jaccard 距离 | > 0.4 | > 0.6 | > 0.8 |
| 同场景 5 个走向分布在 ≥ 2 个策略类别中 | 2 类 | 3 类 | 4+ 类 |

### 创作者盲选

| 指标 | 合格线 | 良好线 | 优秀线 |
|------|--------|--------|--------|
| 作者实际走向在 LLM 盲选的 TOP 2 中 | ≥ 1/2 场景 | 2/2 | 2/2 + 排名第一 |

### 输出产物

1. `results/directions_S1.json` ~ `results/directions_S2.json`（每个场景 5 个走向）
2. `results/diversity_eval.json`（4 维度 × 10 走向评分矩阵）
3. `results/clustering.json`（10 走向叙事策略聚类结果）
4. `results/blind_pick.json`（每个场景的 LLM 盲选排名 + 人工选择）

## 不依赖

纯 Python 脚本 + LLM API。依赖：
- story.yaml（场景描述 + 人物状态）
- 场景原文（咖啡厅重逢、便利店谈心、酒吧表白）

## 结果预期

- **最可能**：5 个走向中 3-4 个合理且差异显著。LLM 能生成"继续暧昧""直接表白""意外事件打断""第三方介入""回到各自生活"等不同策略。最合理的走向大概率接近作者实际选择（因为人物性格已经限定了合理范围）。
- **多样性瓶颈**：S1（咖啡厅重逢后）最容易产生多样化走向（刚重逢，可能性最多），S2（酒吧表白前）最受限（张力已经很高，合理走向收窄到几种变体）。
- **最有价值的发现**：如果某个走向虽然在人物性格上不太合理，但创作者评估为"叙事价值高"（打开了有趣的后续空间），说明 LLM 的走向生成可以突破"安全但无聊"的创作惯性。
- **失败条件**：如果所有走向都是"继续聊天→感情升温→暗示心意"的模板变体（Jaccard > 0.8），说明 LLM 在情节走向上的创造力不足——它只能生成"安全"的走向。
