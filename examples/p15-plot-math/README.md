# p15 — 叙事约束矩阵（Direction C）

验证能否用 story.yaml 中已有的结构化信号（社交距离、亲密阶段、核心冲突）构建约束矩阵，替代 LLM 从原文提取因果图的方式。

## 动机

初版 p15 将 p12 的四个文学概念映射为因果图/密度函数/贝叶斯惊奇等数学对象。第一次运行的数据表明 LLM 从叙事文本中提取的因果图退化为线性时序链（out_degree=1 占 59/64 节点），四个度量因此全部失效：

| 数学概念 | 数据表现 | 根因 |
|---------|:-------:|------|
| 因果图 | 链状，出度全为 1 | LLM 提取的是时序序列，不是因果关系 |
| 图编辑距离 | 未跑通 | 链状 GED 无法区分结构改法和表面改法 |
| 事件密度 d(k) | 方差过低 | 链状节点天然均匀分布 |
| 贝叶斯惊奇 | 全部为 0 或低值 | 基于"事件已发生"而非"基于约束的概率" |

核心问题是：**LLM 当前无法可靠地从叙事文本中构建图结构**。p15 转向方向 C——放弃让 LLM 提取结构，改为直接使用 story.yaml 中已有的结构化数据构建约束矩阵。

## 约束矩阵框架

### 维度定义

对每个场景（story.yaml 中的 plot），在以下 K 个叙事维度上取值：

| # | 维度 | 值域 | 来源 | 与 p12 的映射 |
|---|------|------|------|-------------|
| D1 | `social_distance` | 0(陌生人)~5(恋人) | story.yaml 的 `clue` 中的人物行为类型 | 判因果跳跃：相邻场景跳跃 ≥ 2 为异常 |
| D2 | `tension_type` | approach/retreat/mutual/avoid | story.yaml 的 `tension` 字段 | 判节奏失衡：连续 3+ 场景同类型为单调 |
| D3 | `action_dominant` | internal/dialogue/physical/mixed | story.yaml 的 `clue` 动作标签 | 判动机不明：physical 行为但 social_distance=0 |
| D4 | `info_revealed` | none/hint/partial/full | story.yaml 的 `clue` 信息释放信号 | 判信息节奏：3+ 场景 none 后突然 full |
| D5 | `core_tension` | 0(无)~5(极强) | story.yaml 的 `tension` 字段 | 综合信号：核心约束是否在维护 |

### 矩阵结构

```
         D1   D2        D3        D4     D5
plot_1_1 [3,  "mutual", "physical", "partial", 5]
plot_1_2 [1,  "retreat", "internal", "hint",   4]
plot_2_1 [2,  "mutual", "dialogue", "partial", 4]
...      ...
plot_10_3[5,  "mutual", "mixed",   "full",   2]
```

### 三种检测规则

```
因果跳跃检测：
  if |D1(n+1) - D1(n)| ≥ 2:
     候选区间 (plot_n, plot_n+1) → p12 的 "因果跳跃"

节奏失衡检测：
  if 连续 3+ 场景的 D2 不变:
     候选区间 → p12 的 "节奏失衡"

动机不明检测：
  if D3 = "physical" and D1 ≤ 1:
     候选行为 → p12 的 "动机不明"
```

## 数据来源

story.yaml 的 14 个 plot（1_1 ~ 10_3）的 `clue` 和 `tension` 字段（已在之前的工作中添加）。

输入格式（来自 story.yaml）：

```yaml
plots:
  - id: 1_1
    clue:
      - "他在星标相簿里存了她十年的照片..."
      - "她偷偷标记了他朋友圈出现过的...地址..."
    tension:
      - "心理上彼此熟知十年，社交上却是知名陌生校友..."
```

## 实验步骤

```
步骤 1: 维度编码
  └─ 对 story.yaml 中的 14 个 plot，人工标注 K=5 维度值
  └─ 规则：基于 clue 判定 action_dominant 和 info_revealed；
         基于 tension 判定 social_distance、tension_type、core_tension
  └─ 输出：constraint_matrix.json（14×5 的矩阵）

步骤 2: 规则检测
  └─ 执行三种检测规则，输出异常区间
  └─ 与 p12 对 S1 和 S2 的人工诊断对比
  └─ 验证：检测到的异常区间与 p12 薄弱点的重合度

步骤 3: 解释性验证
  └─ 对每个检测到的异常：
      ├─ 约束矩阵提示的问题是什么？（如 "social_distance 从3跳回1"）
      └─ 这个问题的叙事含义是什么？（如 "上一场还在聊私事，下一场又变回礼貌社交"）
  └─ 人工评估：异常检测的结果是否有叙事意义

步骤 4: 扩展性验证
  └─ 同样构建校园言情（campus-romance）的约束矩阵
  └─ 验证：同一套维度能否描述不同作品的结构特征
  └─ 如果两部作品的约束矩阵有显著的维度差异（如都市言情 social_distance 变化更慢），
     说明矩阵能反映不同叙事策略的结构差异
```

## 验收标准

### 检测准确度

| 指标 | 合格线 | 良好线 | 优秀线 |
|------|--------|--------|--------|
| 异常区间与 p12 诊断的重合度 | ≥ 50% | ≥ 70% | ≥ 90% |
| 检测到而 p12 未标注的异常中有叙事意义 | ≥ 1 | ≥ 2 | ≥ 3 |
| 矩阵生成的可重复性（两次标注的 Cohen's κ） | ≥ 0.6 | ≥ 0.8 | ≥ 1.0 |

### 解释质量

| 指标 | 合格线 | 良好线 | 优秀线 |
|------|--------|--------|--------|
| 异常检测的叙事解释准确度（1-5） | > 2.5 | > 3.5 | > 4.0 |
| 矩阵提示的问题中创作者认为"有用"的比例 | ≥ 50% | ≥ 70% | ≥ 90% |

### 输出产物

1. `results/constraint_matrix.json`（14 plots × 5 维度的标注矩阵）
2. `results/anomalies.json`（规则检测到的异常区间列表 + 叙事解释）
3. `results/validation.json`（与 p12 诊断的对比验证）

## 不依赖

Python 脚本，无需 LLM API（矩阵标注为人工完成）。
依赖：
- story.yaml（plot 的 clue 和 tension 字段）
- p12 的 S1/S2 诊断数据（作为 ground truth）

## 结果预期

- **最强的检测信号**：social_distance 的跳跃。都市言情的关系演进是非常渐进的过程（每场戏推进 0.5~1 级），如果某场戏的 social_distance 显著偏离前后场景，很可能对应一个结构性问题。最可能被检测到的异常是 plot_1_2（深夜失眠）的 tension_type="retreat"——周围人都在主动靠近，这里突然退回内心世界。
- **最弱的检测维度**：action_dominant。人物的行为类型在都市言情中变化频繁（同一场景内可能切换多次），plot 级别的标注粒度可能太粗。
- **跨作品验证**：校园言情因时间跨度短（一周）、事件密度高，social_distance 的变化应比都市言情更快。如果矩阵能捕获这个差异，说明它描述的确实是叙事结构而非噪声。
- **失败条件**：如果两种检测规则（因果跳跃、节奏失衡）所有场景都通过了（0 个异常），说明粒度过粗——plot 级别的维度取值不足以捕捉场景内部的微结构问题。此时需要将标注粒度细化到 sub-plot 级别。
