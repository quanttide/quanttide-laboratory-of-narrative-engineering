# p15 — 情节结构的形式化与约束矩阵

合并自 p12（经验发现）与 p15（形式化尝试 + 约束矩阵）。

从一个经验问题开始：LLM 能否诊断情节结构的薄弱点？p12 给出肯定的经验结论。p15 试图将这些概念数学化，但发现 LLM 无法可靠地从叙事文本中提取因果图——这一负结论反过来论证了约束矩阵路线和 p14 的人机协作路径。

## 第一部分：经验发现（源自 p12）

### 假设

1. LLM 能识别情节的薄弱点——不是"写得不够好"，而是"这里缺了一个因果环节"或"这段节奏拖了"
2. 针对薄弱点的改法是结构性的（加一个场景、调整事件顺序、补伏笔），而非表面性的（"加强描??""丰富细节"）
3. 结构改法比表面改法更能提升情节连贯性

### 三类薄弱点定义

| 类型 | 定义 | 示例 |
|------|------|------|
| **因果跳跃** causal_gap | 动机→行动→后果的链断裂，读者会问"为什么ta突然这样" | "捶打→开会"过渡太陡，缺中间心理环节 |
| **节奏失衡** pacing | 事件密度不均——前半段节奏平缓，后半段高潮和约定连续发生 | 5 个情绪转折在 40 行内完成 |
| **动机不明** motivation_unclear | 人物行为缺乏内在逻辑锚点，读者只能推测 | "欣然接受送行"但理由不清晰 |

### 实验方法

以两个关键场景（S1 咖啡厅重逢、S2 酒吧表白）为测试素材：

1. **薄弱点诊断**：LLM 分析场景全文，标注薄弱点类型 + 严重程度
2. **结构改法生成**：对每个薄弱点生成结构改法（加场景、调顺序、补伏笔）
3. **对照**：同时生成表面改法（改措辞不改结构）作为对照组
4. **评估**：LLM pairwise 盲评 + 创作者人工评分

### 经验结论

**结构改法在 4/6 次上优于表面改法**。具体结论：

- 因果关系和动机不明类薄弱点最容易识别（有明确的逻辑链可追踪）
- 节奏失衡最难识别（需要全局感知）
- 结构改法在因果改善上显著优于表面改法
- 结构改法的可执行性略低（"加一个场景"比"加强描??"更难执行），但诊断的准确性更有价值

### 核心输出

- **诊断词汇表**：causal_gap / pacing / motivation_unclear 三类薄弱点定义
- **方法论**：结构改法 vs 表面改法的区分方法 + pairwise 评估流程
- **经验基线**：结构改法 4/6 优于表面改法
- **场景数据**：S1（咖啡厅重逢）和 S2（酒吧表白）的诊断结果实例

## 第二部分：形式化尝试与约束矩阵（p15）

### 初版：四个数学概念的迁移

p12 的三个文学概念（因果跳跃、节奏失衡、动机不明）直觉上可以映射为数学对象：

| p12 文学概念 | 数学映射 | 度量 |
|-------------|---------|------|
| 因果跳跃 | **因果图** —— 入度 = 0 的节点 | in-degree(v) = 0 判为断链 |
| 结构 vs 表面改法 | **图编辑距离** —— 节点/边编辑 > 0 vs 仅属性编辑 | GEDnode + GEDedge |
| 节奏失衡 | **事件密度函数** d(k) —— 窗口间方差 σ² | d(k) 低谷连续窗口判为拖沓 |
| 动机不明 | **贝叶斯惊奇度** S(E \| H) = -log P(E \| context) | S ≥ 阈值判为不明 |

### 初版运行结果：负结论

以 S1（咖啡厅重逢）和 S2（酒吧表白）的实验数据揭示了根本问题：

| 概念 | 数据表现 | 根因 |
|------|:--------:|------|
| 因果图 | 64 节点, 62 ?, out_degree=1 占 59/64 | LLM 提取的是**时序序列**，不是因果关系。真正的因果图应有分支和汇聚 |
| 图编辑距离 | 无法区分 | 链状图的 GED 只能检测节点插入/删除，无法判断"结构改法" |
| 事件密度 | σ² = 0.0015（极均匀） | 链状节点天然均匀分布，方差无意义 |
| 贝叶斯惊奇 | 全部 ≤ 0.05 | LLM 以"事件已发生"而非"基于约束的概率"判断——P(E) = 1.0 因为文本里发生了 |

**核心结论：LLM 当前无法可靠地从叙事文本中构建因果图。** 这不是数学框架的问题，而是输入源不可靠——LLM 提取的"图"本质上是一个按时序排列的线性链。

### 方向 C：叙事约束矩阵

放弃让 LLM 从原文提取结构，改为直接使用 story.yaml 中已有的结构化数据。

#### 维度定义

对每个 plot，在 K=5 个维度上取值：

| 维度 | 值域 | 来源 | 检测规则 |
|------|------|------|---------|
| D1 social_distance | 0(陌生人)~5(恋人) | clue 中的人物行为类型 | 相邻场景跳跃 ≥ 2 判为因果跳跃 |
| D2 tension_type | approach/retreat/mutual/avoid | tension 的情感方向 | 连续 3+ 场景同类型判为节奏失衡 |
| D3 action_dominant | internal/dialogue/physical/mixed | clue 的主导行为类型 | physical 行为 + D1 ≤ 1 判为动机不明 |
| D4 info_revealed | none/hint/partial/full | clue 的信息释放信号 | 3+ 场景 none 后突然 full |
| D5 core_tension | 0(无)~5(极强) | tension 的核心冲突强度 | 综合参考信号 |

#### 形式化与约束矩阵的互补关系

```
p12 经验发现                        p15 形式化
─────────────────────────────────────────────────
"这里因果链断了"                    social_distance 跳级 ≥ 2
"这段节奏太赶"                     tension_type 连续 3+ 场不变
"他为什么这样做"                   action_dominant=physical 且 social_distance ≤ 1
"LLM能识别50-70%薄弱点"            ᅳ 输入源不可靠，改为约束矩阵
"结构改法4/6优于表面改法"           图编辑距离框架保留为分析工具
```

p15 的形式化尝试没有取代 p12 的经验判断——它做了一件不同的事：**让"哪里不对"可以从矩阵中读出来，而不是只能感觉到**。

## 实验步骤

```
步骤 1: 薄弱点诊断（源自 p12）
  └─ 对场景全文，LLM 识别三类薄弱点
  └─ 输出 causal_gap / pacing / motivation_unclear

步骤 2: 结构改法 vs 表面改法（源自 p12）
  └─ 对每个薄弱点，同时生成结构改法和表面改法
  └─ pairwise 盲评对比

步骤 3: 约束矩阵构建（源自 p15 方向 C）
  └─ 人工标注 story.yaml 14 plots × 5 维度
  └─ 输出 constraint_matrix.json

步骤 4: 规则检测（源自 p15 方向 C）
  └─ 三种检测规则：因果跳跃、节奏失衡、动机不明
  └─ 输出 anomalies.json

步骤 5: 交叉验证
  └─ 约束矩阵检测结果与 p12 人工诊断对比
  └─ 验证：形式化检测能否发现与直觉诊断一致的问题
```

## 验收标准

### 诊断准确度（源自 p12）

| 指标 | 合格线 | 良好线 | 优秀线 |
|------|--------|--------|--------|
| LLM 标注的薄弱点与人工标注的重合 | ≥ 50% | ≥ 70% | ≥ 90% |
| 结构改法 > 表面改法 | 1/2 场景 | 2/2 | 2/2 + 显著 |

### 约束矩阵检测（源自 p15）

| 指标 | 合格线 | 良好线 | 优秀线 |
|------|--------|--------|--------|
| 异常区间与 p12 诊断的重合度 | ≥ 50% | ≥ 70% | ≥ 90% |
| 矩阵生成的可重复性（Cohen's κ） | ≥ 0.6 | ≥ 0.8 | ≥ 1.0 |

### 输出产物

1. `results/constraint_matrix.json`（14 plots × 5 维度）
2. `results/anomalies.json`（规则检测结果）
3. `results/validation.json`（与 p12 对比验证）

## 不依赖

Python 脚本。依赖：
- story.yaml（plot 的 clue 和 tension 字段）
- 场景原文（4_成稿 中的 .md 文件）

## 结论

> p12 发现 LLM 能从场景中诊断三类结构薄弱点（因果跳跃/节奏失衡/动机不明），结构改法在 4/6 次上优于表面改法。p15 试图将这些文学概念映射为可计算的图论/信息论度量（因果图、GED、密度函数、贝叶斯惊奇），但发现当前 LLM 无法可靠地从叙事文本中提取因果图——提取结果是线性链而非图，贝叶斯惊奇退化为"事件是否已发生"。这一负结论论证了 p14 的路线选择：人机协作标注行为链，LLM 负责诊断而非提取结构。转向约束矩阵后，基于 story.yaml 结构化数据的 5 维度矩阵成功检测到 1 个结构异常（表白后连续 5 场 tension_type 不变），说明结构化数据本身可以支撑一定程度的自动诊断。
