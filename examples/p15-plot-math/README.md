# p15 — 情节结构的数学形式化

验证 p12 的三种结构薄弱点（因果跳跃、节奏失衡、动机不明）能否映射为可计算的图论/信息论度量，使结构诊断与人工判断的一致性可被量化验证。

## 假设

1. **因果完整性**：场景的因果图（Causal Graph）中，每个非初始事件节点的入度 ≥ 1，入度为零的节点对应 p12 的"因果跳跃"
2. **结构改法 vs 表面改法**：原文图与改后图之间的图编辑距离（Graph Edit Distance）可以区分两者——节点/边编辑 > 0 为结构改法，仅属性编辑为表面改法
3. **节奏失衡**：事件密度函数 d(k) 的窗口间方差 σ² 超过阈值时，对应 p12 的"节奏失衡"
4. **动机不明**：事件节点的贝叶斯惊奇度 S(E | context) 越高，对应 p12 的"动机不明"越严重

## 背景

p12 用文学语言（"因果跳跃""节奏失衡""动机不明"）描述了情节结构的薄弱点，并通过人工盲评验证了结构改法优于表面改法（4/6 次）。但文学语言描述依赖人的直觉判断，无法复用、无法度量、无法自动化。

这 4 个概念虽然在 p12 中被验证为有效诊断工具，但它们的**判断依据停留在直觉层面**：一个 reviewer 觉得"这里缺了什么"，另一个 reviewer 可能不觉得。p15 的目标不是替代人工判断，而是**为这些判断建立可追溯的数学基础**——让"这里因果断了"不再是一个主观感受，而是一个可以在图上指出的缺失边。

## 数学框架

### 1. 因果图（Causal Graph）

定义场景的有向图 G = (V, E)，其中：

- V = 事件节点（含外部事件"下雨""音乐响起"和内部事件"想起""决定"）
- E ⊆ V × V，有向边 Eᵢⱼ 表示"事件 i 直接导致事件 j"
- 根节点：入度 = 0（场景初始状态或外部事件）
- 悬挂节点：出度 = 0（场景结束状态或未产生影响的事件）

**与 p12 的映射：**

| p12 文学概念 | 数学条件 |
|-------------|---------|
| 因果完整 | ∀ v ∈ V \ {roots}, in-degree(v) ≥ 1 |
| 因果跳跃 | 文本相邻事件对 (vᵢ, vᵢ₊₁) 之间无边 |
| 动机不明 | action(v) 仅有外部入边，无内部状态节点入边 |

### 2. 图编辑距离（Graph Edit Distance）

给定原文图 G₀ 和改后图 G₁，GED(G₀, G₁) 定义为将 G₀ 变为 G₁ 所需的最小编辑操作代价之和：

| 操作 | 代价 | 对应改法类型 |
|------|------|-------------|
| 节点插入/删除 | 1 | 结构改法（增/删事件） |
| 边插入/删除 | 1 | 结构改法（增/删因果关系） |
| 节点属性编辑 | ε (ε → 0) | 表面改法（改措辞不改结构） |

**判定规则：**
- GED(G₀, G₁) 来自节点/边操作 → 结构改法
- GED(G₀, G₁) 仅来自属性编辑 → 表面改法
- 结构改法占比 = (GEDnode > 0 或 GEDedge > 0) 的改法数 / 总改法数

### 3. 事件密度函数 d(k)

将场景文本分割为 K 个等长窗口（每 200 字），对窗口 k 统计因果图节点数 nₖ：

```
d(k) = nₖ / window_length
```

**与 p12 的映射：**

| 文学概念 | 数学条件 |
|-------------|---------|
| 节奏平稳 | σ²(d) ∈ [σ²_min, σ²_max] |
| 拖沓 | d(k) < θ 连续 M 个窗口 |
| 太赶 | d(k) 峰值 > τ，或滑动窗口中 σ²(d) → 0 |

### 4. 贝叶斯惊奇度（Bayesian Surprise）

事件 E 在给定其前因序列 H 下的惊奇度：

```
S(E | H) = -log P(E | character_state, antecedent_events)
```

**与 p12 的映射：**

| p12 诊断 | 条件 |
|----------|------|
| 动机明确 | S(E | H) 偏低 |
| 动机不明 | S(E | H) ≥ 峰度阈值（标记为 outlier） |

惊奇度高的节点需要的修复就是"补充因果边"——降低其条件概率的惊奇度。

## 实验步骤

```
步骤 1: 人工标注因果图（Ground Truth）
  └─ 对 S1（咖啡厅重逢）和 S2（酒吧表白），人工标注因果图
  └─ 标注规范：
      节点命名规则：[actor]_[action]，如 LY_dry_hair(LIN 擦头发)
      边标注规则：Eᵢⱼ 表示 "i 直接导致 j"（无中间节点）
  └─ 输出：G_gt_S1, G_gt_S2（邻接矩阵 + 节点属性表）

步骤 2: LLM 提取因果图
  └─ 对每个场景的文本，由 LLM 提取因果图
  └─ 输入：场景全文
  └─ 输出：G_llm_S1, G_llm_S2
  └─ 与 G_gt 对比：

  | 指标 | 计算方式 |
  |------|---------|
  | 节点召回率 | |V_gt ∩ V_llm| / |V_gt| |
  | 边精确率 | |E_gt ∩ E_llm| / |E_llm| |
  | 边召回率 | |E_gt ∩ E_llm| / |E_gt| |
  | 因果跳跃检测准确率 | 入度 = 0 的非根节点数，LLM vs 人工一致的比例 |

步骤 3: GED 计算与 p12 验证
  └─ 对 p12 中生成的改法（4 条结构改法 + 4 条表面改法），计算：
      ├─ GED(G₀, G_structural_fix) → 期望：节点/边编辑 > 0
      └─ GED(G₀, G_surface_fix) → 期望：仅属性编辑
  └─ 验证：GED 分类与人工分类的一致性

步骤 4: 事件密度函数计算
  └─ 对 S1 和 S2，设 K = 等分12窗口，计算 d(k) 序列
  └─ 标注：
      ├─ d(k) 峰值窗口 → 对应情感高潮段
      └─ d(k) 谷值连续窗口 → p12 标注的"拖沓"段
  └─ 输出：d(k) 曲线图 + 阈值交叉标注
  └─ 验证：d(k) 的局部极值与 p12 薄弱点定位的重合度

步骤 5: 贝叶斯惊奇度计算
  └─ 对 S1 和 S2 中每个事件节点，由 LLM 打分：
      "给定前文所有事件，当前事件发生的概率 P(E | H)"
  └─ 惊奇度 S = -log P
  └─ 标记 S ≥ 2σ 的节点为"动机不明"候选
  └─ 验证：惊奇度标记的节点与 p12 人工诊断的"动机不明"重合度
```

## 验收标准

### 因果图提取

| 指标 | 合格线 | 良好线 | 优秀线 |
|------|--------|--------|--------|
| 节点召回率 | ≥ 60% | ≥ 75% | ≥ 90% |
| 边召回率 | ≥ 50% | ≥ 70% | ≥ 85% |
| 因果跳跃检测准确率 | ≥ 60% | ≥ 80% | ≥ 90% |

### GED 分类

| 指标 | 合格线 | 良好线 | 优秀线 |
|------|--------|--------|--------|
| GED 分类与 p12 人工分类一致 | 6/8 (75%) | 7/8 (87.5%) | 8/8 (100%) |
| 结构改法 GEDnode+GEDedge > 0 | 3/4 (75%) | 4/4 (100%) | 4/4 (100%) |
| 表面改法 GEDnode=GEDedge=0 | 3/4 (75%) | 4/4 (100%) | 4/4 (100%) |

### 密度函数与惊奇度

| 指标 | 合格线 | 良好线 | 优秀线 |
|------|--------|--------|--------|
| d(k) 极值与 p12 薄弱点的重合 | ≥ 60% | ≥ 75% | ≥ 90% |
| 惊奇度标记的"动机不明"与人工诊断一致 | ≥ 60% | ≥ 75% | ≥ 90% |

### 输出产物

1. `results/graph_gt_S1.json` ~ `results/graph_gt_S2.json`（人工标注因果图）
2. `results/graph_llm_S1.json` ~ `results/graph_llm_S2.json`（LLM 提取因果图）
3. `results/ged_eval.json`（4 结构改法 + 4 表面改法的 GED 计算结果）
4. `results/density_S1.json` ~ `results/density_S2.json`（d(k) 逐窗口值）
5. `results/surprise_eval.json`（惊奇度打分与 p12 诊断对比）

## 不依赖

Python 脚本 + networkx（图编辑距离计算）+ LLM API。依赖：
- p12 的因果跳跃/节奏失衡/动机不明诊断数据（作为 ground truth）
- S1 和 S2 场景全文
- p12 生成的 4 条结构改法 + 4 条表面改法的前后文本

## 结果预期

- **因果图提取**：LLM 在节点识别上（"谁做了什么"）应表现良好（≥ 75%），但在细粒度因果关系（"A 的什么属性导致 B 的什么属性"）可能不足（≤ 60%）。因果跳跃检测准确率应是最高指标——入度 = 0 的节点很容易被 LLM 识别。
- **GED 分类**：预期 GED 分类一致性 ≥ 7/8——关键风险是"节点属性编辑"与"边删除+插入"的边界可能存在歧义（改措辞可能间接改变因果指向）。
- **密度函数**：d(k) 的峰值窗口应对应情感密度最高的段落（咖啡厅的"四目相对"段，酒吧的"原谅→愿意"段）。最有趣的结果可能是 d(k) 揭示的 p12 未标注的次要节奏问题。
- **贝叶斯惊奇**：如果惊奇度高的节点恰好是 p12 人工标注的"动机不明"节点，验证了 S(E|H) = -log P(E|H) 作为动机不明的可计算代理。如果两者不重合，说明"动机不明"可能不仅仅是概率问题——还涉及叙事契约（读者对特定类型的文本有特定的概率预期）。
- **最有价值的发现**：如果 4 个数学度量的联合诊断与 p12 人工诊断的重合度 > 80%，则 p15 提供了一种**将结构诊断自动化的路径**——不需要每次依赖人的直觉判断。
- **失败条件**：如果 LLM 提取的因果图与人工标注的边重合度 < 50%，说明当前 LLM 无法可靠地执行"从叙事文本中提取因果关系"这一基础操作——p15 的前提不成立。
