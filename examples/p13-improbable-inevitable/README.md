# p13 — 极小概率→必然实验

验证 LLM 能否帮助创作者把"碰巧发生的"重构成"不得不发生的"——情节设计的核心难点。

## 假设

1. 情节设计的核心难点是把"极小概率的巧合"重构为"人物性格驱动的必然"
2. LLM 能识别出情节中的"巧合点"，并给出"把巧合变成必然"的改写方案
3. 改写后的版本在"宿命感"和"因果逻辑"上优于原版

## 背景

期刊日志："AI 说是因为我在写极小概率的事情，所以 AI 很不擅长。在这个设定之下，怎么让这个极小概率变成一个必然要发生的事情，这个是写作的关键点，是难点。"

作者在实践中已经完成了一次"巧合→必然"的改写：咖啡厅重逢从"她偶然走进一家咖啡店避雨"改为"她收到闺蜜消息后刻意前往他常去的咖啡店"。这个改写让整个故事的逻辑链通了——男主泄露心事→共同好友传递→女主刻意前往→相遇。不再是雨天偶遇，而是一个人走向另一个人的必然。

p13 以这个已知的改写案例为 ground truth，测试 LLM 能否独立完成类似的"巧合→必然"重构。

## 数据来源

### 测试案例（2 个巧合驱动的场景，均有初稿→成稿的改写对照）

| # | 场景 | 初稿版（巧合驱动） | 成稿版（必然驱动） |
|---|------|------------------|------------------|
| C1 | 咖啡厅重逢 | 她雨天避雨走进一家咖啡店，恰好他在里面 | 她收到闺蜜消息后刻意前往他常去的咖啡店 |
| C2 | 酒吧表白 | 两人喝了酒，她回忆起大四聚餐，恰好他也记得那一天 | **待验证**：成稿版是否已经完成了"巧合→必然"的改写？ |

### 巧合点标注

对每个场景，人工标注"巧合点"：

**C1 咖啡厅重逢（初稿版）**：
- 巧合点 A：她刚好在那个雨天走到那条街
- 巧合点 B：她刚好选择那家咖啡店避雨
- 巧合点 C：他刚好那天也在那家咖啡店

**C2 酒吧表白（初稿版）**：
- 巧合点 A：她刚好记得大四聚餐的细节
- 巧合点 B：他刚好也记得同一天
- 巧合点 C：两人刚好在同一时刻卸下了防备

## 实验步骤

```
步骤 1: 巧合点识别
  └─ 对 C1 初稿和 C2，让 LLM 标注情节中的巧合点
  └─ 输出格式：[{"location": "她推门进入咖啡店",
      "coincidence_type": "spatial|temporal|informational",
      "why_improbable": "雨天随机选择一家咖啡店恰好是他在的那家"}]
  └─ 对比 LLM 标注与人工标注的巧合点重合度

步骤 2: 巧合→必然改写
  └─ 对每个巧合点，让 LLM 生成 3 个改写方案：
      约束：改写方案必须通过人物行为链条（而非外部事件）来消除巧合。
      不能简单加"原来她早就知道"的解释——必须展示人物做了什么导致了相遇。
  └─ 输出格式：{"rewrite": "具体改写方案",
      "mechanism": "通过什么人物行为链条消除巧合",
      "causal_chain": "A→B→C的因果链"}

步骤 3: Ground Truth 对比（C1）
  └─ 对比 LLM 的 3 个改写方案 vs 作者实际改写（成稿版）
  └─ 评估维度：
      ├─ 方向一致性：LLM 方案是否与作者方案在同一方向上（都是"增加女主主动性"）？
      ├─ 因果完整度：LLM 方案的人物行为链条是否完整闭合？
      └─ 自然度：改写后的相遇是否仍然自然，不显得刻意？

步骤 4: Ground Truth 对比（C2）
  └─ 同样对比 LLM 改写方案 vs 成稿版
  └─ C2 的"巧合→必然"改写可能不如 C1 明显——如果成稿版本身仍保留了一定的巧合性，
      这也是有价值的发现：某些场景的巧合可能不需要被完全消除

步骤 5: 宿命感评估
  └─ 对 C1 初稿、C1 成稿、C1 LLM改写最佳方案、C2 原文、C2 LLM改写最佳方案，做盲评
  └─ 评估标准：在 1-5 分上评价"这个相遇读起来是命中注定的还是碰巧的"
  └─ 验证：成稿和 LLM 改写版在宿命感上是否 > 初稿版
```

## 验收标准

### 巧合点识别

| 指标 | 合格线 | 良好线 | 优秀线 |
|------|--------|--------|--------|
| LLM 标注的巧合点与人工标注的重合 | ≥ 60% | ≥ 80% | 100% |

### C1 Ground Truth 对比

| 指标 | 合格线 | 良好线 | 优秀线 |
|------|--------|--------|--------|
| LLM 方案中 ≥ 1 个与作者改写方向一致 | 1/3 | 2/3 | 3/3 |
| LLM 最佳方案在因果完整度上 > 初稿 | 显著提升 | 提升 > 1 分 | 接近成稿 |

### C2 Ground Truth 对比

| 指标 | 合格线 | 良好线 | 优秀线 |
|------|--------|--------|--------|
| LLM 方案中 ≥ 1 个与作者改写方向一致 | 1/3 | 2/3 | 3/3 |
| LLM 最佳方案在因果完整度上 > 初稿 | 显著提升 | 提升 > 1 分 | 接近成稿 |

### 宿命感

| 指标 | 合格线 | 良好线 | 优秀线 |
|------|--------|--------|--------|
| 成稿宿命感 > 初稿宿命感 | 方向一致 | 显著 | 显著 |
| LLM 改写版宿命感 > 初稿宿命感 | 方向一致 | 显著 | 接近成稿 |

### 输出产物

1. `results/coincidence_detection_C1.json` ~ `results/coincidence_detection_C2.json`（巧合点标注）
2. `results/rewrites_C1.json` ~ `results/rewrites_C2.json`（每个巧合点 × 3 个改写方案）
3. `results/gt_comparison.json`（C1 LLM 方案 vs 作者成稿对比）
4. `results/human_eval_C2.json`（C2 创作者评估）
5. `results/destiny_eval.json`（宿命感盲评）

## 不依赖

纯 Python 脚本 + LLM API。依赖：
- 咖啡厅重逢初稿版（巧合驱动）
- 咖啡厅重逢成稿版（必然驱动，ground truth）
- 便利店谈心原文

## 结果预期

- **C1 最有价值**：如果 LLM 的改写方案与作者实际选择方向一致（增加女主主动性），说明"巧合→必然"是一个可被 LLM 学习的叙事操作，不是作者独有的直觉。
- **C2 最开放**：没有 ground truth，LLM 的改写方案完全独立创作。如果创作者对某个方案给出"会考虑采纳"，说明这个能力可以实用化——用于帮助作者发现"自己还没想到的必然性"。
- **改写方案的常见陷阱**：LLM 可能倾向于"加一个解释"（"其实她一直知道这家店"）而非"重构行为链条"（"她通过共同好友的社交网络追溯到了这家店的位置"）。前者是信息补丁，后者是情节重构——实验的核心是区分这两者。
- **宿命感**：如果 LLM 改写版的宿命感评分接近成稿，说明"让故事读起来是命中注定的"不只是作者的天赋，也是一种可被辅助的技艺。
- **失败条件**：如果 LLM 的所有改写方案都是信息补丁（"原来她早就知道"），而非行为链条重构——说明 LLM 在情节因果推理上只能做表面工作。
