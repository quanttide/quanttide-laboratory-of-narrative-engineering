# p04 — Review 意图理解可靠性实验

验证 Review（文本意图理解）的稳定性：同一篇文章被 LLM 多次 Review，输出是否一致。

## 为什么重要

按新的 3R 定义，Review 是**唯一不依赖前一轮结果的独立环节**（Reflect 依赖 Review，Rewrite 依赖 Reflect）。如果 Review 不稳定，整个链条的第一环就是沙土。

## 假设

1. 同一篇文章多次 Review，体裁、意图、阶段的判断应一致
2. 总结摘要的措辞可以不同，但核心判断不应矛盾
3. 不同文章的意图描述应有可量化的区分度

## 数据来源

从案例库取 3 篇不同体裁、不同质量的文章：

| # | 文章 | 类型 | 预期体裁 | 文件 |
|---|------|------|---------|------|
| A | 咖啡厅重逢 | 职场言情 draft | 慢热重逢场景 | `../../../assets/fiction/职场言情/3_初稿/1_1_咖啡厅重逢.md` |
| B | 便利店闲坐 | 职场言情 final | 日常对话场景 | `../../../assets/fiction/职场言情/4_成稿/4_1_便利店闲坐.md` |
| C | 第六章（论坛热搜） | 校园言情 draft | 事件驱动场景 | `../../../assets/fiction/校园言情/3_初稿/6_第六章.md` |

选 A 和 B 是因为同一作者、同一系列但不同质量阶段，可以验证 Review 能否区分 draft vs final。C 是不同作者，验证体裁区分度。

## 实验步骤

```
步骤 1: 对每篇文章，重复 Review 5 次
  └─ 每次使用完全相同的提示词，temperature=0.3
  └─ 提示词要求输出：
      {
        "genre": "场景体裁分类",
        "intent": "该段文本的创作意图",
        "stage": "所处的写作阶段描述",
        "summary": "一句话总结"
      }

步骤 2: 分析稳定性
  └─ 体裁一致性：5 次中体裁判断是否相同
  └─ 意图一致性：5 次中意图描述的核心是否矛盾
  └─ 摘要相似度：用 LLM 判断 5 条摘要是否指向同一结论

步骤 3: 区分度验证
  └─ 将 A/B/C 各 5 次的 Review 输出混合，去掉文章标签
  └─ 让 LLM 按体裁/意图相似度聚类
  └─ 衡量：同文章的 5 次输出是否自然聚为一类
```

## 验收标准

### 稳定性

| 指标 | 合格 | 良好 | 优秀 |
|------|------|------|------|
| 体裁一致性（3篇×5次） | 12/15 相同 | 14/15 相同 | 15/15 相同 |
| 意图无矛盾（3篇×5次） | 无直接矛盾 | — | — |

### 区分度

| 指标 | 合格 | 良好 | 优秀 |
|------|------|------|------|
| 同文章聚类率 | > 60% | > 80% | 100% |

## 输出产物

1. 15 次 Review 的原始输出 JSON
2. 稳定性分析表（每篇文章 × 每次的体裁/意图/阶段）
3. 聚类混淆矩阵
4. 结论：Review 阶段是否可靠
