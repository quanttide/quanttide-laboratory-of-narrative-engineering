# p01 — 叙事框架对比实验

验证"起承转合"是否适合做产品默认的叙事分析框架。

## 假设

不同叙事框架（起承转合、三幕结构、情感曲线）标注同一篇文章时，段落角色分配有可量化的差异。如果差异过大，说明框架选择显著影响分析结果，产品不能默认选一个而不验证。

## 数据来源

从案例库取 3 篇文章，覆盖不同叙事类型：

| # | 文章 | 类型 | 文件 |
|---|------|------|------|
| A | 咖啡厅重逢 | 慢热重逢，情绪积累型 | `assets/fiction/职场言情/3_初稿/1_1_咖啡厅重逢.md` |
| B | 酒吧表白 | 情感释放型 | `assets/fiction/职场言情/3_初稿/8_2_酒吧表白.md` |
| C | 第六章（论坛热搜） | 事件驱动、多角色型 | `assets/fiction/校园言情/3_初稿/6_第六章.md` |

选择理由：三种类型可能适合不同的框架。A 是情绪暗涌，B 是高潮释放，C 是外部事件推进。

## 框架定义

### 框架一：起承转合

| 角色 | 定义 | 检测锚点 |
|------|------|---------|
| 起 | 引入人物、场景、情绪基调 | 段落开头，环境描写，人物出场 |
| 承 | 展开关系，铺垫矛盾 | 对话展开，细节积累，情绪递进 |
| 转 | 关键变化，情感转折 | 意外事件，告白，决定，冲突爆发 |
| 合 | 收束，新的状态 | 结局，反思，情绪落定，新常态 |

### 框架二：三幕结构

| 角色 | 定义 | 检测锚点 |
|------|------|---------|
| Setup | 建立人物、情境、冲突种子 | 引入角色，展示日常，隐含矛盾 |
| Confrontation | 冲突升级，障碍出现 | 矛盾表面化，角色被迫行动，张力上升 |
| Resolution | 高潮到结局 | 关键对决/告白，冲突解决，新平衡 |

### 框架三：情感曲线（5 段）

| 角色 | 定义 | 检测锚点 |
|------|------|---------|
| 常态 | 情绪基线 | 日常描写，平静状态 |
| 触发 | 情绪偏离的起点 | 意外相遇，消息到来，回忆触发 |
| 爬升 | 情绪积累 | 内心独白，细节堆叠，张力渐强 |
| 峰值 | 情绪释放 | 告白，哭泣，拥抱，冲突爆发 |
| 回落 | 情绪余韵 | 安静对话，身体放松，场景收束 |

## 实验步骤

```
步骤 1: 准备文章样本
  └─ 从案例库复制 3 篇文章的正文（去除元数据）
  └─ 每篇按自然段分割，编号

步骤 2: 对每个框架编写分析提示词
  └─ 提示词结构：框架定义 + 角色说明 + 段落列表 + 输出格式
  └─ 3 个框架 × 3 篇文章 = 9 次 LLM 调用
  └─ 使用 DeepSeek（与产品一致的模型），temperature=0.3

步骤 3: 收集标注结果
  └─ 输出格式：每段标注 [{段落号, 原文缩略, 角色, 理由}]
  └─ 每篇文章 × 每个框架 = 一个 JSON 文件

步骤 4: 对比分析
  └─ 同篇文章、不同框架的段落角色分布对比
  └─ 差异分类：
      ├─ 粒度差异（框架 A 标 1 段，框架 B 标 2 段）
      ├─ 角色定义差异（同一段被标为"起" vs "Setup"）
      └─ 框架不适配（某框架无法对某类文章给出合理标注）
```

## 验收标准

1. 9 次标注结果以 JSON 格式输出到 `results/` 目录
2. 每篇文章输出一张框架对比表（段落号 × 3 个框架的角色分配）
3. 输出一份差异分析报告：分类统计差异数量 + 典型案例

## 不依赖

不依赖任何产品代码。只需 Python + LLM API + 案例库文本。

## 结果预期

如果起承转合与三幕结构在 3 篇文章上标注结果高度重合（>80%），说明框架选择对产品影响不大，默认用起承转合没问题。如果差异显著（<50%），则需要产品根据文章类型选择框架，或重新设计分析prompt。
