# p14 — 片段内部情节建议

验证给定一个完整场景（成稿片段），LLM 能否诊断其内部行为链和情感递进的薄弱点，并生成可采纳的局部修改建议。

## 动机

p12 验证了 LLM 能诊断场景间的结构性薄弱点（因果跳跃、节奏失衡、动机不明），p13 验证了 LLM 能把外部巧合改写为人物驱动的必然。但这两个实验的粒度都偏大——p12 关心的是场景与场景之间的结构，p13 关心的是"巧合→必然"单一线索。

实际创作中，作者更常遇到的问题是"这场戏内部哪里不对劲"：一个动作缺了前因、一句对话时机不对、一段情绪的转折太陡。p14 专注在这个粒度——**场景内部的微因果链**。

## 数据来源

### 测试场景（2 个场景）

| # | 场景 | 选材理由 |
|---|------|---------|
| S1 | 1_1 咖啡厅重逢 | 高信息密度场景（6 页），包含日记、闺蜜消息、毛巾、点单等多段行为链。同时是"社交陌生人+心理熟人"核心约束最极致的一个场景——每个行为都在越过真实身份的边界。 |
| S2 | 8_2 酒吧表白 | 情感高潮场景，对话承载了整部作品的十年重量。情感曲线陡峭，对话密度大，音乐插入时机敏感。 |

### 输入

- 场景全文（4_成稿 中的完整文本）
- story.yaml 中的 scene description（人物状态 + 关系阶段）
- motif.yaml（可选，作为场景的情感/母题上下文）

## 实验步骤

```
步骤 1: 行为链标注
  └─ 对每个场景，将全文分解为"行为-反应"序列
  └─ 每个序列节点标注双层动机：

      ├─ actor: 谁在行动
      ├─ action: 做了什么/说了什么
      ├─ outward_motivation: 出行动机（露给别人看的理由）
      ├─ inward_motivation: 内隐动机（真实内心，可能未说出口）
      ├─ tension: 两个动机之间是否有冲突/张力
      ├─ reaction: 对方的反应（是否有因果连接？）
      └─ chain_status: 完整 | 有意断裂 | 无意断裂

      三值链状态含义：
        ├─ 完整：动机→行动的因果链在文本内有支撑
        ├─ 有意断裂：链断裂，但这是文本的设计意图（张力所在，不需修补）
        └─ 无意断裂：链断裂，且文本未提供合理支撑（真薄弱点）

  └─ 输出格式：[{"seq": 1, "actor": "林远亭", "action": "掏出毛巾擦她头发",
      "outward_motivation": "陌生人之间不该这么做（社交越界）",
      "inward_motivation": "本能心疼她，不想让她着凉",
      "tension": "冲突——他的手比脑子快，说明内心已经默认'可以接近她'",
      "reaction": "她怔住，忘记拒绝也忘记邀请",
      "chain_status": "有意断裂"}]

步骤 2: 薄弱点诊断
  └─ 基于行为链标注，识别因果链断裂处
  └─ 4 种薄弱点类型（从 p12 的 3 种扩展为 4 种）：
      ├─ causal_gap: 行为 A→行为 B 缺少逻辑过渡
      ├─ motivation_blind: 人物做了某事，文本未提供动机支撑
      ├─ emotional_jump: 情感状态变化跳过中间态（如从紧张→放松无过渡）
      └─ info_density: 同一时刻释放的信息量过多或过少
  └─ 输出格式：[{"location": "他直接点了焦糖玛奇朵（未问她口味）",
      "issue_type": "motivation_blind", "severity": 1-3,
      "reasoning": "他作为社交陌生人不应该知道她爱喝什么",
      "before": "当前写法...", "after": "改写方案..."}]

步骤 3: 建议生成
  └─ 对每个薄弱点，生成 3 类修改建议：
  └─

  建议类型 A：因果补链（基于 p12 的结构改法思路）
    └─ 不改变行为本身，补一个中间环节（动作/念头/外部事件）
      让行为的因果链完整闭合
    └─ 适用：chain_status = "无意断裂"（因果链缺失）
    └─ 示例："在他直接点单前加一句——他犹豫了一下，
      还是选了焦糖玛奇朵，因为他记得十年前她说过这个"

  建议类型 B：动机刻深（基于 p13 的必然化改写思路）
    └─ 改写行为本身，让动机来源从"恰好"变为"性格驱动"
    └─ 适用：动作依赖巧合或外部事件，而非人物性格
    └─ 示例："改成她提出'你们这什么好喝'，他说'焦糖玛奇朵不错'——
      用店主推荐代替个人记忆，保持社交陌生人的身份"

  建议类型 C：边界校准（新增）
    └─ 保留行为链结构，调整该行为的"社交距离"
    └─ 适用：行为链完整，但 outward/inward 动机冲突导致行为在
      陌生人/熟人边界上偏移——亲密度过高或过低
    └─ 操作方向：
        ├─ 调远（降低亲密度）：改成更适合陌生人的版本
        └─ 调近（升高亲密度）：如果 inward_motivation 需要更清晰
    └─ 示例（调远）：
      "改成他问店主'你们这什么好喝'，店主推荐焦糖玛奇朵，
      他说'那来一杯吧'。她喝了一口说'好喝'——
      他不知道她爱喝这个，是命运的安排让他们选了同一杯"

  └─ 输出格式：{"location": "...", "issue_type": "...",
      "chain_status": "有意断裂|无意断裂|完整",
      "motivation_conflict": "outward/inward 是否冲突",
      "fix_A": {"category": "因果补链", "description": "...",
        "expected_effect": "..."},
      "fix_B": {"category": "动机刻深", "description": "...",
        "expected_effect": "..."},
      "fix_C": {"category": "边界校准", "direction": "调远|调近",
        "description": "...", "expected_effect": "..."}}

步骤 4: 创作者评估
  └─ 作者对每条建议做 1-5 评分
  └─ 评估维度：
      ├─ 诊断准确度：这个薄弱点真实存在吗？
      ├─ 建议自然度：改完后像不像这个人物会做的事？
      └─ 实用价值：我会采纳这个建议吗？
```

## 验收标准

### 诊断质量

| 指标 | 合格线 | 良好线 | 优秀线 |
|------|--------|--------|--------|
| 行为链节点标注与人工标注重合 | ≥ 70% | ≥ 85% | 100% |
| 薄弱点中作者认为"真实存在"的比例 | ≥ 60% | ≥ 75% | ≥ 90% |
| LLM 发现的薄弱点中作者未意识到的比例 | ≥ 1/场景 | ≥ 2/场景 | ≥ 3/场景 |

### 建议质量

| 指标 | 合格线 | 良好线 | 优秀线 |
|------|--------|--------|--------|
| "因果补链"建议采纳率（作者评分 ≥ 3） | ≥ 50% | ≥ 70% | ≥ 90% |
| "动机刻深"建议采纳率（作者评分 ≥ 3） | ≥ 50% | ≥ 70% | ≥ 90% |
| "边界校准"建议采纳率（作者评分 ≥ 3） | ≥ 50% | ≥ 70% | ≥ 90% |
| 作者评分 > 4 的建议数 | ≥ 1/场景 | ≥ 2/场景 | ≥ 3/场景 |

### 链状态区分

| 指标 | 合格线 | 良好线 | 优秀线 |
|------|--------|--------|--------|
| "有意断裂"与"无意断裂"的区分准确率 | ≥ 60% | ≥ 80% | ≥ 90% |
| 双重动机冲突的标注与人工判断一致 | ≥ 60% | ≥ 75% | ≥ 90% |

### 输出产物

1. `results/chain_S1.json` ~ `results/chain_S2.json`（行为链标注）
2. `results/diagnosis_S1.json` ~ `results/diagnosis_S2.json`（薄弱点诊断 + 修改建议）
3. `results/eval.json`（创作者评估）

## 不依赖

纯 Python 脚本 + LLM API。依赖：
- story.yaml（场景描述）
- 场景原文（4_成稿 中的 .md 文件）

## 结果预期

- **先用 S1 跑通全流程后再扩展到 S2**。S2 的行为链由对话主导（"说""沉默""回忆"），而标注格式主要为非对话行为设计。p13 中 S2 已暴露数据质量问题（LLM 生成了与都市言情无关的改写方案）。建议先以 S1 验证标注格式和边界校准类建议的有效性。
- **S1（咖啡厅重逢）** 最容易产生有价值的诊断。"社交陌生人+心理熟人"张力贯穿始终，LLM 如果能识别"焦糖玛奇朵""毛巾缩回""直接点单"中的双层动机冲突，说明它理解了核心约束。标注格式的三值链状态在这里能直接验证： "缩回"应判为"有意断裂"，"点单"应判为"无意断裂"。
- **S2（酒吧表白）** 最难且最不确定。对话主导的行为链标注稳定性存疑。如果 S1 通过后再扩展，S2 的真正薄弱点可能在"音乐插入时机""原谅→愿意之间的心理转折"等情感节奏层面，而非行为因果链层面。
- **失败条件**：如果 LLM 标注的双层动机与人工判断重合 < 50%，说明 LLM 无法区分"人物表面上的理由"和"人物真实但未说出的理由"——这个能力是理解"社交陌生人+心理熟人"的前提，如果失败，p14 的设计假设不成立。
