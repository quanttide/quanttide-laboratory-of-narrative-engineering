## p11 — 情节一致性检测

### 假设

1. story.yaml 的 14 章场景描述构成可验证的情节基线
2. LLM 能检测跨场景的人物行为矛盾（如林远亭在某个场景说"我哭不出来了"但在另一个场景哭了出来——这不是矛盾，这是情节弧线）
3. LLM 能区分"真正的情节矛盾"和"有意的人物弧线"

### 实验设计

**数据**：story.yaml 14 章 + 原文，手动植入 3 处人工矛盾（如篡改某个细节让两个场景的时间线冲突）。

**步骤**：LLM 逐对比较相邻场景 → 标注 inconsistencies → 人工验证哪些是真正的矛盾、哪些是有意弧线。

### 验收

| 指标 | 合格线 |
|------|--------|
| 人工植入的 3 处矛盾全被检测 | 3/3 |
| 误报率（将有意的弧线标记为矛盾） | < 30% |

## p12 — 情节-母题联动

### 假设

1. 小说的每个情节转折点都对应一个或多个母题的出现
2. LLM 能识别这种对应关系——如"递毛巾"同时是情节转折（两人第一次身体接触）和手势母题的体现

### 实验设计

**数据**：story.yaml 的 14 章 + motif.yaml 的 5 母题。人工标注每个场景使用了哪些母题（ground truth 来自 p05 的提取结果）。

**步骤**：LLM 判断每个场景的情节功能（起/承/转/合）→ LLM 提取每个场景的母题 → 计算情节功能与母题分布的关联强度。

### 验收

| 指标 | 合格线 |
|------|--------|
| "转"和"合"场景的母题密度 > "承"场景 | 显著差异 |
| "手势"母题出现在关键身体接触场景的频率 > 随机分布 | p < 0.1 |

## p13 — 风格偏离检测

### 假设

1. 同一系列内，成稿的风格评分应高于刻意偏离风格的改写版
2. LLM 能定量区分"符合风格的原文"和"故意写偏的版本"
3. 偏离量最大的维度应与改写的目标方向一致

### 实验设计

**数据**：都市言情成稿 3 篇（深夜失眠、书房陪伴、海边散步）。对每篇生成 2 个"偏离版"——人工或 LLM 协助改写，刻意破坏某个风格维度（如把含蓄克制的情感表达改成直白热烈、把静态室内场景改成激烈外部冲突）。

**步骤**：对原文和偏离版分别跑 review → 对比同篇文章的原文 vs 偏离版评分差异 → 验证评分下降的维度是否与改写方向一致。

### 验收

| 指标 | 合格线 |
|------|--------|
| 偏离版评分 < 原文评分 | 至少 2/3 篇 |
| 评分下降最大的维度与改写方向一致 | 至少 2/3 篇 |
| 偏离版在风格上的降幅 > 随机波动 | 显著差异 |

## p13 — 契约层生成验证

### 假设

1. style+motif+story 三份 YAML 构成完整生成约束
2. 在契约约束下生成的场景比无约束场景在所有三层指标上都更好
3. 缺失任何一层约束都会导致对应指标的下降

### 实验设计

**数据**：都市言情 style+motif+story。场景模板：从 story.yaml 中选取 3 个已有场景（跳过，让 LLM 重新生成同一场景的新版本）。

**步骤**：四组生成（全约束 / 缺 style / 缺 motif / 无约束）→ 分别评审风格一致性 + 母题覆盖 + 情节连贯性。

### 验收

| 指标 | 合格线 |
|------|--------|
| 全约束组在三项指标上均 > 无约束组 | 方向一致 |
| 缺 style 组风格分 < 全约束组 | 下降 > 20% |
| 缺 motif 组母题覆盖 < 全约束组 | 下降 > 20% |
