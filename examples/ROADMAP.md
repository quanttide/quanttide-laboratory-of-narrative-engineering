## p11 — 情节方向生成

### 假设

1. 给定当前场景的人物状态（关系阶段、情绪状态、冲突状态），LLM 能生成多个合理的情节走向
2. 不同走向在母题使用、情感强度、叙事节奏上有可量化的差异
3. 创作者可以在这些走向中选择，而非从零构思

### 为什么这很重要

期刊日志："两个人的互动没有蓝本可以参照，也非常难写。现在写出来的有点太熟了，太陌生的也不好写。" 情节方向生成不是替代创作者做决定，而是给创作者提供"这里还可以怎么走"的选项。

### 实验设计

**数据**：story.yaml 中选取 3 个关键场景（咖啡厅重逢、便利店谈心、酒吧表白前一刻）。每个场景提供人物当前状态描述。

**步骤**：对每个场景，让 LLM 生成 5 个不同的情节走向 → 评估走向的多样性（是否真的不同）、合理性（是否符合人物性格）、母题契合度（是否延续系列母题）。

### 验收

| 指标 | 合格线 |
|------|--------|
| 5 个走向中有 ≥ 3 个被人工评定为"合理且有差异" | 3/5 |
| 走向之间的母题使用模式有差异（不是同一个模板换词） | Jaccard < 0.6 |

## p12 — 情节结构优化

### 假设

1. LLM 能识别情节的薄弱点——不是"写得不够好"，而是"这里缺了一个因果环节"或"这段节奏拖了"
2. 针对薄弱点的改法是结构性的（加一个场景、调整事件顺序），而非表面性的（"加强描写"）
3. 结构改法比表面改法更能提升情节连贯性

### 为什么这很重要

期刊日志："我需要一个更合适的创作情境。怎么才能够让这个设定变得更加……这种比较细致的方式去刻画它，因为能够感觉到这个文本已经复杂度已经开始超出我的直觉感知了。"

当情节复杂度超出直觉时，创作者需要的不是"写得好一点"的建议，而是"这里因果关系断了——考虑在前一章加一个伏笔"的结构诊断。

### 实验设计

**数据**：选取 story.yaml 中情节最复杂的 3 个场景（书房陪伴、海边散步、酒吧表白）。人工标注每个场景的情节薄弱点。

**步骤**：LLM 分析场景的情节结构 → 标注薄弱点 + 给出结构改法 → 对比结构改法 vs 表面改法（来自 p08 inspire）在情节连贯性上的提升。

### 验收

| 指标 | 合格线 |
|------|--------|
| LLM 标注的薄弱点与人工标注的重合 | ≥ 50% |
| 结构改法的情节连贯性评分 > 表面改法 | 显著差异 |
| 结构改法包含"加场景""调顺序""补伏笔"等操作（非"改措辞"） | ≥ 60% 的改法 |

## p13 — 极小概率→必然

### 假设

1. 情节设计的核心难点是把"极小概率的巧合"重构为"人物性格驱动的必然"
2. LLM 能识别出情节中的"巧合点"，并给出"把巧合变成必然"的改写方案
3. 改写后的版本在"宿命感"和"因果逻辑"上优于原版

### 为什么这很重要

期刊日志："AI 说是因为我在写极小概率的事情，所以 AI 很不擅长。在这个设定之下，怎么让这个极小概率变成一个必然要发生的事情，这个是写作的关键点，是难点。"

这是情节设计的元问题——不是"这个情节好不好"，而是"这个事情为什么会发生"。咖啡厅重逢不是雨天偶遇，是女主收到消息后刻意前往。这个改写本身就是"巧合→必然"的实践。

### 实验设计

**数据**：选取 2 个"巧合驱动"的场景描述（cafe 重逢的初稿版 vs 成稿版，作为 ground truth——成稿已经完成了巧合→必然的改写）。

**步骤**：
1. LLM 识别初稿中的巧合点
2. LLM 给出"把巧合变成必然"的改写方案
3. 人工对比 LLM 方案 vs 作者实际做的改写（成稿版）
4. 评估 LLM 方案在因果逻辑和宿命感两个维度上的质量

### 验收

| 指标 | 合格线 |
|------|--------|
| LLM 方案中至少 1 个与作者实际改写方向一致 | 1/2 |
| LLM 方案在因果逻辑上 > 初稿 | 显著提升 |
| LLM 方案不是简单加一个"原来她早就在等"的解释，而是通过人物行为链条重构巧合 | 至少 1 篇 |
