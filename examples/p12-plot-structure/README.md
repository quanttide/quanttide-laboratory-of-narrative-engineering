# p12 — 情节结构优化实验

验证 LLM 能否识别情节的薄弱点（因果断裂、节奏失衡），并给出结构性改法而非表面修补。

## 假设

1. LLM 能识别情节的薄弱点——不是"写得不够好"，而是"这里缺了一个因果环节"或"这段节奏拖了"
2. 针对薄弱点的改法是结构性的（加一个场景、调整事件顺序、补伏笔），而非表面性的（"加强描写""丰富细节"）
3. 结构改法比表面改法更能提升情节连贯性

## 背景

期刊日志："我需要一个更合适的创作情境。怎么才能够让这个设定变得更加……这种比较细致的方式去刻画它，因为能够感觉到这个文本已经复杂度已经开始超出我的直觉感知了。"

当情节复杂度超出直觉时，创作者需要的不是"写得好一点"的建议，而是"这里因果关系断了——考虑在前一章加一个伏笔"的结构诊断。p12 测试 LLM 能否提供这种诊断。

## 数据来源

### 测试场景（2 个情节结构最需要诊断的场景）

| # | 场景 | 情节特征 | 已知薄弱点（人工标注） |
|---|------|---------|------|
| S1 | 咖啡厅重逢 | 相遇场景。两线并行（他的视角/她的视角）→ 交汇于门口。时间结构复杂（当下+回忆+日记）。 | **薄弱点**：他的日记段和她收到消息段的篇幅比例是否合理？她的"蓄谋"动机在场景前半段是否铺垫充分？ |
| S2 | 酒吧表白 | 高潮场景。醉酒→回忆→音乐插入→"原谅"→"你喜不喜欢我"。多个情绪层次交替。 | **薄弱点**：音乐《原谅》的插入是否打断了表白的情感累积？"原谅自己"的哲思与前后对话的因果关系是否足够清晰？ |

### 对照组改法

| 组 | 策略 | 示例（S1 "捶打→开会"过渡太陡） |
|----|------|------------------------------|
| **结构改法**（实验组） | 调整叙事结构 | "在捶打和开会之间加入一段——他把她放在腿上之前，她看着屏幕上的共享文档愣了一下，想起自己第一次看到他的代码也是这种感觉" |
| **表面改法**（对照组） | 改措辞不改结构 | "丰富捶打和开会之间的过渡描写，加入更多的心理活动" |

## 实验步骤

```
步骤 1: 情节薄弱点诊断
  └─ 对每个测试场景，提供场景全文 + story.yaml 中的上下文（前后场景关系）
  └─ LLM 分析情节结构，标注薄弱点：
      输出格式：[{"location": "捶打→开会过渡段", "issue_type": "因果跳跃|causal_gap
      |节奏失衡|pacing|动机不明|motivation_unclear|伏笔断裂|foreshadow_break",
      "severity": 1-3, "reasoning": "为什么这是个薄弱点"}]

步骤 2: 结构改法生成
  └─ 对每个薄弱点，生成结构改法（实验组）：
      约束：改法必须是结构性的——加场景、调顺序、补伏笔、插入过渡段落，
      不能是"加强描写""丰富细节"等措辞层面建议
  └─ 输出格式：{"fix_type": "add_scene|reorder|add_foreshadow|insert_bridge",
      "description": "具体怎么做", "expected_effect": "这样做会如何改善情节连贯性"}

步骤 3: 对比评估
  └─ 对每个薄弱点，生成结构改法 + 表面改法（对照组）各一条
  └─ LLM pairwise 盲评（p10 验证的方法）：哪条改法更能提升情节连贯性？
  └─ 评估维度：
      ├─ 因果改善：改法是否填补了因果链的缺口？
      ├─ 节奏改善：改法是否让叙事节奏更合理？
      └─ 可执行性：改法是否给出了具体的操作步骤？

步骤 4: 人工验证
  └─ 创作者对结构改法做 1-5 评分：
      ├─ 这个诊断对吗？（薄弱点定位准确度）
      └─ 这个改法我会用吗？（实用价值）
```

## 验收标准

### 诊断准确度

| 指标 | 合格线 | 良好线 | 优秀线 |
|------|--------|--------|--------|
| LLM 标注的薄弱点与人工标注的重合 | ≥ 50% | ≥ 70% | ≥ 90% |
| LLM 的 issue_type 分类与人工判断一致 | ≥ 60% | ≥ 80% | ≥ 90% |

### 改法质量

| 指标 | 合格线 | 良好线 | 优秀线 |
|------|--------|--------|--------|
| 结构改法 > 表面改法（pairwise 因果改善） | 1/2 场景 | 2/2 | 2/2 + 显著 |
| 结构改法 > 表面改法（pairwise 节奏改善） | 1/2 场景 | 2/2 | 2/2 + 显著 |
| 结构改法中"加场景/调顺序/补伏笔"占比 | ≥ 50% | ≥ 70% | ≥ 90% |

### 人工验证

| 指标 | 合格线 | 良好线 | 优秀线 |
|------|--------|--------|--------|
| 薄弱点定位准确度（创作者评分 1-5） | > 2.5 | > 3.5 | > 4.0 |
| 改法实用价值（创作者评分 1-5） | > 2.5 | > 3.5 | > 4.0 |

### 输出产物

1. `results/diagnosis_S1.json` ~ `results/diagnosis_S2.json`（薄弱点诊断）
2. `results/structural_fixes_S1.json` ~ `results/structural_fixes_S2.json`（结构改法）
3. `results/surface_fixes_S1.json` ~ `results/surface_fixes_S2.json`（表面改法对照）
4. `results/pairwise_eval.json`（结构 vs 表面 pairwise 对比）

## 不依赖

纯 Python 脚本 + LLM API。依赖：
- story.yaml（场景描述 + 上下文）
- 场景原文（书房陪伴、海边散步、酒吧表白）
- p10 的 pairwise 盲评方法（改法质量对比）

## 结果预期

- **最可能**：LLM 能识别 50-70% 的薄弱点。因果关系和动机不明的薄弱点最容易识别（有明确的逻辑链可追踪），节奏失衡最难识别（需要全局感知）。
- **结构改法 vs 表面改法**：结构改法在因果改善上应显著优于表面改法，但在可执行性上可能低于表面改法（"加一个场景"比"加强描写"更难执行）。这是合理的——结构改法的价值在于诊断的准确性，而非执行的便捷性。
- **最有价值的发现**：如果 LLM 发现了一个创作者未意识到的薄弱点，这是 p12 的核心价值——情节复杂度超出直觉时，外部诊断可以弥补盲区。
- **S1（书房陪伴）**：最敏感的测试。7 个情绪转折在 40 行内完成，如果 LLM 不标注任何薄弱点，说明它对"高密度情感叙事"的容忍度太高。
- **失败条件**：如果结构改法的 pairwise 结果 ≈ 表面改法，说明 LLM 无法生成真正结构性的改法——它只是在"加场景"的标签下写了一段"加强描写"的建议。
