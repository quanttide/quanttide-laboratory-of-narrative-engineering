# p14 — 片段内部情节建议

验证给定一个完整场景（成稿片段），LLM 能否诊断其内部行为链和情感递进的薄弱点，并生成可采纳的局部修改建议。

## 动机

p12 验证了 LLM 能诊断场景间的结构性薄弱点（因果跳跃、节奏失衡、动机不明），p13 验证了 LLM 能把外部巧合改写为人物驱动的必然。但这两个实验的粒度都偏大——p12 关心的是场景与场景之间的结构，p13 关心的是"巧合→必然"单一线索。

实际创作中，作者更常遇到的问题是"这场戏内部哪里不对劲"：一个动作缺了前因、一句对话时机不对、一段情绪的转折太陡。p14 专注在这个粒度——**场景内部的微因果链**。

## 数据来源

### 测试场景（2 个场景）

| # | 场景 | 选材理由 |
|---|------|---------|
| S1 | 1_1 咖啡厅重逢 | 高信息密度场景（6 页），包含日记、闺蜜消息、毛巾、点单等多段行为链。同时是"社交陌生人+心理熟人"核心约束最极致的一个场景——每个行为都在越过真实身份的边界。 |
| S2 | 8_2 酒吧表白 | 情感高潮场景，对话承载了整部作品的十年重量。情感曲线陡峭，对话密度大，音乐插入时机敏感。 |

### 输入

- 场景全文（4_成稿 中的完整文本）
- story.yaml 中的 scene description（人物状态 + 关系阶段）
- motif.yaml（可选，作为场景的情感/母题上下文）

## 实验步骤

```
步骤 1: 行为链标注
  └─ 对每个场景，将全文分解为"行为-反应"序列
  └─ 每个序列节点标注：
      ├─ actor: 谁在行动
      ├─ action: 做了什么/说了什么
      ├─ motivation: 行为动机（文本内是否有支撑？）
      └─ reaction: 对方的反应（是否有因果连接？）
  └─ 输出格式：[{"seq": 1, "actor": "林远亭", "action": "掏出毛巾擦头发",
      "motivation": "本能（不想让她着凉）", "motivation_evidence": "文本内有'手比脑子更快'",
      "reaction": "她怔住→忘记拒绝也忘记邀请",
      "reaction_evidence": "文本内有", "chain_status": "完整"}]

步骤 2: 薄弱点诊断
  └─ 基于行为链标注，识别因果链断裂处
  └─ 4 种薄弱点类型（从 p12 的 3 种扩展为 4 种）：
      ├─ causal_gap: 行为 A→行为 B 缺少逻辑过渡
      ├─ motivation_blind: 人物做了某事，文本未提供动机支撑
      ├─ emotional_jump: 情感状态变化跳过中间态（如从紧张→放松无过渡）
      └─ info_density: 同一时刻释放的信息量过多或过少
  └─ 输出格式：[{"location": "他直接点了焦糖玛奇朵（未问她口味）",
      "issue_type": "motivation_blind", "severity": 1-3,
      "reasoning": "他作为社交陌生人不应该知道她爱喝什么",
      "before": "当前写法...", "after": "改写方案..."}]

步骤 3: 建议生成
  └─ 对每个薄弱点，生成 2 类修改建议：
  └─

  建议类型 A：因果补链（基于 p12 的结构改法思路）
    └─ 不改变行为本身，补一个中间环节（动作/念头/外部事件）
      让行为的因果链完整闭合
    └─ 示例："在他直接点单前加一句——他犹豫了一下，
      还是选了焦糖玛奇朵，因为他记得十年前她说过这个"

  建议类型 B：动机刻深（基于 p13 的必然化改写思路）
    └─ 改写行为本身，让动机来源从"恰好"变为"性格驱动"
    └─ 示例："改成她提出'你们这什么好喝'，他说'焦糖玛奇朵不错'——
      用店主推荐代替个人记忆，保持社交陌生人的身份"

  └─ 输出格式：{"location": "...", "issue_type": "...",
      "fix_A": {"category": "因果补链", "description": "...",
        "expected_effect": "..."},
      "fix_B": {"category": "动机刻深", "description": "...",
        "expected_effect": "..."}}

步骤 4: 创作者评估
  └─ 作者对每条建议做 1-5 评分
  └─ 评估维度：
      ├─ 诊断准确度：这个薄弱点真实存在吗？
      ├─ 建议自然度：改完后像不像这个人物会做的事？
      └─ 实用价值：我会采纳这个建议吗？
```

## 验收标准

### 诊断质量

| 指标 | 合格线 | 良好线 | 优秀线 |
|------|--------|--------|--------|
| 行为链节点标注与人工标注重合 | ≥ 70% | ≥ 85% | 100% |
| 薄弱点中作者认为"真实存在"的比例 | ≥ 60% | ≥ 75% | ≥ 90% |
| LLM 发现的薄弱点中作者未意识到的比例 | ≥ 1/场景 | ≥ 2/场景 | ≥ 3/场景 |

### 建议质量

| 指标 | 合格线 | 良好线 | 优秀线 |
|------|--------|--------|--------|
| "因果补链"建议采纳率（作者评分 ≥ 3） | ≥ 50% | ≥ 70% | ≥ 90% |
| "动机刻深"建议采纳率（作者评分 ≥ 3） | ≥ 50% | ≥ 70% | ≥ 90% |
| 作者评分 > 4 的建议数 | ≥ 1/场景 | ≥ 2/场景 | ≥ 3/场景 |

### 输出产物

1. `results/chain_S1.json` ~ `results/chain_S2.json`（行为链标注）
2. `results/diagnosis_S1.json` ~ `results/diagnosis_S2.json`（薄弱点诊断 + 修改建议）
3. `results/eval.json`（创作者评估）

## 不依赖

纯 Python 脚本 + LLM API。依赖：
- story.yaml（场景描述）
- 场景原文（4_成稿 中的 .md 文件）

## 结果预期

- **S1（咖啡厅重逢）** 最容易产生有价值的诊断。这个场景的"社交陌生人+心理熟人"张力贯穿始终，LLM 如果能识别出"焦糖玛奇朵""毛巾缩回""直接点单"等行为中的动机模糊，说明它理解了核心约束。
- **S2（酒吧表白）** 最难。这个场景的每个行为都有八年积累的情感铺垫，薄弱点不在行为链本身，而在"音乐插入时机""对话密度""原谅→愿意之间的心理转折"等更微妙的地方。如果 LLM 只能诊断动机模糊而无法诊断这些情感节奏问题，说明它对"高潮场景的张力结构"缺乏理解。
- **失败条件**：如果 LLM 标注的行为链与人工标注重合 < 50%，说明 AI 在"将叙事文本分解为行为序列"这个基础能力上就有问题，后续诊断无法展开。
