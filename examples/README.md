# 叙事工程实验室 — 概念验证清单

验证叙事工程的核心假设：**写作能否被系统性地分析、度量和改进。**

每个 PoC 独立于产品代码，在此实验室目录下完成。通过则验证假设，未通过则暴露产品依赖的风险。

---

## 当前假设依赖链

```
起承转合框架 ─── 能否可靠标注叙事结构？ ───→ provider/LLM 分析
      │
      3R 循环    ─── 能否度量改进效果？   ───→ p01 引擎 + 产品 3R 功能
      │
      风格语料库  ─── 能否提取可对比的特征？ ──→ provider/StyleStore
      │
      写作阶段模型 ─── 是否匹配真实写作行为？ ──→ cli/FSM 引擎
```

四个假设中任何一个不成立，产品就有根本性风险。PoC 逐一验证。

---

## PoC 清单

### p01 — 叙事框架对比实验

**验证假设**：不同叙事框架（起承转合、三幕结构、英雄之旅）标注同一篇文章时，结果有可量化的差异和重合。

**为什么重要**：provider 默认使用"起承转合"。如果换一个框架效果更好，或者不同框架标注结果矛盾，就需要重新选择。

**方法**：
- 取 3 篇不同风格的文章（叙事文、议论文、技术说明）
- 对每篇文章，分别用 3 种框架编写分析提示词，调用同一 LLM 标注段落角色
- 对比标注结果：重合率、差异类型、框架间转换成本

**验收标准**：
- 输出每篇文章 × 每个框架的段落标注矩阵
- 标注结果以 JSON 格式输出，可复现
- 不同框架的差异可分类（粒度差异 / 角色定义差异 / 框架不适配）

**不依赖**：任何产品代码。纯 Python 脚本 + LLM API 调用。

---

### p02 — 3R 循环效果度量实验

**验证假设**：运行 3R 循环（Review → Reflect → Rewrite）后，文章的量化评分显著提升，且多轮循环存在收益递减点。

**为什么重要**：整个 p01 引擎和产品 3R 功能假设"3R 有用"。如果 2 轮后不再改善，或者第一轮就改善不明显，需要重新设计循环策略。

**方法**：
- 取 5 篇中等质量的文章
- 对每篇文章运行 5 轮 3R 循环，每轮记录多维度评分
- 统计：首轮提升幅度、收益递减轮次、不同维度改善率差异
- 选 2 篇用不同策略重跑（如 review-heavy / rewrite-heavy），对比效果

**验收标准**：
- 输出每篇文章的 5 轮评分变化曲线
- 标出收益递减拐点
- 不同策略的对比数据

**不依赖**：产品代码。在 p01 引擎基础上扩展，或独立实现精简版 3R。

---

### p03 — 风格可提取性实验

**验证假设**：同一作者的多篇文章共享可识别的风格特征，且这些特征可以用自然语言描述并复现。

**为什么重要**：provider 的 StyleStore 假设"好文章有共同风格特征"。如果同一作者的文章之间差异大于不同作者的文章，风格对比功能就站不住脚。

**方法**：
- 收集 3 位作者各 4 篇文章（共 12 篇）
- 对每篇文章，让 LLM 提取"风格特征描述"（不说好坏，只描述模式）
- 混合后让 LLM 做两件事：
  1. 聚类：把风格描述相近的文章归到同一组
  2. 归因：给定一篇新文章，判断属于哪位作者
- 统计聚类准确率和归因准确率

**验收标准**：
- 同作者文章的聚类准确率 > 60%（随机为 33%）
- 归因准确率 > 50%
- 若两者都低于随机基线，说明"风格"在此实验条件下不可提取，需重新设计

**不依赖**：产品代码。纯 Python 脚本 + LLM API。

## 推荐实施顺序

| 优先级 | PoC | 风险等级 | 预估工作量 | 验证的假设 |
|--------|-----|---------|-----------|-----------|
| 1 | p01 叙事框架对比 | 高 | 小 | 起承转合是否可靠 |
| 2 | p03 风格可提取性 | 高 | 小 | 风格特征是否存在 |
| 3 | p02 3R 效果度量 | 中 | 中 | 3R 循环是否有用 |

p01 和 p03 风险更高（假设不成立则产品需改方向），优先做。p02 风险适中（3R 效果差可以优化策略，不是方向性问题）。

## 工作约定

- 每个 PoC 独立目录，自包含 README 说明实验设计和结果
- 原始数据（文章样本、LLM 输出）随目录提交，确保可复现
- 结果以数据和可视化图表呈现，不依赖主观判断
