## 创作建议

### 基于 p11：卡文时让 AI 生成选项，不做决策

p11 验证了 LLM 可以从一个场景状态生成 5 个真正不同的情节走向。pairwise 盲评中，最高分走向（wins=13）与最低分（wins=3）差距显著——说明走向之间有真实的质量分化。

**怎么用**：写到卡住的时候，把当前人物状态（关系阶段、情绪、冲突）喂给 LLM，让它生成 5 个可能的接下来。你不是让它替你选——你是看它列出的选项，找到那个你自己没想到但觉得"对"的方向。

### 基于 p12：改结构，不是改措辞

p12 发现 LLM 能识别三种常见的结构薄弱点：因果跳跃（动机→行动→后果的链断了）、节奏失衡（事件密度不均）、动机不明（人物行为缺乏理由）。结构改法在 4/6 个薄弱点上优于表面改法（"丰富细节""加强描写"）。

**怎么用**：改稿时不要在措辞层面磨。先跑一遍结构诊断，看看有没有因果跳跃——如果读者会问"为什么ta突然这样"，那就是因果没连上。加一个过渡动作或内心转折，比改三遍形容词有效。

### 基于 p13：巧合要拆，不要藏

p13 验证了 LLM 能从情节中识别出 3-7 个巧合点，并生成人物行为链条驱动的改写。5 次 pairwise 对比中，改写版在因果逻辑和必然性上 3 次优于巧合版。

**怎么用**：写出一个依赖巧合的情节走向后，让 LLM 标注所有"恰好""正巧""突然"的节点。然后逐个问：如果去掉这个巧合，人物自己的性格和之前的行动能自然导致这个结果吗？不是加一个"原来ta早就知道"的解释——是重构行为链条。咖啡厅重逢从"雨天偶遇"改成"闺蜜消息→刻意前往"就是这种改写。

**注意**：不是所有巧合都需要消除。高潮场景（如酒吧表白）的情绪张力本身可能依赖某种"刚刚好"的时刻——强行把每一处巧合都改写成必然，反而会损失情感的自然感。判断标准：**如果这个巧合破坏了读者对因果链的信任，就改。如果它增强了"命中注定"的感觉，可以留。**
