# 叙事工程实验室 — 概念验证清单

验证叙事工程的核心假设：**写作能否被系统性地分析、度量和改进。**

每个 PoC 独立于产品代码，在此实验室目录下完成。案例库统一使用 [qtcloud-write/assets/fiction/](../../assets/fiction/)。

---

## 当前假设依赖链

```
起承转合框架 ─── 能否可靠标注叙事结构？ ───→ p01 叙事框架对比
      │
      3R 循环    ─── 能否度量改进效果？   ───→ p02 3R 效果度量
      │
      风格语料库  ─── 能否提取可对比的特征？ ──→ p03 风格可提取性
      │
      写作阶段模型 ─── 是否匹配真实写作行为？ ──→（待定，依赖前三个结果）
```

---

## PoC 清单

| # | PoC | 验证假设 | 数据来源 | 工作量 |
|---|-----|---------|---------|-------|
| [p01](p01-narrative-frameworks/) | 叙事框架对比 | 起承转合 vs 三幕 vs 情感曲线，标注结果差异多大 | 咖啡厅重逢、酒吧表白、第六章 | 小 |
| [p02](p02-3r-effectiveness/) | 3R 循环效果度量 | 多轮 3R 后评分是否提升，拐点在哪 | 咖啡厅重逢、赏雪谈心、第六章（对照：4 篇成稿） | 中 |
| [p03](p03-style-extraction/) | 风格可提取性 | 同一作者文章能否聚类和归因 | 职场 5 篇 + 校园 3 篇 | 小 |

---

## 实施顺序

p01 → p03 → p02。前两个风险更高（假设不成立则产品需改方向），先做。
