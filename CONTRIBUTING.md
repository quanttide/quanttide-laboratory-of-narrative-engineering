# 贡献指南

实验室的实验是探索性的。出发前不需要完整的实验设计——需要的是一个方向、一个最小可运行的版本、和记录结果的习惯。

## 方法论基础

这个实验室的工作方法建立在以下四种探索和创新框架之上。

### 1. 科学探索循环

```
观察/困惑 → 提出方向性问题 → 做实验 → 记下发生了什么 → 发现新问题 → 重复
```

核心原则：不是"假设 → 验证 → 通过/不通过"，而是"做 → 看发生了什么 → 问新的问题"。

每个实验的 STATUS.md 是这个循环的记录。关键发现和方法论反思是核心产出，通过/不通过是附带的。

### 2. 设计思维（Design Thinking）

```
共情 → 定义 → 构思 → 原型 → 测试
        ↑______________|
```

- **共情**：观察使用者在干什么、卡在哪里
- **定义**：说清楚问题到底是什么
- **构思**：想几个可能的解法
- **原型**：做一个最简陋的版本
- **测试**：给使用者试，看反应——然后重新定义问题

关键：原型要多快有多快、多丑有多丑。设计思维要求先写代码、不是先写文档。

### 3. 双钻模型（Double Diamond）

```
发现(发散) → 定义(收敛) → 发展(发散) → 交付(收敛)
```

| 阶段 | 动作 | 注意 |
|------|------|------|
| 发现 | 收集信息、观察、找问题 | 不要急着收敛 |
| 定义 | 聚焦到一个具体的问题上 | 问题定义清楚再进入下一步 |
| 发展 | 想多个可能的解法、做原型测试 | 不要跳过原型直接定方案 |
| 交付 | 收拢、实现、迭代 | 迭代的节奏要快 |

常见陷阱：在"发现 → 定义"之间反复摇摆，或者在定义未完成时跳过发展直接交付。

### 4. RAT（Research → Attempt → Training）

```
Research（1-2天）: 读相关实验、记笔记、形成直觉
   ↓
Attempt（几小时）: 写最小可运行版本、跑一次、记录结果
   ↓
Training（半天）: 从不完整的结果中学习、决定下一步
   ↓
Repeat
```

Attempt 阶段的产出不是"结论"，是"数据"。结果再烂也有用。

## 做实验

### 步骤

1. 在 `examples/` 下创建 `p##-name/` 目录
2. 写 `README.md`，三两句话说明方向即可
3. 写代码、跑一次、看发生了什么
4. 把发现写在 `STATUS.md` 里
5. 决定下一步：改方向、继续深入、或终止

### 目录结构

```
examples/p##-name/
├── README.md          # 出发方向（草稿级别，不需要完整设计）
├── STATUS.md          # 跑完后记下的结果和反思（核心产出）
├── TODO.md            # 待办（可选）
├── src/               # 代码
├── data/              # 输入输出数据
│   ├── input/
│   └── output/
└── docs/              # 设计记录（可选）
```

### README.md

出发前的草稿，不是蓝图。写清楚即可：

- 想探索什么（一句话）
- 需要什么数据
- 怎么跑
- 预期（如果预期错了，记在 STATUS.md 里）

### STATUS.md

这是实验的真正产出。跑完后记：

- 发生了什么
- 和出发时的预期有哪些一致和偏差
- 发现了什么新问题
- 如果终止，原因是什么

## 技术栈

- **语言**：Python 3.11+
- **包管理**：`uv`（每个实验在各自目录下 `uv sync`）
- **公共工具**：`packages/` 在仓库根目录，包含 LLM 调用（`packages/llm.py`）、JSON 读写（`packages/io.py`）、统计函数（`packages/stats.py`）
- **LLM 模型**：默认 DeepSeek Chat，与 p09/p10 一致
- **不做**：不引入 pandas、matplotlib、PyTorch

### 路径约定

所有 `src/` 中的脚本通过 `GIT_ROOT = Path(__file__).resolve().parents[n]` 定位仓库根，然后 `sys.path.insert(0, str(GIT_ROOT))` 导入 `packages/`。

## 数据资产

实验共用的文本和元数据在 `assets/fiction/`：

| 文件 | 内容 |
|------|------|
| `4_成稿/*.md` | 职场言情系列成稿 14 篇 |
| `3_初稿/*.md` | 对应初稿 |
| `style.yaml` | 克制浪漫风格配置（11 维度 + boundaries） |
| `motif.yaml` | 母题模型（5 母题 + 权重） |
| `story.yaml` | 故事结构（角色 + 情节线 + tensions） |

## 已有的实验
