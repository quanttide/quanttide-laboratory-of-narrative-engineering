# ROADMAP

从探索到实施的行动路线。

## 当前状态

4 个 PoC 完成了概念验证，3R 定义已确定。下一步是填补探索阶段留下的缺口，为产品实施铺路。

---

## 阶段一：验证高风险假设（当前 → 2周）

这些缺口如果不成立，实施方向需要调整。优先处理。

### 1. Reflect 对人类作者的可用性实验

**问题**：全部实验是 AI→AI 闭环。Reflect 的多视角归因对人类来说是启发还是噪音？

**行动**：
- [ ] 取 3 篇真实人类初稿，运行 Review + Reflect
- [ ] 将 Reflect 输出（空隙列表 + 4 视角归因）交给人类作者
- [ ] 记录作者反馈：哪些归因有帮助、哪些是废话、哪些看不懂
- [ ] 输出：Reflect 输出格式的改进清单

**成功标准**：作者认为 > 60% 的归因有帮助

### 2. 跨体裁 3R 适配

**问题**：仅验证了小说体裁。技术文档、营销文案、学术论文是否也适用？

**行动**：
- [ ] 收集 3 类非小说文本（技术文档、产品文案、分析报告）各 2 篇
- [ ] 对每篇运行完整的 3R 循环（Review→Reflect→Rewrite）
- [ ] 评估：空隙类型是否通用？起承转合是否适配？Review 的意图理解是否准确？
- [ ] 输出：跨体裁适配报告 + 需要调整的配置项

**成功标准**：3R 在至少 2 种非小说体裁上有效

### 3. 独立评分模型选型

**问题**：评分必须脱离 3R 链条，但用什么模型、评什么维度、基线在哪？

**行动**：
- [ ] 定义评分维度（与 3R 解耦的独立指标）
- [ ] 对比 3 种方案：LLM 单次打分 / 规则引擎 + LLM / 纯规则
- [ ] 标注 10 篇人工评分作为基线
- [ ] 输出：评分方案选型报告 + 推荐方案

**成功标准**：推荐方案与人工评分的偏差 < 15%

## 阶段二：产品化验证（2周 → 1月）

高风险假设通过后，进入产品化设计。

### 4. Rewrite 情境引导接口

**问题**：如何从 Reflect 归因生成人类可用的改写引导，而非直接让 AI 改写？

**行动**：
- [ ] 设计"归因→引导提示"的转换规则
- [ ] 生成 5 组样本（空隙 + 归因 → 情境引导问题）
- [ ] 人工评估引导质量
- [ ] 输出：情境引导生成器原型 + prompt 模板

### 5. 完成度光谱标定

**问题**：Review 当前仅二元判断（draft/final），产品是否需要更细粒度？

**行动**：
- [ ] 收集 10 篇不同完成度阶段的文本
- [ ] 让 Review 输出 1-5 完成度评分 + 依据
- [ ] 对比人工排序
- [ ] 输出：完成度评分方案 + 依据模板

### 6. 短文本边界实验

**问题**：多短的文本会让 3R 失效？

**行动**：
- [ ] 准备 200/100/50/20 字的文本样本
- [ ] 逐级运行 3R，记录每级的输出质量
- [ ] 输出：最短有效文本长度建议

---

## 阶段三：实施准备（1月 → 2月）

### 7. 混合空隙检测方案评估

- [ ] 对比规则引擎与 LLM 检测的成本和召回率
- [ ] 设计混合方案（规则检常规 + LLM 检复杂）
- [ ] 输出：推荐方案 + 成本估算

### 8. 单语言验证

- [ ] 选 1 种非中文语言（推荐英语），运行完整 3R
- [ ] 输出：语言适配报告
